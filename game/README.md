# game/ — 표현 계층 (뷰)

플랫폼 §6.1·§6.2의 뷰 계층. `sim/`(순수 시뮬)을 소유·구동하고, 시뮬이 내보내는
틱별 이벤트(완료·과열 경고·의뢰 도착/철회·납품 매칭)를 연출로 번역한다.

**분리 규율(플랫폼 §6.2)**: 시뮬은 뷰를 모른다. 뷰만 시뮬을 안다. 이 방향성 덕에
`sim/`은 `godot --headless`로 렌더링 없이 초당 수천 라운드를 돌 수 있다(결정론 테스트·봇).

## 상태: 스캐폴드 (M1 주작업)
M0의 초점은 시뮬 코어 결정론과 재미 게이트 검증이며, 뷰는 다음과 같이 M1에서 채운다.

계획된 씬 흐름(플랫폼 §6.2):
```
boot → title → map ⇄ (forge | ledger) → round(브리핑 → 플레이 → result) → map
                       └ 미언락 거래처 2+ 입장 시 페이월 오버레이(수익 §2)
```
- `scenes/` — 위 씬들
- `view/` — 시뮬 이벤트 → 애니메이션·사운드 바인딩

M0 뷰 최소 목표(다음 작업): 도형 스프라이트로 시설·일꾼·의뢰 칸을 그리고, 탭 조작
(운반·착수·수납·납품)을 시뮬 커맨드로 변환, 유휴 일꾼 수 상시 표시.
지금은 `tools/verify_replay.gd`가 시뮬을 헤드리스로 구동하는 유일한 실행 진입점이다.
