## 1. 목적과 소유 범위

이 문서는 거래처 1곳과 5개 라운드로 구성된 MVP 수직 슬라이스에서 사용자가 무엇을 하고, 시스템으로부터 어떤 응답을 받으며, 어떤 상태로 끝나는지를 정의한다. 정상 흐름뿐 아니라 실패, 중단, 복귀, 저장 오류, 설비 부족, 골드 부족을 동등한 제품 경로로 다룬다.

이 문서가 Source of Truth인 항목은 다음과 같다.

- MVP 최초 실행부터 수직 슬라이스 완료까지의 사용자 여정
- 각 시나리오의 진입 조건, 사용자 행동, 시스템 응답, 종료 상태
- 실패·취소·중단·복구 경로에서 지켜야 하는 사용자 경험 보장
- 시나리오별 필요한 화면 상태와 사용자에게 알려야 할 정보
- Android/Google Play 무료 MVP에서 앱 내부 상업 UI를 두지 않는 경험 경계

이 문서는 다음을 소유하지 않는다.

- 제작·강화·판정·진행·경제의 게임 규칙
- 시설·아이템·레시피·라운드·밸런스 값
- 화면 레이아웃과 컴포넌트 배치
- 세이브 파일 형식, 원자적 쓰기, 복구 알고리즘
- 앱 스토어 가격과 배포 정책
- 아트 에셋, 오디오 파일, 햅틱 구현 API

규칙이나 수치는 상위 문서를 참조하며 이 문서에 복제하지 않는다. 이 문서의 예시는 화면 문구와 흐름을 설명하기 위한 비규범적 예시다.

## 2. MVP 범위와 사용자 약속

### 2.1 검증 범위

MVP는 다음 경험을 한 번의 작은 제품 루프로 검증한다.

1. 첫 거래처와 5개 라운드를 순서대로 진행한다.
2. 기본 제작에서 시작해 병렬 작업, 수납, 과열과 트리아지, 예고와 비축을 익힌다.
3. 마지막 라운드에서 결정론적 `+1` 강화 공정을 완주한다.
4. 라운드 결과로 별과 골드를 받고, 필요한 설비를 상점에서 구매한다.
5. 마지막 라운드 클리어로 첫 계약을 체결하고 MVP 완료 상태에 도달한다.
6. 실패하거나 앱을 떠나도 합리적인 비용으로 즉시 재도전·복귀할 수 있다.

MVP의 목표는 장기 확장 진행을 미리 증명하는 것이 아니라 다음 핵심 가설만 검증하는 것이다.

- 탭 지시와 일꾼 동시성만으로 스케줄링의 재미가 생기는가.
- 과열과 의뢰 철회가 억울한 처벌이 아니라 읽을 수 있는 선택으로 느껴지는가.
- 브리핑 예고 → 비축 → `+1` 강화의 계획 루프가 클러치를 만드는가.
- 실패 후 재도전과 중단 후 복귀가 마찰 없이 이어지는가.
- 설비 구매가 통행료가 아니라 다음 선택을 여는 성장으로 느껴지는가.

### 2.2 UX 불변 조건

- 플레이 중 모든 생산 지시는 한 번 또는 두 번의 탭으로 끝난다.
- 사용자가 직접 확인하지 않은 파괴적 행동은 실행하지 않는다.
- 앱 이탈은 즉시 일시정지로 취급한다. 백그라운드 시간은 라운드 시간에 포함하지 않는다.
- 중단 저장이 정상이라면 마지막 저장 지점에서 동일한 상태로 이어진다.
- 저장 오류를 조용히 무시하거나 새 게임으로 자동 초기화하지 않는다.
- 라운드 실패는 짧고 명확하게 보여주며 `다시`를 가장 가까운 다음 행동으로 제공한다.
- 의뢰 철회는 감점이나 보유 아이템 손실처럼 표현하지 않는다.
- 과열 이외의 실패 피드백은 최상급 위험 신호처럼 표현하지 않는다.
- 부족한 설비와 부족한 골드는 서로 다른 문제로 설명하고 각각의 해소 경로를 제공한다.
- 현재 Android/Google Play 무료 MVP에는 실물화폐 가격, 현금 구매, 구매 복원, 페이월, `정식판 열기`를 노출하지 않는다. 후속 유료 스테이지가 승인되기 전에는 이를 암시하는 자리도 만들지 않는다.
- 게임·저장·복구와 개인정보·오픈소스 고지 열람은 네트워크 없이 가능하다. 공개 웹페이지나 외부 메일 앱의 실패는 게임을 차단하지 않는다.

## 3. 사용자와 상태 모델

### 3.1 사용자 유형

| 유형 | 상태 | 주요 목표 |
|---|---|---|
| 신규 사용자 | 프로필과 라운드 스냅샷이 없음 | 설명을 읽기보다 첫 행동을 하며 기본 루프를 이해한다 |
| 진행 사용자 | 프로필은 있고 진행 중 스냅샷은 없음 | 다음 라운드나 상점으로 이어간다 |
| 중단 복귀 사용자 | 프로필과 유효한 진행 중 스냅샷이 있음 | 중단했던 라운드를 잃지 않고 이어간다 |
| 실패 후 사용자 | 직전 라운드 결과가 0성 | 원인을 이해하고 즉시 다시 시도한다 |
| 막힌 사용자 | 설비 또는 골드가 부족함 | 막힌 이유와 가장 가까운 해소 행동을 이해한다 |
| 복구 사용자 | 프로필 또는 스냅샷 로드·저장에 문제가 있음 | 안전한 데이터 범위를 알고 명시적으로 복구 경로를 선택한다 |
| MVP 완료 사용자 | 다섯 번째 라운드를 클리어해 첫 계약을 체결함 | 결과를 확인하고 완료 상태에서 재플레이한다 |

### 3.2 사용자에게 노출되는 상태

| 상태 | 의미 | 반드시 알려야 하는 것 |
|---|---|---|
| 신규 | 저장된 진행 없음 | 새로 시작한다는 사실 |
| 계속 | 프로필은 정상, 진행 중 라운드 없음 | 현재 별·골드·다음 라운드 |
| 이어하기 | 유효한 라운드 스냅샷 있음 | 라운드 ID와 중단 상태가 남아 있음 |
| 잠김 | 선행 라운드 미클리어 | 잠금 해제 조건 |
| 설비 부족 | 라운드의 의뢰를 현재 설비로 이행 불가 | 부족한 설비와 상점 경로 |
| 골드 부족 | 선택한 구매를 결제할 게임 내 골드 부족 | 현재 골드, 필요 골드, 골드를 얻는 방법 |
| 저장 중 | 결과·구매·스냅샷의 영속화가 끝나지 않음 | 화면을 닫아도 되는지 여부 |
| 저장 경고 | 최근 변경을 안전하게 기록하지 못함 | 보존된 범위, 위험 범위, 재시도 행동 |
| 복구 필요 | 저장물을 자동으로 신뢰할 수 없음 | 어떤 데이터가 문제인지와 비파괴적 선택지 |
| 완료 | MVP 계약 체결 완료 | 수직 슬라이스가 끝났으며 재플레이 가능함 |

  ├─ 0성 ─▶ [다시 / 지도]

## 5. 시나리오 카탈로그

### 5.1 부팅·진입·복귀

| ID | 시나리오 | 진입 조건 | 사용자 행동과 시스템 응답 | 종료 상태 / 필요한 화면 |
|---|---|---|---|---|
| `UX-BOOT-01` | 최초 실행 | 프로필·스냅샷 없음 | 로고 뒤에 새 게임 진입점을 표시한다. 사용자가 시작하면 프로필 생성이 완료된 뒤 지도로 이동한다. | 신규 프로필 / 타이틀, 지도 |
| `UX-BOOT-02` | 정상 계속하기 | 정상 프로필, 스냅샷 없음 | 현재 거래처 진행 요약과 `계속`을 표시한다. 탭 시 마지막 허브인 지도로 이동한다. | 기존 진행 유지 / 타이틀, 지도 |
| `UX-BOOT-03` | 중단 라운드 이어하기 | 정상 프로필, 유효 스냅샷 있음 | 라운드와 중단 여부를 알리고 `이어하기`를 1차 행동으로, `지도에서 보기`를 2차 행동으로 제공한다. 지도에서 보기로 가도 스냅샷은 삭제하지 않는다. | 스냅샷 유지 또는 플레이 재개 / 타이틀, 이어하기 카드, 플레이 |
| `UX-BOOT-04` | 콜드 스타트 중 로드 | 로드가 즉시 끝나지 않음 | 진행 표시를 보이고 입력을 이중 처리하지 않는다. 목표 시작 시간 초과 시 정지 화면 대신 재시도 가능한 오류로 전환한다. | 로드 완료 또는 복구 안내 / 부팅 상태 |
| `UX-BOOT-05` | 기본 설정 적용 | 정상 설정 데이터 있음 | 타이틀을 표시하기 전에 저장된 표시·소리·햅틱 설정을 적용해 화면·소리가 뒤늦게 바뀌지 않게 한다. | 설정이 적용된 첫 화면 / 부팅 상태 |

### 5.2 지도·해금·입장

| ID | 시나리오 | 진입 조건 | 사용자 행동과 시스템 응답 | 종료 상태 / 필요한 화면 |
|---|---|---|---|---|
| `UX-MAP-01` | 다음 라운드 선택 | 해금되고 설비가 충족된 라운드 | 라운드 노드를 탭하면 최고 별, 학습 주제, 입장 가능 상태를 보여준다. `브리핑 보기`로 진행한다. | 브리핑 전 상태 / 지도, 입장 시트 |
| `UX-MAP-02` | 선행 라운드 잠김 | 이전 라운드 미클리어 | 잠긴 노드는 탭 가능하되 입장은 불가하다. `이전 라운드에서 ★1 이상`처럼 해금 조건을 한 문장으로 표시한다. | 지도 유지 / 잠금 상태 입장 시트 |
| `UX-MAP-03` | 설비 부족으로 입장 불가 | 라운드는 해금됐지만 필수 설비가 없음 | 부족 설비 이름과 그 설비가 필요한 이유를 표시한다. `상점에서 보기`가 1차 행동이고 `닫기`가 2차 행동이다. 일꾼 수나 권장 사양은 강제 조건처럼 표시하지 않는다. | 상점 또는 지도 / 설비 부족 입장 시트 |
| `UX-MAP-04` | 중단 라운드가 있는 상태에서 다른 라운드 선택 | 유효 스냅샷 존재 | 기존 라운드를 보존한 채 새 라운드를 동시에 시작할 수 없다. `중단 라운드 이어하기`와 `중단 라운드 포기`를 제공하며 포기는 확인 뒤 스냅샷만 삭제한다. | 기존 라운드 재개 또는 새 브리핑 / 확인 대화상자 |
| `UX-MAP-05` | 완료 라운드 재플레이 | 해당 라운드 최고 기록 존재 | 최고 별과 재플레이 가능을 표시한다. 에너지·입장권·감소 보상 경고는 없다. | 브리핑 / 지도, 입장 시트 |
| `UX-MAP-06` | MVP 전체 진행 확인 | 어느 진행 상태든 | 첫 거래처와 다섯 라운드만 보인다. 향후 거래처를 가짜 잠금 노드나 구매 유도로 표시하지 않는다. | 지도 유지 / 지도 |

## 6. Android 무료 MVP의 앱 내부 상업 UX 경계

`BUS-MVP-001`에 따라 첫 공개 MVP는 Android/Google Play 무료 앱이며, R1~R5 전체를 결제 없이 제공한다. 아래 경계는 현재 MVP에만 적용한다. 후속 유료 스테이지는 승인된 기능이 아니며, 승인될 경우 `BUS-MVP-001`과 이 문서를 먼저 개정한 뒤 별도의 구매·복원 사용자 여정을 추가한다.

- 현재 R1~R5 사용자 여정에는 IAP, 페이월, 체험 경계, entitlement, 실물화폐 가격, 현금 구매, 구매 복원, 스토어 오류 흐름을 두지 않는다.

- `정식판`, `잠금 해제`, `구매 예정`처럼 후속 구매를 암시하는 문구, 빈 화면, CTA를 넣지 않는다.

- `UI-MVP-001`, `TECH-MVP-001`, `QA-MVP-001`은 현재 무료 MVP를 위해 앱 내부 상업 분기를 만들지 않는다.
- 후속 유료 스테이지가 승인되기 전에는 미래 구매 흐름의 상태·오류·복구 계약을 선제 정의하지 않는다.

## 8. 예외 우선순위와 피드백 규칙

동시에 여러 상태가 발생하면 다음 순서로 사용자에게 처리한다.

1. 데이터 유실 가능성 또는 저장 실패
2. 과열 임박처럼 현재 아이템 손실 가능성이 있는 인라운드 정보
3. 라운드 기한 임박
4. 작업 완료와 의뢰 도착
5. 입력 거절, 골드 부족, 매칭 없음 같은 국소 오류
6. 장식성 연출

상위 우선순위가 하위 피드백을 영구히 지워서는 안 된다. 예를 들어 결과 저장 실패 중 별 연출은 건너뛸 수 있지만, 저장 재시도와 이번 결과 요약은 남아야 한다.

## 9. 검증 방법

`QA-MVP-001`은 최소한 다음을 검증해야 한다.

- 신규 사용자가 외부 설명 없이 R1을 시작하고 제작·납품 한 사이클을 완수한다.
- R1~R5의 새 학습 항목이 한 번에 하나씩 도입되며 R5에서 `+1` 강화 의뢰를 완주할 수 있다.
- 0성 결과에서 두 번 이내의 탭으로 재도전을 시작할 수 있다.
- 일시정지 중 모든 시뮬레이션 상태가 멈추고 관찰 외 지시는 반영되지 않는다.
- 백그라운드·강제 종료 후 동일 스냅샷에서 복귀하며 오프라인 경과 시간이 반영되지 않는다.
- 프로필 손상, 스냅샷 손상, 버전 불일치, 결과 저장 실패를 각각 독립적으로 재현한다.
- 저장 오류에서 자동 초기화·중복 보상·부분 구매가 발생하지 않는다.
- 설비 부족과 골드 부족을 테스트 참가자가 서로 다른 문제로 설명할 수 있다.
- 모든 핵심 상태가 색이나 소리 하나만 제거해도 판별된다.
- Android/Google Play 무료 MVP 빌드에 현금 결제·실물화폐 가격·페이월·구매 복원 문구와 UI가 존재하지 않는다.
- 기내 모드에서 개인정보처리방침·오픈소스 라이선스·지원 이메일과 설치 버전을 모두 읽을 수 있다.
- 메일 앱·브라우저 있음/없음과 전환 실패를 재현해도 진행·설정·호출 화면이 유지되고 주소 복사가 가능하다.
- 앱에 표시된 정책 원문과 `REL-PLAY-001`의 canonical GitHub Pages 원문이 일치한다.

## 10. 미결 사항

현재 UX 소유 범위의 미결 사항은 없다. 저장 주기와 복구 알고리즘은 `TECH-MVP-001`, 수치 튜닝은 `BAL-MVP-001`이 소유하며 이 문서의 사용자 보장을 바꾸지 않는다.

## 11. 완료 조건

이 문서는 다음을 모두 만족할 때 `Review`로 전환할 수 있다.

- `CNT-MVP-ROUND-001`의 5개 라운드와 모든 기능이 시나리오에 연결된다.
- 정상·실패·중단·복귀·저장 오류·설비 부족·골드 부족 경로에 빈 상태가 없다.
- 각 시나리오에 진입 조건, 사용자 행동, 시스템 응답, 종료 상태가 있다.
- 게임 규칙·수치·레이아웃·기술 구현을 중복 소유하지 않는다.
- `UI-MVP-001`의 모든 MVP 화면이 최소 하나의 시나리오를 충족하고, 모든 시나리오가 화면 또는 비시각 시스템 응답에 매핑된다.
- 현재 Android/Google Play 무료 MVP에는 앱 내부 상업 UX가 없고, 후속 유료 스테이지 승인 시 문서를 먼저 개정한다는 경계가 명시된다.
- `UX-LEGAL-*`의 오프라인 열람, 외부 앱 전환·실패와 fallback이 `UI-MVP-001`의 화면 상태에 모두 매핑된다.
