## 1. 이 문서가 답하는 질문

- 첫 출시에서 실제로 무엇을 만들고 무엇을 만들지 않는가?
- 핵심 재미를 검증하는 데 필요한 최소 콘텐츠는 무엇인가?
- MVP를 “완료”했다고 판정하려면 어떤 증거가 필요한가?
- 범위 추가 요청을 어떤 기준으로 거절하거나 교체하는가?

## 2. MVP 한 줄 정의

**거래처 한 곳의 5개 라운드에서 기본 제작부터 예고·비축·결정론적 +1 강화까지 경험하고, 납품 결과가 원정 결과로 돌아오는 모바일 세로형 오프라인 수직 슬라이스.**

MVP는 축소된 풀게임이 아니다. 아래 가설을 검증하는 데 직접 필요하지 않은 미래 콘텐츠와 확장 기반은 만들지 않는다.

## 3. 검증할 제품 가설

| ID | 가설 | MVP에서 필요한 관찰 |
|---|---|---|
| H1 | 탭 지시만으로도 대장간 사장 판타지가 전달된다. | 플레이어가 직접 조작보다 작업 순서를 핵심 행위로 설명한다. |
| H2 | 제한된 일손과 복수 공정의 병목 관리가 재미를 만든다. | 숙련될수록 유휴 시간과 막힘을 스스로 줄인다. |
| H3 | 모든 의뢰를 처리하지 못하는 상황에서 트리아지가 의미 있다. | 낮은 가치 의뢰를 의도적으로 포기하고 이유를 설명한다. |
| H4 | 예고와 비축이 반응 속도보다 계획을 보상한다. | 후반 예고 의뢰에 필요한 자원을 미리 준비한다. |
| H5 | 결정론적 +1 강화가 한 판의 절정을 만든다. | 강화 실패 확률 없이도 마감 전 완주에 긴장과 성취를 느낀다. |
| H6 | 원정 결과 피드백이 납품 점수에 정서적 의미를 준다. | 결과를 단순 점수보다 원정대의 성패로 기억한다. |
| H7 | 실패 후 즉시 다른 계획으로 재도전하고 싶어진다. | 실패 원인을 말하고 같은 라운드를 다시 시도한다. |

## 4. 포함 범위

### 4.1 플레이 형태

- 싱글 플레이
- Android 모바일 세로 화면, 한 손 중심 입력
- 첫 실행부터 R5 완료, 저장·복구와 앱 내부 법적 고지까지 오프라인에서 동작
- 라운드 중 일시정지, 앱 이탈 저장, 재개와 재도전
- 한국어 단일 언어

첫 공개 빌드는 **Android 단독**이다. Android 외 플랫폼은 이번 MVP의 설계·빌드·QA 범위에 포함하지 않는다. 배포 스토어와 획득 가격은 `BUS-MVP-001` §2의 승인 결정을 따른다.

설치된 앱은 서버·계정·인터넷 권한·원격 SDK 없이 완결되어야 한다. 공개 개인정보처리방침과 지원 페이지는 Google Play 등록을 위해 GitHub Pages로 제공하되, 앱은 같은 원문을 로컬에 포함해 사이트 장애나 오프라인 상태에서도 고지를 열 수 있어야 한다. 공개 정보와 개인정보 경계는 `REL-PLAY-001`이 소유한다.

### 4.2 콘텐츠 경계

- 거래처/스테이지: **1곳 — 새싹 원정대**
- 라운드: **5개**
- 하나의 짧은 학습 곡선 안에서 다음 경험을 순서대로 공개한다.
  1. 공급 → 변환 → 제작 → 납품
  2. 병렬 작업, 시설 비우기와 인벤토리
  3. 과열 위험과 의뢰 트리아지
  4. 브리핑 예고와 선행 비축
  5. 결정론적 **+1 강화**를 포함한 종합 마감
- MVP 공정과 위 5개 라운드에 실제 사용되는 재료·장비·기자재만 포함
- 원정대 한 팀과 라운드 결과에 필요한 최소 캐릭터·배경·연출만 포함

아이템 이름, 공정 관계와 시설 목록은 각각 `CNT-ITEM-001`, `CNT-RECIPE-001`이 소유하고 이 문서에 복제하지 않는다.

### 4.3 핵심 시스템

- 브리핑과 라운드 시작
- 탭 기반 작업 지시와 일꾼 배정
- 복수 시설의 병렬 진행과 병목
- 재료 변환, 재료 합성, 장비 제작, +1 강화
- 한정 인벤토리와 완성품 회수
- 사전에 보이는 과열 위험과 재료 손실
- 복수 의뢰, 인내 만료, 의도적 포기와 납품
- 라운드 점수, 성공/실패, 별 결과와 원정 리포트
- 정산 골드와 MVP 안에서 실제 필요한 최소 구매/해금
- 5개 라운드의 순차 진행, 최고 성과 기록과 재도전
- 로컬 프로필 저장, 진행 중 라운드 복귀, 손상 시 안전한 복구 또는 명시적 재시작
- 최소 설정: 음악·효과음 독립 조절, 햅틱 켬/끔, 색 보조, 큰 글씨

세부 동작은 규칙 문서, 확정 수치는 `BAL-MVP-001`, 예외 경로는 `UX-MVP-001`이 소유한다.

### 4.4 필요한 화면 범주

- 타이틀/계속하기
- MVP 진행 선택
- 라운드 브리핑
- 라운드 플레이와 일시정지
- 결과/정산
- 최소 구매 또는 해금
- 설정과 저장 오류 안내

이는 범위 확인용 범주이며 실제 화면 수, 통합 여부와 레이아웃은 `UI-MVP-001`이 소유한다.

### 4.5 출시 운영

- 한 번 설치한 빌드만으로 5개 라운드를 끝까지 플레이 가능해야 한다.
- 서버, 계정과 상시 네트워크 없이 핵심 검증이 가능해야 한다.
- R1~R5 전체를 한 설치물에 포함한다.
- 현재 상업 기능의 포함·제외 경계는 `BUS-MVP-001` §3을 따른다.
- 지원 이메일, 개인정보처리방침, Data safety와 Google Play 등록 산출물은 `REL-PLAY-001`을 따른다.
- 후속 스테이지의 유료 해금 가능성은 `BUS-MVP-001`이 허용하는 미래 선택지일 뿐, 현재 콘텐츠·화면·데이터·SDK 범위가 아니다.

## 5. 명시적 제외 범위

아래 항목은 “나중에 필요할 수 있음”을 이유로 골격이나 빈 화면까지 만들지 않는다.

### 5.1 콘텐츠와 진행

- 두 번째 이후 거래처와 R1~R5 밖의 라운드
- 방어구·장신구 카테고리와 이를 위한 전용 제작 시설
- +1을 넘는 강화 단계
- 부품 제작, 다인 조립과 거대 장비
- 거래처별 특수 규칙, 단골 의뢰, 복합 던전 속성
- 전체 장부·도감·업적·수집 메타
- 엔드게임, 반복 퀘스트, 일일 과제와 기간 이벤트

### 5.2 제품과 서비스

- 계정, 서버, 클라우드 저장과 기기 간 동기화
- 리더보드, PvP, 협동, 소셜 기능
- 광고 SDK, 분석 SDK, 푸시 알림과 라이브 운영 도구
- 현재 MVP용 IAP·DLC·꾸미기 상점·시즌 패스·체험판 페이월
- 영어를 포함한 다국어 현지화
- 게임패드·PC·콘솔·가로 화면·태블릿 전용 UI

### 5.3 제작 품질의 비목표

- 모든 기기에 대한 완벽한 연출 차등화
- 적응형 다층 음악, 보이스오버와 대량 컷신
- 미래 콘텐츠용 범용 에디터나 플러그인
- 출시 후 확장을 가정한 분산 서비스 또는 범용 백엔드
- 구현되지 않은 전체 게임의 밸런스 시뮬레이션

## 6. 필수 사용자 경로

MVP는 최소한 다음 경로를 끝까지 지원한다.

1. 첫 실행 → 새 게임 → 학습 → 첫 라운드 완료
2. 다음 라운드 해금 → 준비/구매 → 진입 → 완료
3. 과열 손실, 재화 부족, 시설 부족과 의뢰 만료의 이해 가능한 피드백
4. 라운드 실패 → 원인 확인 → 재도전
5. 라운드 중 일시정지 → 복귀
6. 앱 백그라운드/강제 종료 → 진행 중 라운드 복구
7. 프로필 없음, 저장 손상과 저장 버전 불일치의 안전한 처리
8. 5번째 라운드 완료 → MVP 종료 상태와 다시 플레이할 경로
9. 설정 → 정보·법적 고지 → 오프라인 개인정보처리방침·오픈소스 고지·지원 주소 확인

현재 무료 MVP에서는 스토어에서 앱을 획득한 이후에도 결제·구매 복원 경로가 존재하지 않는다. 후속 유료 콘텐츠가 별도 승인될 때에만 해당 버전의 UX·UI·기술·QA 명세를 새로 작성한다.

## 7. 선행 결정과 하위 문서에 주는 출력

### 입력으로 받은 결정

- `GDD-001`: 사장 판타지, 계획 중심 실력, 결정론, 가벼운 실패, 중단 가능성

### 하위 문서에 주는 결정

- 콘텐츠 상한: 거래처 1곳, 라운드 5개
- 학습 종료점: 예고·비축·결정론적 +1 강화
- 지원 경험: Android 세로형 오프라인 싱글 플레이
- 배포·가격 정책 입력: `BUS-MVP-001` §2
- 공개 지원·개인정보 경계 입력: `REL-PLAY-001`
- 영속 범위: 최소 진행·구매·설정·중단 라운드
- 명시적 비목표와 삭제 기준

## 8. 정상·예외·경계 상태

### 정상 상태

- 신규 플레이어가 1라운드부터 5라운드까지 순차 진행한다.
- 클리어한 라운드를 다시 플레이해 더 나은 결과를 얻는다.
- 라운드 사이에 필요한 최소 성장 선택을 하고 다음 라운드로 간다.

### 예외 및 경계 상태

- 아직 필요한 구매나 설비가 없어 다음 라운드를 시작할 수 없음
- 골드 부족으로 권장 성장을 구매할 수 없음
- 진행 중 앱 중단, 프로세스 종료와 다음 실행 복귀
- 저장 데이터 없음, 손상, 지원하지 않는 버전
- 마지막 라운드 완료 후 후속 콘텐츠가 없음

각 상태의 화면 행동과 복구 경로는 `UX-MVP-001` 및 `UI-MVP-001`에서 정의한다.

## 9. 범위 변경 규칙

새 항목은 다음 조건을 모두 만족할 때만 추가할 수 있다.

1. §3의 가설 중 하나를 검증하는 데 필요하다.
2. 기존 포함 항목으로 같은 증거를 얻을 수 없다.
3. 구현·아트·QA·저장·UX 비용이 함께 산정되었다.
4. 동등한 비용의 기존 범위를 제거하거나, 출시 기준 변경을 명시적으로 승인받았다.

조건을 만족하지 않는 아이디어는 이 저장소의 MVP 문서에 예약석, 빈 스키마 또는 TODO로 남기지 않는다. 필요해지는 시점에 Git 이력이나 별도 기획 작업에서 다시 제안한다. 후속 유료 스테이지 가능성도 예외가 아니므로 현재 빌드에 빈 상품·권한 필드·가짜 잠금 화면을 만들지 않는다.

## 10. 검증 방법

- 모든 콘텐츠 ID가 5개 라운드 중 하나에서 실제 사용되는지 정적 검사한다.
- 제외 범위의 화면, 데이터 스키마, 플랫폼 SDK와 테스트가 저장소에 생기지 않았는지 리뷰한다.
- 처음 보는 외부 플레이테스터가 별도 구두 설명 없이 5개 라운드를 진행하도록 관찰한다.
- H1~H7마다 행동 관찰, 인터뷰 질문과 로그 없이 확인 가능한 결과를 `QA-MVP-001`에 연결한다.
- 저장/복귀와 오류 경로를 실제 기기에서 강제 중단·데이터 손상으로 검증한다.
- 기내 모드에서 게임 전 경로와 앱 내부 법적 고지를 검증하고, 최종 AAB에 인터넷 권한·원격 SDK가 없음을 검사한다.

## 11. 완료 조건

- 5개 라운드가 임시 치트나 개발자 개입 없이 처음부터 끝까지 플레이된다.
- 기본 제작, 병렬 작업, 과열/트리아지, 예고/비축, +1 강화가 각기 도입되고 마지막 라운드에서 함께 작동한다.
- 모든 필수 사용자 경로가 정의되고 차단 버그 없이 통과한다.
- 플레이테스트 결과로 H1~H7 각각을 지지하거나 반박할 근거가 기록된다.
- 포함되지 않은 풀게임 기능이 코드·에셋·데이터·화면·문서에 남아 있지 않다.
