## 1. 화면 목록과 플로우

| 화면 | 목적 한 줄 |
|---|---|
| 타이틀 | 저장 데이터 확인 후 한 탭으로 가도 지도 진입 |
| 가도 지도 (허브) | 진행 상황(별·계약·개방 요건)을 한눈에 보고 다음 라운드를 고른다 |
| 상점 | 골드를 성장 축(일꾼·시설·슬롯·수납·도구)에 배분한다 |
| 배치 모드 | 시설의 설치·재배치 — 플레이 화면과 동일 격자에서 |
| 장부 | 계약 현황과 원정 기록의 컬렉션 열람 |
| 설정 | 소리·햅틱·접근성·데이터 |
| 라운드 입장 시트 | 입장 요건 확인과 입장 결정 (지도 위 하단 시트) |
| 브리핑 | 시간이 멈춘 채 계획을 세우고, 탭으로 개시 |
| 플레이 | 게임의 전부 — 탭 지시로 만들고 납품 |
| 일시정지 시트 | 관찰·계획 허용, 재시작·이탈 |
| 결과 | 별 판정의 서사적 연출과 정산, 즉시 재도전 |

```
타이틀 ─▶ 가도 지도 ◀──────────────────────────────┐
            │ (하단 탭바: 가도·상점·장부·설정)        │
            │   상점 ─▶ 배치 모드                    │
            ▼                                       │
      라운드 입장 시트 ─▶ 브리핑 ─▶ 플레이 ─▶ 결과 ──┤
                            ▲          │ ▲     │    │
                            │          ▼ │     │    │
                            │      일시정지    │    └ [가도로]
                            └──(재도전: 브리핑 생략)─┘  [다시]
```

- 메타 화면(지도·상점·장부·설정)은 가도 지도를 허브로 하는 **하단 탭바 4개**로 묶는다. 라운드 사이는 "머리의 시간"(progression §2)이므로 스크롤·상세 열람을 허용한다. **무스크롤 원칙은 플레이 화면에만 적용된다.**

### 2.2 엄지 도달 존 분석 (오른손 기본)

- **홈 존(하단 1/3, 도달 용이)**: 납품 카운터(가장 빈번한 목적지), 인벤토리 트레이, 유휴 벤치. 납품 카운터는 **우하단 최적점**에 대형(높이 2셀분)으로 둔다.
- **작업 존(중단, 도달 보통)**: 시설 격자. 하단 2행이 상단 2행보다 닿기 쉬우므로, **재배치 자유(progression §7)** 가 곧 접근성 도구다 — 플레이어가 자주 쓰는 시설을 아래 행으로 내린다.
- **읽기 존(상단 1/3, 도달 어려움)**: HUD와 의뢰 밴드. 의뢰 카드는 **읽는 것이지 조작하는 것이 아니므로**(고르거나 거절 불가, mechanics §9) 상단 배치가 옳다. 오탭 비용이 큰 쓰레기통도 엄지에서 먼 **격자 우상단 모서리 고정**이다.
- 일시정지 버튼은 우상단 — 빈도가 낮고 오탭 비용이 0(즉시 재개 가능)이므로 먼 곳이 허용된다.

### 2.3 유휴 일꾼의 상시 표시 (mechanics §4 — 확정)

**이중 표현**: ① 하단 밴드 좌측에 숫자 배지 `👷 2/4`(유휴/총원) 상시 표시 ② 유휴 일꾼은 배지 옆 **대기 벤치에 실제로 서 있다**(디제틱). 착수 탭 시 벤치의 일꾼이 해당 시설로 달려가 인과가 보인다. 유휴 0이면 배지가 회색+빈 벤치 — 색이 아니라 **벤치의 비어 있음(형태)** 이 1차 신호다.

### 2.4 무스크롤 하의 공간 전략 (10종 + 중복 설치)

- **설치 슬롯의 UI 상한 = 14칸** (격자 16칸 − 고정 2칸). 시설 10종 각 1대 + 중복 4대까지 수용한다. 밸런스 문서의 슬롯 확장 축은 이 상한 안에서 가격·순서만 정한다. *(밸런스 문서에 부과하는 표현 제약 — §11 대조표 참조.)*
- **셀 크기·격자 위치는 보유량과 무관하게 불변**(약 88×88pt). 시설이 적은 초반에도 확대하지 않는다 — 근육 기억이 성장 내내 유지된다. 미구매 슬롯은 자물쇠 실루엣으로 구매 축을 광고한다.
- 중복 시설은 병합하지 않고 **각각 독립 셀**이다. 산출물·과열·진행이 개별이기 때문이다.
- 셀 하나가 표현하는 것: 시설 아이콘 / 진행 게이지(테두리 링) / 완료 산출물(셀 위 아이템) / 과열 상태(§4.3) / 배정 일꾼(작업 중이면 셀 안에서 망치질).

### 2.5 인벤토리 = 상시 노출 트레이 (확정: 서랍 아님)

- **하단 고정 트레이 2행×6열 = 12칸이 항상 보인다.** 서랍·패널을 확정 배제한다: 인벤토리는 운반 문법(탭-탭)의 출발지이자 목적지인데, 서랍을 여는 순간 3탭이 되어 리듬(§3.4)이 죽는다.
- **인벤토리 용량의 UI 상한 = 12칸.** 미구매 칸은 자물쇠 표시(시작 용량·확장 가격은 밸런스 문서, 이 상한 안에서).
- 수납(원탭)된 산출물은 좌상단부터 자동 채움. 꺼내기는 슬롯 탭 → 목적지 탭.

## 3. 탭 인터랙션 상세

### 3.1 운반의 2탭 상태 머신

```
[중립] ──유효 출발지 탭──▶ [선택됨] ──유효 목적지 탭──▶ 운반 실행 ──▶ [중립]
   ▲                        │ │ │
   │◀── 출발지 재탭 (취소) ──┘ │ └── 다른 유효 출발지 탭 → 선택 교체
   │◀── 빈 바닥 탭 (취소) ─────┘      무효 목적지 탭 → 선택 유지 + 흔들림
```

- **선택 상태의 시각 표시**: 출발지에 백색 펄스 링 + 들린 아이템이 살짝 떠오름. 동시에 **유효 목적지 전부에 브리딩 테두리 + ▲배지**가 켜진다 — "레시피에 없는 조합은 받지 않는다"(mechanics §6.3)를 조명으로 가르치는 장치다. 무효 대상은 시각 변화 없음(딤 처리 안 함 — 판 전체의 판독성 유지).
- **취소**: 출발지 재탭 또는 빈 바닥 탭. **시간 경과 자동 해제는 없다** — 일시정지 관찰·계획(mechanics §11)과 충돌하기 때문.
- **오탭 처리**: 무효 목적지를 탭하면 **선택은 유지**되고 대상이 짧게 흔들린다(0.2초). 선택이 풀리면 다시 2탭을 해야 하므로, 유지가 리듬에 옳다.
- 선택 상태에서 시설 탭은 항상 "목적지" 해석이 우선한다. **착수 탭은 중립 상태에서만** 성립한다.
- 납품·파기도 동일 문법이다: 장비 탭 → 납품 카운터 탭(자동 최고점 매칭, mechanics §9) / 아이템 탭 → 쓰레기통 탭.

### 3.2 착수 탭과 거절 피드백 (유휴 일꾼 0)

- 중립 상태에서 재료가 놓인 시설을 탭 → 벤치의 일꾼이 달려가 착수. 착수는 확정(취소 불가, mechanics §4)이므로 **착수 직전 확인 팝업은 두지 않는다** — 판단의 무게는 UI가 아니라 규칙이 싣는다.
- **거절 피드백(팝업 없음, 전부 인라인 0.6초)**: ① 시설이 좌우로 흔들림 ② `👷 0/4` 배지가 적색 굵은 테두리로 펄스 ③ 시설 위에 미니 라벨 "일손 없음!" ④ 둔탁한 '통-통'(하강 2음 — sound §4.2 "착수 거절") 사운드 + 짧은 햅틱. 게임은 멈추지 않고 다음 탭을 즉시 받는다.

### 3.3 수납과 파기

- **수납 = 원탭**: 시설 위 완성 산출물 탭 → 포물선을 그리며 트레이로 날아가고 시설이 빈다. 트레이 만석이면 수납 거절 — 트레이 전체가 한 번 번쩍 + 흔들림(착수 거절과 동일 문법).
- **쓰레기통 파기 = 즉시, 확인 팝업 없음 (확정).** 근거: ① mechanics.md가 "운반과 같은 탭-탭, 즉시"로 명세(§2 용어) ② 파기는 이미 2탭짜리 의도적 행위 ③ 라운드 중 모달 금지 원칙(§3.4). 오파기 방어는 확인창이 아니라 **배치**로 한다 — 쓰레기통은 엄지에서 가장 먼 우상단 모서리 고정, 납품 카운터(우하단)와 대각 반대. 되돌리기(undo)도 두지 않는다 — "파기는 선택"(mechanics §6.3)의 결기를 UI도 공유한다.

### 3.4 연속 조작의 리듬 원칙 (라운드 중 불변 규칙)

1. **라운드 중 모달 0개.** 확인창·전면 팝업·토스트로 화면을 막지 않는다. 모든 피드백은 대상 위 인라인.
2. **어떤 조작도 2탭을 넘지 않는다.** (운반·납품·파기 = 2탭, 착수·수납 = 1탭)
3. **연출은 입력을 막지 않는다.** 운반 포물선이 날아가는 중에도 다음 탭은 즉시 처리 — 입력 대기열이 아니라 즉시 반영.
4. **위치 불변.** 셀·트레이·버튼의 크기와 자리는 라운드 중 절대 움직이지 않는다.
5. 의뢰 카드 탭 = 카드 확대 + 필요 공정 체인 미리보기(1.5초 후 자동 축소, 시간은 계속 흐름). 선택 상태 중에는 무효 탭으로 취급한다.

## 4. HUD와 게이지

### 4.1 납품기한 (라운드 타이머)

- 최상단 좌측 고정, 고정폭 숫자 `m:ss`. 화면에서 시각 정보가 이곳에만 있는 **유일한 전역 시계**다(과열·인내는 각 대상 위에만).
- 잔여 30초: 숫자가 주황+확대, 테두리 느린 펄스. 잔여 10초: 초당 카운트다운 음 + 적색. 기한 도래 즉시 판정(progression §4) — 연장 수단 없음, 연장 UI 없음.

### 4.2 의뢰 카드와 강화 단계의 즉시 식별 (GDD §0.2 위임 — 확정)

- 카드 구성(위→아래): **장비 아이콘 / 요구 강화 핍 / 점수 / 인내 게이지**(§2.1 참조).
- **◆핍 뱃지 — 화면 전체 공통 (아트 §5와 동일 정의·상호 참조)**: 강화 단계 뱃지는 **자수정 퍼플 바탕 위에 ◆ 핍을 단계 수만큼(1~3개) 얹고**, +3만 금테를 두른다 — 핍 개수가 1차 신호(카운트 비교·색약 대응), 좁은 자리에서는 작은 픽셀 숫자 `+N`을 보조로 병기할 수 있다. 요구 강화 단계(의뢰 카드)와 장비 아이템의 현재 단계 모두 **같은 뱃지를 카드/아이템 우상단에 오버레이**한다. 카드의 핍과 손 안의 핍을 눈으로 맞대보는 것만으로 충족 여부가 판별된다 — 숫자 읽기 불필요. +0은 뱃지 없음. 뱃지의 조형·색은 아트 §5 소관.
- 인내 게이지는 카드 하단의 **길이 바**(색+길이 이중 부호화). 마지막 25%에서 카드 전체가 느리게 점멸. 철회 시 카드가 옆으로 미끄러져 나가고 새 카드가 들어온다 — 감점 연출 없음(철회는 기회 상실일 뿐, mechanics §9).
- 대형·예고 의뢰 카드는 왁스 인장 마크가 붙는다(브리핑 예고와 동일 마크 — §5).
- **의뢰 동시 칸의 UI 상한 = 4칸.** 칸 수 자체는 밸런스 문서가 이 안에서 정한다.

### 4.3 과열 경고 — 단계적 표현 (유일한 손실원)

"화면 어디를 보고 있어도 인지"를 위해 **셀 내 표시 + 전역 신호(소리·햅틱·가장자리 글로우)** 를 겹친다.

| 단계 | 셀 게이지 | 아이콘(형태) | 색 | 소리·햅틱 | 전역 신호 |
|---|---|---|---|---|---|
| 가열 중 | 링 채워짐 | 불꽃 | 파랑 | 장작 소리(약) | 없음 |
| 완료(회수 대기) | 가득+체크 | ✓ | **골드 옐로** (트윙클, 점멸 아님 — 아트 §2.2) | '딩' 벨 1회 | 없음 |
| 과열 유예 전반 | **링이 역방향으로 줄어듦** | 모래시계 | 주황 | 완료 벨 2연타 + 스팀 '칙' (sound §4.2 1단계) | 없음 |
| 유예 후반(위험) | 잔여 절반 미만 | **!** | 적색 | 빠른 벨 3연타 + 스팀 휘슬 상승 (sound §4.2 2단계) + 햅틱 2연타 | **해당 시설 방향의 화면 가장자리가 글로우 펄스** |
| 소실 | 빈 셀 | 연기 퍼프 | — | 둔탁음 1회 | 없음 — 짧고 가볍게, 분노가 아니라 학습(mechanics §6.3) |

- **청각 권위는 sound 문서**: 과열 경고의 소리는 sound §4.2(벨 연타 + 스팀 휘슬)를 그대로 따른다. "째깍"음은 **마감 카운트다운 전용**(§4.1 · sound §4.2)이므로 과열에는 쓰지 않는다.
- 색약 대응: 단계마다 **아이콘 모양이 다르고**(✓/모래시계/!), 링의 진행 방향(정→역)과 점멸 주기가 달라 색 없이 판별된다.

### 4.4 현재 총점과 커트라인 거리 — **표시한다 (확정)**

- HUD 중앙: 진행 바 + **★ 눈금 3개**(커트라인 위치) + `128점 · ★2까지 24` 텍스트. 실패선 미달 구간은 바가 빗금 패턴.
- 근거: 트리아지("무엇을 흘려보낼지")는 커트라인까지의 거리를 알아야 성립하고, 별이 게이트이자 수입(progression §2, §5)인 이상 목표 거리 은닉은 재도전 동기를 깎는다. 커트라인 수치 자체는 밸런스 문서 — UI는 눈금 위치만 받는다.

## 5. 브리핑 화면

목적: 시간이 멈춘 채(progression §3) **계획이 서게 하는** 화면. 정보 밀도는 "한 화면, 3구획, 스크롤 없음"으로 제한한다.

```
┌────────────────────────────────────┐
│  홍염 상단 · 3팀        ⏱ 기한 150초 │ ① 누가·언제
│  목표: 화산 동굴 🌋 (던전 일러스트)    │
├────────────────────────────────────┤
│  의뢰 경향   🗡▂ 🛡▆ 💍▂  "방어구 다수"│ ② 무엇이 올까
│  ┌줄무늬 인장┐ 예고: 종반 +2 지팡이 56점│   (대형 의뢰 예고
│  └────────┘  ← 왁스 인장 마크        │    = 카드와 동일 마크)
│  ⚡ 특수 규칙: (있을 때만 배너 표시)    │
├────────────────────────────────────┤
│  내 준비: 👷4 · 강화석 라인 가능 ✓      │ ③ 나는 준비됐나
│ ┌──────────────────────────────┐   │
│ │        ▶ 개시 (탭)            │   │ 하단 전폭 버튼
│ └──────────────────────────────┘   │
└────────────────────────────────────┘
```

- 의뢰 경향은 분류(무기·방어구·장신구)별 비중 막대 + 한 줄 문장. 대형 의뢰 예고는 **도착 시기(초반/중반/종반) + 장비 + 요구 핍 + 점수**를 카드 실물 미리보기로 보여준다 — 예고(브리핑)–비축(인벤토리)–결정론(강화)의 삼각(mechanics §7)을 UI가 완성한다.
- 특수 규칙은 있을 때만 배너로 — 없는 라운드에 빈 칸을 보여주지 않는다.
- 개시 전 무제한 체류 가능. 뒤로가기(입장 시트로) 허용.
- 일시정지 메뉴에 **"의뢰서 다시 보기"** 를 두어 브리핑 정보를 라운드 중에도 열람 가능하게 한다(열람은 조작이 아니다, mechanics §11).

## 6. 결과 화면

결과 화면 전체를 **원정 리포트 화면**이라 부른다(아트 §6.4와 동일). 그 안의 서술 텍스트는 **리포트 문장 카드**로 구분한다 — "화면"과 "그 안의 문장 카드"는 다른 층위다.

### 6.1 성공 연출 순서 — 원정 리포트 화면 (어느 단계든 탭 = 다음 단계로 스킵)

1. **장착 매핑 컷 → 원정 실루엣 파노라마**(약 3초 — 이 화면의 시그니처, 아트 §6.4와 길이·구성 일치): 이번에 납품된 장비 아이콘들이 팀원 스프라이트에게 하나씩 날아가 **장착되고**, 무장을 마친 팀이 던전 입구로 걸어 들어가는 실루엣 파노라마로 이어진다 — "내 손이 결정한 남의 운명"(GDD 원칙 10)의 화면. (장착음은 sound §4.2 "장착 매핑".)
2. **공략 배너**: "화산 동굴 — 공략 성공!"
3. **별 스탬프**: ★이 하나씩 쾅·쾅·쾅(햅틱 동기). 최고 기록 갱신 시 "기록 갱신!" 리본.
4. **점수 내역**: 총점, 납품 카드 축약 목록(장비·핍·점수), 철회된 의뢰 수(회색, 감점 아님을 명시하는 톤).
5. **정산 골드**: 별 등급 차등(progression §5) 카운트업.
6. **리포트 문장 카드**: 2~3문장의 서사 카드(예: *"3팀은 +2 지팡이의 화력으로 용암 골렘을 돌파했다."*). 장부에 자동 보관. 문체·일러스트는 아트 문서와 공동(아트 §6.4).
7. 버튼: `[다시]` `[가도로]` 동급 크기.

### 6.2 실패 시 — 짧고 가볍게, "다시"가 가장 크다 (progression §8)

```
│  후퇴 컷 (0.8초, 팀은 무사히 귀환)      │
│  "공략 실패 — 다치지 않았다"           │
│  총점 71 · 실패선까지 9점 모자람        │ ← 다음 시도의 힌트
│ ┌──────────────────────────────┐   │
│ │        ▶ 다시  (전폭·최대)      │   │
│ └──────────────────────────────┘   │
│           가도로 (소형 텍스트)          │
```

- 연출 총 길이 3초 이내. 골드·리포트 연출 생략(지급 여부는 밸런스 문서 — 지급 시 한 줄 표기만).
- **"다시" 탭 → 브리핑 생략 → 플레이 화면이 "개시 대기" 상태로 로드**된다: 판 전체가 보이고 시간은 멈춰 있으며, 화면 하단의 `▶ 탭하여 개시` 배너를 탭하면 시작. 재시작이 즉시이면서도 관찰·계획(mechanics §11)의 권리는 유지된다. 일시정지 메뉴의 "재시작"도 동일 경로다.

## 7. 가도 지도

### 7.1 레이아웃 — 세로 길, 상단 고정 바

```
┌────────────────────────────────────┐
│ 누적 ★ 34        다음 개방까지 ★ 6   │ ← 고정 바 (스크롤 불변)
├────────────────────────────────────┤
│      (4) ❓❓❓ 상단      🔒          │
│       │   요건: (3) 계약 + 누적 ★40  │
│      (3) 은빛 방패 상단   [공사 중]   │
│       │   ●●●○○ R3/5 · ★7        │ ← 라운드 점·별 합계
│      (2) 홍염 상단       [계약 ✅]   │
│       │   ★★★★★★★★ (8/12)        │
│      (1) 여명 조합    [계약 🏅 금테]  │
│           전 라운드 ★★★             │
├────────────────────────────────────┤
│  [가도]   [상점]   [장부]   [설정]    │ ← 하단 탭바
└────────────────────────────────────┘
```

- **상단 고정 바가 GDD §0.2 위임 제약의 답이다**: 누적 별 현황과 **다음 거래처 개방 요건까지의 잔여**가 스크롤 위치와 무관하게 항상 보인다. 바 탭 → "어디서 별을 더 벌까" 뷰(별 여지가 남은 라운드 목록 정렬) — 누적 별 게이트의 재도전 선택(progression §2)을 전략으로 만드는 보조 화면.
- 노드 상태 4종: **잠김**(🔒 + 요건 명시) / **공사 중**(진행 점 + 별 합계) / **계약**(도장) / **금테 승급**(🏅, progression §6). 미개방 거래처는 이름을 ❓로 가리되 요건은 보여준다 — 목표는 항상 보이게.

### 7.2 라운드 입장 시트 (progression §2.2 입장 요건)

노드 탭 → 라운드 목록 → 라운드 탭 → 하단 시트:

```
│  R4 · 용암 심부   기록 ★★☆ · 기한 180초 │
│  필요 설비                              │
│   ✓ 용광로      ✓ 합성대            │
│   ✗ 강화 시설 — 미보유               │ ← 부족 설비 적색
│  [상점에서 구매 →]   ← 부족 시에만 표시  │
│  ┌────────────────────────────┐     │
│  │  입장 (요건 미달 시 비활성+사유) │     │
└──└────────────────────────────┘─────┘
```

- 이행 불가 의뢰가 있는 라운드는 시작 불가(progression §2.2) — 입장 버튼 비활성 + 부족 설비를 체크리스트로 명시 + 상점 바로가기. "시설 구매가 라운드를 여는 열쇠"라는 경제 루프를 UI 동선으로 직결한다.

## 8. 상점·장부·설정

### 8.1 상점 — 구매 축별 카테고리 칩

- 상단: 보유 골드 고정 표시. 칩: `[일꾼] [시설] [슬롯·수납] [도구]`.
  - **일꾼**: 현재 n/6 명 표시, 다음 고용 가격. 상한 6 도달 시 "만원 — 대장간의 정원입니다"로 축하 톤 마감(mechanics §4 구조적 확정).
  - **시설**: 계열(변환·합성·제작·강화)별 그룹. 신규 종류 / 추가 설치(중복) / 등급업을 한 카드에서. 계열 상한(mechanics §5.2) 도달 계열은 "이 계열은 완성" 뱃지.
  - **슬롯·수납**: 설치 슬롯 확장(UI 상한 14), 인벤토리 확장(UI 상한 12).
  - **도구**: 패시브 아이템 목록.
- 각 카드: 아이콘 · 효과 한 줄 · 가격 · 구매 버튼. 구매 확인은 **버튼 2탭**(1탭 = 가격 확대, 2탭 = 확정)으로 팝업 없이.
- **시설 구매 직후 배치 모드로 자동 진입**한다.

### 8.2 배치 모드 (재배치 UX)

- 플레이 화면과 **동일한 격자·동일한 축척**에서 편집한다 — 배치가 곧 플레이 동선 설계이므로 다른 뷰를 쓰지 않는다.
- 조작은 운반과 같은 문법: **시설 탭 → 빈 슬롯 탭 = 이동, 시설 탭 → 다른 시설 탭 = 자리 교환.** 무료·무제한(progression §7). 고정 시설(공급함·쓰레기통·납품 카운터)은 잠금 표시로 이동 불가.

### 8.3 장부 (도감)

- 탭 2개: **거래처** / **원정 기록**.
  - 거래처: 8칸 카드 그리드 — 로고, 상태(공사 중/계약/금테), 별 합계. 카드 탭 → 상세: 회사 소개, 레이드 팀, 라운드별 최고 별, 계약 체결일.
  - 원정 기록: 결과 화면의 리포트(§6.1-6) 아카이브, 최신순. 수집욕의 화면 — 부가 경제 효과 없음(progression §6)을 UI도 존중해 숫자 보상을 붙이지 않는다.

### 8.4 설정 항목 (확정 목록)

효과음 / 음악 / 햅틱(강·약·끔) / **왼손 모드** / **색 보조 모드**(패턴·아이콘 강조) / **큰 글씨** / 언어 / 튜토리얼 다시 보기 / 데이터 백업·복원(완전 오프라인 전제) / 크레딧. — 광고·계정·알림 항목은 없다(유료 선불, GDD §1.1 표).

## 9. 온보딩 — 스테이지 1이 튜토리얼 (스테이지 설계서와 공동)

### 9.1 가이드 연출 방식 (확정)

- **스포트라이트 + 말풍선 + 유도 탭**: 화면을 어둡게 깔고 대상만 밝히는 스포트라이트, 12자 이내 말풍선, 대상 위 펄스 링. 유도 스텝 동안 시간 정지(브리핑과 같은 "멈춘 시간" 문법) — 첫 수행 1회만 강제하고 즉시 자유로 돌려보낸다.
- 전면 텍스트 페이지·긴 설명 금지. 가르침은 항상 "지금 이 탭 한 번"의 단위다.

### 9.2 조작 문법을 가르치는 순서 (라운드 배치는 스테이지 설계서 재량, 순서는 UI 제안)

| 순서 | 가르치는 것 | 방식 |
|---|---|---|
| 1 | 운반 (탭-탭) | 공급함 → 용광로. 유효 목적지 브리딩을 처음부터 노출 |
| 2 | 착수 (시설 탭) | 제작대에 재료 운반 후 탭 — 일꾼이 달려가는 인과를 보여줌 |
| 3 | 납품 (탭-탭) | 완성 장비 → 납품 카운터. 점수 상승 연출 |
| 4 | 수납 (원탭) | 시설이 막히는 상황을 먼저 겪게 한 뒤 해법으로 제시 |
| 5 | 과열 | 안전한 저가 재료로 1회 소실을 **의도적으로 경험**시킴 — 유일한 손실원을 몸으로 |
| 6 | 인내·철회 | 저가 의뢰 하나를 흘려보내게 두고 "버린 게 아니라 고른 것" 말풍선 |
| 7 | 인벤토리·쓰레기통 | 비축 후 꺼내 쓰기, 과잉분 파기 |
| 8 | 강화 (해금 시점) | 브리핑 예고 → 강화석 비축 → 강화 패스의 삼각을 한 라운드로 |
| 9 | 착수 거절 | 유휴 0 상황을 만들어 거절 피드백의 의미를 학습 |

## 10. 접근성

- **색약 대응 — 색+형태 이중 부호화 (기본 탑재, 옵션 아님)**: 과열은 아이콘 모양(✓/모래시계/!)과 게이지 진행 방향·점멸 주기로(§4.3), 인내는 바의 길이로, 강화는 ◆핍 개수(필요 시 숫자 병기, §4.2)로, 별·금테는 도형으로 — 색을 제거해도 게임 전체가 판별된다. "색 보조 모드"는 여기에 패턴(빗금·점)을 추가 강조하는 옵션이다.
- **폰트**: 본문 최소 14pt, 점수·타이머는 숫자 전용 고정폭. "큰 글씨" 옵션은 HUD·카드 텍스트를 1.25배(레이아웃 불변, 카드 내 재배치로 흡수).
- **햅틱**: 완료 벨(약 1회) / 과열 위험(강 2연타) / 착수 거절(짧은 버즈) / 별 스탬프(쿵) — 이벤트별 패턴을 구분해 화면을 안 보는 손에도 정보를 준다. 설정에서 강·약·끔.
- **왼손 모드**: 하단 밴드(납품 카운터↔트레이, 유휴 벤치)와 일시정지 버튼을 좌우 미러. 시설 격자는 미러하지 않는다 — 재배치가 그 역할을 한다.
- **일시정지 = 관찰의 UI (mechanics §11)**: 정지 시 화면을 가리는 어두운 전면 오버레이를 **금지**한다. 판은 채도만 살짝 낮춰 전부 보이고, 메뉴는 하단 시트(재개·의뢰서 다시 보기·재시작·나가기·설정)로 최소화. 정지 중 의뢰 카드 탭 = 상세 열람 허용(열람은 조작이 아니다), 지시 탭은 무시. 재시작·나가기만 확인 1회를 둔다(라운드 중 유일한 확인 — 모달 금지 원칙의 명시적 예외).
- 최소 탭 타깃 44pt, 인접 타깃 간격 8pt 이상. 앱 이탈 시 자동 정지·저장(mechanics §11)으로 어떤 중단도 손실이 되지 않는다.
