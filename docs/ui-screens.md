## 1. 목적과 소유 범위

이 문서는 `UX-MVP-001`의 시나리오를 거래처 1곳·5개 라운드 MVP에서 실제로 조작 가능한 화면으로 구현하는 방법을 정의한다. UX가 정한 행동과 결과를 바꾸지 않고, 화면 목록, 내비게이션, 레이아웃, 표시 정보, 컴포넌트, 화면 상태와 피드백을 소유한다.

이 문서가 Source of Truth인 항목은 다음과 같다.

- MVP에 존재하는 화면·시트·대화상자·배너의 전수
- 화면별 목적, 레이아웃, 표시 정보, 사용자 행동, 화면 전이
- 로딩·빈 상태·잠금·부족·오류·저장 중 상태의 화면 표현
- 플레이 화면의 고정 영역과 탭 상호작용 표현
- 화면이 기술 계층에 요구하는 데이터와 명령의 의미

이 문서는 다음을 소유하지 않는다.

- 사용자가 경험해야 할 결과와 복구 정책
- 게임 규칙, 콘텐츠 목록, 밸런스 값
- 엔진 씬 구조, API 이름, 저장·결제 구현
- 색·폰트·스프라이트·사운드 파일의 제작 규격
- 상업 정책과 결제·페이월 화면

## 2. 화면 설계 원칙

1. **세로 한 화면:** 플레이 중 스크롤·팬·줌은 없다. 모든 생산 대상과 핵심 상태가 동시에 보인다.
2. **엄지 한 개:** 빈번한 생산 행동은 화면 중앙과 하단에 두며 최소 탭 영역을 지킨다.
3. **위치 불변:** 라운드 중 시설·인벤토리·납품·의뢰 카드의 위치와 크기가 바뀌지 않는다.
4. **최대 두 탭:** 운반·납품·파기는 두 탭, 착수·수납은 한 탭이다.
5. **인라인 피드백:** 라운드 중 입력 오류는 전면 팝업으로 흐름을 막지 않는다.
6. **형태 우선:** 색·소리 없이도 과열, 완료, 일손 없음, 잠김, `+1`을 구분할 수 있다.
7. **위험 신호 보존:** 레드+점멸 조합은 과열 임박에만 사용한다. 골드 부족, 일손 없음, 의뢰 철회에는 사용하지 않는다.
8. **저장 상태 정직성:** 영속화가 끝나기 전에 결과·구매 완료를 확정된 것처럼 닫지 않는다.
9. **MVP만 표시:** 첫 거래처, 5개 라운드, 실제 구매 품목과 실제 설정만 표시한다. 향후 콘텐츠를 빈 탭이나 가짜 잠금으로 광고하지 않는다.

## 3. 표시 좌표와 반응형 계약

### 3.1 기준 좌표

- UI 기준 캔버스는 세로 `360 × 640` 논리 단위다. `TECH-MVP-001`의 기준 렌더 `1080 × 1920`과 정확히 3배로 대응한다.
- `390 × 844`처럼 더 길거나 넓은 기기는 반응형 검증 프레임으로 사용한다. 기준 영역을 비균일하게 늘리지 않고 위아래 안전 여백과 배경 영역을 확장한다.
- `TECH-MVP-001`의 실제 기기 렌더 해상도는 구현 입력이며, 화면 구성의 Source of Truth가 아니다.
- 모든 인터랙티브 요소는 OS 세이프 에어리어 안에 둔다. 배경과 비인터랙티브 장식만 바깥까지 확장한다.
- 최소 지원 폭에서는 좌우 여백과 간격을 먼저 줄이고, 탭 타깃이나 정보 글자를 기준 이하로 줄이지 않는다.
- 넓은 화면과 태블릿에서는 플레이 영역 폭을 무한히 늘리지 않고 중앙 고정 폭으로 유지한다.

### 3.2 접근성 기준

| 항목 | UI 계약 |
|---|---|
| 탭 타깃 | 최소 `44 × 44` 논리 단위, 인접 타깃 사이 최소 8 |
| 본문 | 기본 14 이상. 큰 글씨 모드에서 1.25배를 수용 |
| 숫자 | 타이머·점수·골드는 고정폭 숫자 사용 |
| 색 | 상태를 색 하나로만 구분하지 않음 |
| 모션 | 정보 전달 모션은 최종 정지 상태에도 같은 의미가 남음 |
| 소리·햅틱 | 핵심 상태는 항상 시각 신호를 함께 가짐 |
| 스크린 리더 | OS 스크린 리더의 포커스 탐색·제스처·자동 읽기는 MVP에서 제외. 색+형태, 큰 글씨, 시각·오디오·햅틱 대체까지만 채택 |

MVP 접근성 범위는 색+형태 이중 부호화, 큰 글씨, 음악·효과음 독립 조절, 햅틱 켬/끔, 모든 정보음의 시각 대체다. OS 스크린 리더, 외부 키보드·스위치 탐색, 음성 조작은 이번 커스텀 게임 캔버스의 검증 범위에서 제외하며 구현된 것처럼 접근성 설명에 표기하지 않는다.

## 4. MVP 화면 인벤토리

| 화면 ID | 이름 | 형식 | 목적 | 주요 UX 추적 |
|---|---|---|---|---|
| `UI-SCR-BOOT` | 부팅 | 전체 화면 상태 | 저장·설정 로드가 끝날 때까지 안전하게 대기 | `UX-BOOT-04~05` |
| `UI-SCR-TITLE` | 타이틀 | 전체 화면 | 새 게임·계속·이어하기의 단일 진입점과 최초 프로필 생성 재시도 | `UX-BOOT-01~03`, `UX-SAVE-09` |
| `UI-SCR-MAP` | 거래처 지도 | 전체 화면 | 첫 거래처의 5개 라운드·진행·골드 확인과 선택 | `UX-MAP-*`, `UX-RESULT-06` |
| `UI-SHEET-ROUND` | 라운드 입장 시트 | 지도 위 하단 시트 | 라운드 상태와 입장 가능 여부 확인 | `UX-MAP-01~05` |
| `UI-SCR-SHOP` | 상점 | 전체 화면 | MVP 설비 구매와 부족 상태 해소 | `UX-SHOP-*` |
| `UI-SCR-BRIEF` | 브리핑 | 전체 화면 | 시간이 멈춘 채 라운드 계획 후 개시 | `UX-BRF-*` |
| `UI-SCR-PLAY` | 플레이 | 전체 화면 | 생산·강화·납품과 상태 판단 | `UX-PLAY-*`, `UX-TUT-*` |
| `UI-SHEET-PAUSE` | 일시정지 | 플레이 위 하단 시트 | 관찰하며 재개·재시작·저장 후 나가기 | `UX-PAUSE-*`, `UX-INT-*` |
| `UI-SCR-RESULT` | 원정 결과 | 전체 화면 | 0~3성·보상·기록·계약과 다음 행동 표시 | `UX-RESULT-*`, `UX-SAVE-06` |
| `UI-SCR-SETTINGS` | 설정 | 전체 화면 또는 일시정지 위 전환 | 실제 MVP 옵션 조정 | `UX-SET-*` |
| `UI-SCR-LEGAL` | 정보·법적 고지 | 전체 화면 | 오프라인 정책·라이선스·지원 주소와 외부 앱 전환 | `UX-LEGAL-*` |
| `UI-SCR-DATA-ERROR` | 데이터 복구 | 전체 화면 | 프로필·스냅샷 오류를 비파괴적으로 처리 | `UX-SAVE-01~04` |
| `UI-DLG-CONFIRM` | 파괴 행동 확인 | 대화상자 | 재시작·포기·새 게임의 명시적 확인 | `UX-MAP-04`, `UX-PAUSE-03~05`, `UX-SAVE-02` |
| `UI-BANNER-SAVE` | 저장 상태 배너 | 공통 인라인 | 저장 중·실패·재시도 상태 표시 | `UX-SAVE-05~08` |

다음 화면은 MVP에 존재하지 않는다.

- 현금 결제, 페이월, 실물화폐 가격, 구매 복원, entitlement 오류 — 현재 Android/Google Play 무료 MVP에는 없음
- 두 번째 이후 거래처, 전체 가도, 장부·도감
- 시설 자유 배치 모드와 장식 편집
- 업적, 클라우드 세이브, 계정, 알림, 광고, 라이브 이벤트
- 출시판 전체 아이템·거래처를 미리 보여주는 카탈로그

후속 유료 스테이지가 승인되기 전에는 결제용 빈 화면·CTA·아이콘·잠금 상태를 만들지 않는다. 승인될 경우 `UX-MVP-001`을 먼저 개정하고, 그 결과를 입력으로 이 문서의 화면 인벤토리와 상태를 추가한다.

## 5. 화면 내비게이션

```text
UI-SCR-BOOT
  ├─ 정상 ─▶ UI-SCR-TITLE
  └─ 복구 필요 ─▶ UI-SCR-DATA-ERROR ─▶ UI-SCR-TITLE

UI-SCR-TITLE
  ├─ 새 게임/계속 ─▶ UI-SCR-MAP
  ├─ 이어하기 ─────▶ UI-SCR-PLAY + UI-SHEET-PAUSE
  └─ 설정 ─────────▶ UI-SCR-SETTINGS ─▶ UI-SCR-LEGAL

UI-SCR-MAP
  ├─ 라운드 탭 ────▶ UI-SHEET-ROUND ─▶ UI-SCR-BRIEF ─▶ UI-SCR-PLAY
  ├─ 상점 ─────────▶ UI-SCR-SHOP ─────▶ UI-SCR-MAP 또는 UI-SHEET-ROUND
  └─ 설정 ─────────▶ UI-SCR-SETTINGS ─▶ UI-SCR-LEGAL

UI-SCR-PLAY
  ├─ 일시정지 ─────▶ UI-SHEET-PAUSE
  └─ 기한 종료 ────▶ UI-SCR-RESULT

UI-SCR-RESULT
  ├─ 다시 ─────────▶ UI-SCR-BRIEF 또는 개시 대기 UI-SCR-PLAY
  ├─ 다음 라운드 ──▶ UI-SHEET-ROUND
  └─ 지도 ─────────▶ UI-SCR-MAP
```

OS 뒤로가기는 다음 우선순위로 처리한다.

1. 열린 확인 대화상자 닫기
2. 열린 하단 시트 닫기
3. 설정에서 호출한 화면으로 돌아가기
4. 정보·법적 고지에서 설정으로 돌아가기
5. 플레이 중에는 앱 종료 대신 일시정지 열기
6. 지도에서만 OS 기본 종료 흐름 허용

### 6.2 공통 로딩·오류 규칙

- 로딩이 300ms 미만이면 스피너를 깜빡이지 않는다.
- 화면 전체 로딩 중에는 주요 행동을 한 번만 받으며 중복 명령을 막는다.
- 국소 오류는 오류가 발생한 대상 가까이에 표시한다.
- 부팅 중 프로필·스냅샷의 무결성 또는 버전 문제로 사용자 선택이 필요하면 `UI-SCR-DATA-ERROR`로 승격한다. 실행 중의 국소 저장 실패는 발생 화면의 지속 오류 상태로 처리한다.
- 오류 문구는 `무엇이 실패했는가 → 무엇이 보존됐는가 → 지금 할 수 있는 행동` 순서다.
- 토스트만으로 데이터 유실 가능성을 알리지 않는다. 저장 경고는 사용자가 확인하거나 해결할 때까지 남는다.

## 7. 화면별 상세 명세

### 7.1 `UI-SCR-BOOT` 부팅

**레이아웃**

- 중앙: 게임 로고 또는 단색 심볼
- 하단 세이프 에어리어: 진행 표시와 현재 단계 한 줄
- 상호작용 버튼은 정상 로드 중 표시하지 않는다.

**상태**

| 상태 | 표시 | 전이 |
|---|---|---|
| 초기화 | 로고, 무음 진행 표시 | 프로필·설정 검사 |
| 지연 | `게임을 준비하는 중`과 진행 표시 | 성공 또는 재시도 가능한 오류 |
| 복구 가능 | 자동 복구 결과를 준비 | `UI-SCR-DATA-ERROR`의 복구 안내 상태 |
| 복구 불가 | 기술 오류 코드의 사용자용 요약 | `UI-SCR-DATA-ERROR` |

부팅 화면에는 광고, 뉴스, 상점 배너, 네트워크 로그인, 구매 상태 확인을 넣지 않는다.

### 7.2 `UI-SCR-TITLE` 타이틀

**레이아웃**

- 상단 40%: 로고와 첫 거래처를 암시하는 배경
- 기존 프로필이 있으면 행동 영역 바로 위에 현재 다음 라운드·최고 별 요약을 한 줄로 표시한다. 중단 스냅샷이 있으면 라운드 ID와 `일시정지된 판` 표식을 같은 자리에 우선 표시한다.
- 하단 행동 영역: 문맥에 따른 단일 주요 버튼
- 우상단: 설정 버튼
- 하단 작은 글자: 빌드 버전. 상업 문구는 없음

**주요 버튼 규칙**

| 프로필 상태 | 주요 버튼 | 보조 버튼 |
|---|---|---|
| 저장 없음 | `새로 시작` | 설정 |
| 정상 진행 | `계속` | 설정 |
| 중단 스냅샷 | `이어하기` | `지도에서 보기`, 설정 |
| MVP 완료 | `계속` | 설정 |
| 새 프로필 생성 실패 | `시작 다시 시도` | 오류 정보, 설정 |

`새 게임`으로 기존 프로필을 덮어쓰는 행동은 일반 타이틀에 노출하지 않는다. 프로필 복구 불가 흐름에서만 `새 게임 시작`을 제공하며 2단계 파괴 확인을 거친다.

**필요 데이터**

**표시 순서**

### 7.5 `UI-SCR-SHOP` 상점

**레이아웃**

- 상단: 뒤로가기, `상점`, 현재 골드
- 이 화면은 R4 최초 클리어 뒤에만 진입할 수 있다. 본문에는 MVP의 유일한 상품 `SHOP_ENHANCEMENT_KIT` 카드 하나만 표시하고, 카드에는 `FAC_SYNTH_BENCH`와 `FAC_ENHANCE_ANVIL`을 함께 얻는다는 사실을 보여준다.
- 설비 부족 경로로 진입했다면 해당 카드에 `R5에 필요` 표식을 붙인다.
- 하단: 필요할 때만 `라운드로 돌아가기`

**품목 카드 정보**

- 아이콘, 이름, 한 줄 기능, 해금 상태
- 가격, 구매 후 잔액
- `구매`, `골드 부족`, `보유 중` 중 정확히 하나의 행동 상태

구매 확인은 별도 전면 페이지가 아니라 카드 내부 확장 또는 작은 확인 시트로 처리한다. 성공 표시는 저장 완료 뒤에만 나타난다.

| 오류 | 표현 | 행동 |
|---|---|---|
| 골드 부족 | 가격 옆 부족분, 동전 아이콘, 앰버 테두리 | `라운드로` |
| 구매 저장 실패 | 카드 원상 복귀, 지속 오류 문구 | `다시 시도` |
| 설치 공간 문제 | MVP에서는 고정 슬롯 설계로 발생시키지 않음 | 해당 없음 |

`SHOP_ENHANCEMENT_KIT`은 R4 최초 클리어 뒤 해금된다. 구매 즉시 두 설비를 §7.7.1의 고정 슬롯에 함께 설치한다. 자유 재배치와 배치 모드는 비범위다.

### 7.7 `UI-SCR-PLAY` 플레이

#### 7.7.1 고정 레이아웃

```text
┌────────────────────────────────┐
│  기한       점수·별 눈금      ⏸ │ A. 전역 HUD
├────────────────────────────────┤
│       [의뢰 슬롯: BAL]            │ B. 의뢰 밴드
├────────────────────────────────┤
│    [시설]  [시설]  [시설]         │
│    [시설]  [시설]  [시설]         │ C. 고정 3×2 시설 영역
├────────────────────────────────┤
│ 유휴 일꾼 [인벤토리 슬롯: BAL] 납품│ D. 엄지 행동 영역
└────────────────────────────────┘
```

- A는 남은 기한, 현재 점수, 별 커트라인 진행, 일시정지만 표시한다.
- B는 `BAL-MVP-001` §7의 해당 라운드 `활성 의뢰 슬롯`만큼만 카드를 배치한다. 그 밖의 빈 카드나 미래 확장 자리는 만들지 않으며 값을 UI 상수로 복제하지 않는다.
- C는 `FAC_SUPPLY`, `FAC_FURNACE`, `FAC_WEAPON_BENCH`, `FAC_SYNTH_BENCH`, `FAC_ENHANCE_ANVIL`, `FAC_TRASH` 여섯 칸만 배치한다. 구매 전 설비 자리는 해당 설비의 바닥 설계도 실루엣으로 표시하되 탭해 구매시키지 않는다.
- `FAC_SUPPLY` 셀에는 현재 라운드에서 허용한 원자재만 `UI-CMP-SUPPLY-SLOT`으로 직접 표시한다. R1~R4에서는 철광석 하나를 중앙에, R5에서는 철광석·목재·마력 파편을 2×2 안의 세 자리로 배치한다. 각 슬롯의 탭 영역은 44×44 이상이며 빈 네 번째 자리는 표시하지 않는다.
- D는 `BAL-MVP-001` §4의 `일꾼 수`에서 계산한 `유휴/전체` 수, 인벤토리, 납품 카운터가 항상 보인다. 납품 카운터는 양손 어느 엄지에서도 닿는 하단 중앙·중앙측 영역에 둔다. 일꾼 고용·선택 UI는 없다.
- 인벤토리는 `BAL-MVP-001` §4의 `인벤토리 용량`만큼 슬롯을 표시한다. 존재하지 않는 확장 칸이나 자물쇠 칸을 추가하거나 값을 UI 상수로 복제하지 않는다.
- 쓰레기통은 납품과 혼동되지 않도록 시설 영역 상단 모서리에 고정한다.

시설 영역은 콘텐츠 카탈로그의 여섯 시설을 위한 `3×2` 고정 좌표를 소유한다. 인벤토리와 의뢰 카드 수는 `BAL-MVP-001`의 현재 파라미터만 렌더하며 빈 칸으로 미래 용량을 예고하지 않는다.

시설 좌표의 Source of Truth는 다음과 같다. 모든 라운드에서 좌표를 바꾸지 않는다.

| 좌표 | 시설 | 구매 전 표현 |
|---|---|---|
| 1행 1열 | `FAC_SUPPLY` | 설치됨 |
| 1행 2열 | `FAC_FURNACE` | 설치됨 |
| 1행 3열 | `FAC_TRASH` | 설치됨 |
| 2행 1열 | `FAC_WEAPON_BENCH` | 설치됨 |
| 2행 2열 | `FAC_SYNTH_BENCH` | 바닥 설계도 실루엣 |
| 2행 3열 | `FAC_ENHANCE_ANVIL` | 바닥 설계도 실루엣 |

#### 7.7.2 탭 상태 머신

```text
[중립]
  ├─ 유효 출발지 탭 ─▶ [아이템 선택]
  ├─ 준비된 작업자 시설 탭 ─▶ 착수 또는 일손 없음 피드백
  └─ 완료 셀의 수납 행동 탭 ─▶ 수납 또는 인벤토리 만석 피드백

[아이템 선택]
  ├─ 유효 목적지 탭 ─▶ 운반 실행 ─▶ [중립]
  ├─ 다른 출발지 탭 ─▶ 선택 교체
  ├─ 같은 출발지/빈 바닥 탭 ─▶ 선택 취소
  └─ 무효 목적지 탭 ─▶ 선택 유지 + 국소 거절
```

선택 중에는 출발지에 백색 선택 링을, 유효 목적지에는 형태가 있는 호흡 테두리를 표시한다. 무효 목적지를 전체 딤 처리해 판독성을 떨어뜨리지 않는다.

공급함은 별도 열기 탭이나 팝업을 사용하지 않는다. 보이는 원자재 슬롯 자체가 출발지이므로 `원자재 슬롯 → 목적지` 두 탭으로 끝난다. 현재 라운드에서 쓰지 않는 원자재는 잠금 슬롯으로 예약하지 않고 숨긴다.

완료 시설은 산출물 아이콘과 44×44 이상의 `수납` 행동을 분리한다. 산출물 아이콘 탭은 다른 아이템과 같이 선택 상태로 들어가 다음 시설·납품·파기 목적지로 직접 옮길 수 있고, `수납` 탭은 첫 빈 인벤토리 칸으로 한 번에 보낸다.

용광로는 완전한 입력이 도착한 즉시 자동 가동하므로 별도 착수 탭 상태를 만들지 않는다. 제작대·합성대·강화 모루만 입력 완료 뒤 착수 탭을 받는다.

#### 7.7.3 시설 상태

| 상태 | 셀 표현 | 전역 피드백 |
|---|---|---|
| 비어 있음 | 기본 실루엣과 레시피 입력 소켓 | 없음 |
| 부분 입력 | 채워진 입력 아이콘, 비어 있는 필수 소켓과 누락 재료 형태 | 착수 불가. 넣은 입력은 다시 선택·회수 가능 |
| 입력 완료 | 모든 입력 아이콘, `시작 가능` 형태 | 작업자 시설만 착수 탭 가능 |
| 작업 중 | 정방향 진행 링, 작업 일꾼 | 없음 |
| 산출물 대기 | 산출물 아이콘, 1탭 수납 행동, 체크, 골드 트윙클 | 완료 시 1회 알림 |
| 과열 유예 | 역방향 링, 모래시계 | 단계별 정보음·형태 |
| 과열 임박 | `!`, 빠른 역방향 링, 레드 점멸 | 화면 가장자리 방향 신호 |
| 과열 소실 | 짧은 연기 뒤 빈 시설 | 소실 1회 피드백 |

레드+점멸은 `과열 임박` 한 상태에만 허용한다. `일손 없음`은 앰버 윤곽, 빈 벤치, 흔들림으로 표현한다.

#### 7.7.4 의뢰 카드

카드는 위에서부터 다음 정보를 고정 순서로 표시한다.

1. 장비 아이콘과 화면에 보이는 이름 라벨
2. 요구 강화 단계 — `+0`은 무표식, `+1`은 보라 ◆ 한 개와 `+1` 보조 글자
3. 획득 점수
4. 인내 게이지 — 길이와 단계 아이콘을 함께 사용
5. 예고 의뢰 표식 — R4 `REQ_SWORD_FORECAST`와 R5 `REQ_SWORD_P1_FORECAST`에 사용

철회 때 카드가 옆으로 빠지고 빈 칸이 채워진다. `-점수`나 레드 경고를 표시하지 않는다.

#### 7.7.5 튜토리얼 오버레이

- 대상 하나만 밝히고 나머지는 채도를 낮춘다.
- 말풍선은 행동 한 문장과 동사 하나로 제한한다.
- 가이드 중 시간은 정지하고 목표 탭 외 입력을 받지 않는다.
- 첫 성공 직후 오버레이를 제거한다.
- R1~R5의 구체적 교육 순서는 `CNT-MVP-ROUND-001`과 `UX-MVP-001`을 따른다.

### 7.8 `UI-SHEET-PAUSE` 일시정지

- 플레이 화면을 가리는 불투명 전체 오버레이를 사용하지 않는다.
- 배경은 채도만 낮추고 시설·의뢰·인벤토리 상태를 읽을 수 있게 유지한다.
- 하단 시트 행동 순서는 `재개`, `의뢰서 보기`, `재시작`, `저장 후 지도 나가기`, `설정`이다.
- 재시작은 `UI-DLG-CONFIRM`을 거친다.
- 저장 실패 중에는 지도 나가기 버튼을 `저장 다시 시도`로 바꾸고 위험을 숨기지 않는다.
- 설정에서 돌아오면 자동 재개하지 않는다.

### 7.9 `UI-SCR-RESULT` 원정 결과

**단계**

1. 납품 장비가 첫 거래처 팀에게 전달되는 짧은 인과 컷
2. 공략 실패 또는 성공 배너
3. 별 0~3개
4. 이번 점수, 커트라인, 납품 요약과 철회 건수
5. 획득 골드와 최고 기록 갱신 여부
6. 행동 버튼

각 단계는 탭으로 다음으로 넘길 수 있다. 모든 연출을 넘겨도 수치 요약과 행동은 남는다.

| 결과 상태 | 주요 버튼 | 보조 버튼 |
|---|---|---|
| 0성 | `다시` | 지도 |
| R1~R4 성공 | `다음 라운드` | 다시, 지도 |
| R5 성공 | `지도에서 계약 보기` | 다시 |
| 저장 중 | 버튼 잠금, `저장 중` | 없음 |
| 저장 실패 | `저장 다시 시도` | 오류 정보 |

R5 성공 시 같은 화면 안에서 계약 도장 단계를 추가한다. 별도의 장부 화면은 만들지 않는다. 계약 후 지도 헤더가 완료 상태로 바뀐다.

철회가 한 건 이상이면 납품 요약 아래에 회색 중립 행으로 `철회 N건 · 감점 없음`을 표시한다. 실패 경고나 음수 점수 애니메이션을 사용하지 않으며, R3에서는 첫 철회 결과에 짧은 설명 한 줄을 붙인다.

### 7.10 `UI-SCR-SETTINGS` 설정

MVP에 실제 구현할 항목만 표시한다.

| 항목 | 제어 | 상태 |
|---|---|---|
| 음악 | 0~100 슬라이더와 음소거 | 즉시 적용 |
| 효과음 | 0~100 슬라이더와 음소거 | 즉시 샘플 재생 |
| 햅틱 | 켬/끔 | 켤 때 약한 확인 샘플 1회. 미지원 기기는 `지원하지 않음` |
| 색 보조 | 켬/끔 | 패턴과 아이콘 강조 미리보기 |
| 큰 글씨 | 켬/끔 | 샘플 텍스트 즉시 반영 |
| 정보·법적 고지 | 화면 이동 행 | 설치 버전, 개인정보·오픈소스 원문과 지원 주소를 오프라인으로 열기 |

한국어 단일 언어 MVP에는 언어 선택을 표시하지 않는다. 모노 오디오, 무음 부스트, 튜토리얼 다시 보기, 데이터 백업, 클라우드, 계정도 표시하지 않는다.

설정 화면은 지도와 일시정지에서 열 수 있다. 호출 위치를 기억하고 같은 곳으로 돌아간다.

### 7.11 `UI-SCR-LEGAL` 정보·법적 고지

이 화면은 `REL-PLAY-001`의 canonical Markdown을 앱 리소스에서 읽는다. WebView나 원격 요청을 사용하지 않는다.

**레이아웃 순서**

1. 상단 바: `정보·법적 고지`, 뒤로가기
2. 앱 정보: `던전 오피스`, 설치 버전
3. 지원 카드: `2020promoking@gmail.com`, `복사`, `메일 앱 열기`
4. 문서 선택: `개인정보처리방침`, `오픈소스 라이선스`
5. 세로 스크롤 원문 영역
6. 공개 지원·개인정보 URL 텍스트와 각각의 `복사`, 선택적 `브라우저 열기`

원문 스크롤은 게임 플레이의 무스크롤 원칙 대상이 아니다. 본문은 큰 글씨를 적용하고, 링크 의미를 색 하나로만 표현하지 않으며, 긴 URL과 라이선스 텍스트가 화면 밖으로 가로 넘침 없이 줄바꿈되어야 한다.

| 상태 | 표현 | 행동 |
|---|---|---|
| 정상 | 번들 Markdown 렌더, 문서 시행일·마지막 갱신일 표시 | 문서 전환, 주소 복사, 외부 앱 열기 |
| 기내 모드·오프라인 | 정상과 동일. 네트워크 배너 없음 | 모든 로컬 열람·복사 가능 |
| 메일 앱 없음·실행 실패 | 지원 카드 아래 `메일 앱을 열 수 없습니다` | 이메일 복사를 1차 행동으로 유지 |
| 브라우저 없음·실행 실패 | URL 아래 비차단 오류 | URL 복사와 로컬 원문 유지 |
| Markdown 렌더 실패 | 같은 파일의 일반 텍스트 표시 | 열람·복사 유지 |
| 원문 파일 누락 | `고지 파일을 불러올 수 없습니다`, 설치 버전, 지원 이메일과 복사 행동 | 오류 정보 복사. 공개 후보에서는 출시 차단 |

외부 앱을 열기 전 플레이가 진행 중이었다면 `UI-SHEET-PAUSE` 상태를 유지한다. 앱 복귀 시 스크롤 위치와 선택한 문서를 복원하며 라운드를 자동 재개하지 않는다. 외부 앱 호출에 본문, 저장 파일, 진단 로그나 기기 정보를 자동 첨부하지 않는다.

### 7.12 `UI-SCR-DATA-ERROR` 데이터 복구

**레이아웃 순서**

1. 문제 유형 아이콘과 제목
2. 문제가 있는 범위: 프로필 또는 진행 중 라운드
3. 보존된 범위
4. 주요 행동 `다시 시도` 또는 `복구본으로 계속`
5. 보조 행동 `오류 정보 보기`
6. 최후 행동 `새 게임 시작` — 프로필 복구 불가에서만 표시

스냅샷 손상과 버전 불일치에서는 `현재 라운드 다시 시작`을 제공하고 새 게임 버튼을 표시하지 않는다. 새 게임 확인은 프로필 대체 사실을 명시하는 2단계 확인이다.

기술 오류 코드는 기본 화면에 그대로 노출하지 않고 `오류 정보 보기` 안에서 복사 가능하게 한다.

### 7.13 `UI-DLG-CONFIRM` 파괴 행동 확인

재시작, 중단 라운드 포기, 복구 불가 프로필의 새 게임 시작에만 사용한다. 제목은 실행할 행동, 본문은 사라지는 상태와 보존되는 상태, 버튼은 `취소`와 구체적인 파괴 동사 순으로 표시한다.

| 행동 | 사라짐 | 보존됨 | 확인 단계 |
|---|---|---|---:|
| 현재 라운드 재시작 | 현재 판과 스냅샷 | 프로필, 골드, 최고 기록 | 1 |
| 중단 라운드 포기 | 해당 스냅샷 | 프로필, 골드, 최고 기록 | 1 |
| 새 게임 시작 | 복구 불가 기존 프로필 파일 | 오류 정보와 로컬 진단 로그 | 2 |

OS 뒤로가기와 대화상자 바깥 탭은 항상 취소로 처리한다. 기본 선택은 `취소`이며, 장비·재료 파기는 출발지와 `FAC_TRASH`를 잇는 두 번의 의도적 탭 자체가 확인이므로 이 대화상자를 띄우지 않는다.

## 11. 아트·오디오 인계

`ART-MVP-001`은 다음 화면 상태를 식별할 에셋을 제공해야 한다.

- 라운드 노드 5상태, 설비 체크 2상태, 상점 카드 3상태
- 시설의 비어 있음·작업·완료·과열 유예·과열 임박·소실
- 아이템 분류와 `+1`, 의뢰 예고, 별 0~3
- 첫 거래처 팀의 실패·성공·계약 상태
- 선택·유효 목적지·입력 거절·저장 경고
- 정보·법적 고지는 공통 문서·복사·외부 열기 아이콘만 재사용하며 원격 웹 화면이나 전용 일러스트를 요구하지 않는다.

`AUD-MVP-001`은 다음 우선 이벤트에 피드백을 제공해야 한다.

- 작업 완료, 과열 경고·임박·소실, 마감 임박
- 착수·운반·수납·납품·강화 완료
- 일손 없음, 인벤토리 만석, 무효 행동
- 별 판정, 계약 체결, 상점 구매

오디오와 햅틱이 꺼져도 이 문서의 시각 상태만으로 판별 가능해야 한다.

## 12. 완료 조건

이 문서는 다음을 모두 만족할 때 `Review`로 전환한다.

- `UX-MVP-001`의 모든 시나리오가 화면·시트·대화상자·비시각 응답 중 하나에 매핑된다.
- 현재 Android/Google Play 무료 MVP의 화면 인벤토리에 현금 결제·페이월·실물화폐 가격·구매 복원 화면이 없다.
- `REL-PLAY-001`의 개인정보·오픈소스·지원 정보가 오프라인 화면, 주소 복사와 외부 앱 실패 상태로 구현된다.
- 첫 거래처와 5개 라운드 밖의 화면·탭·가짜 콘텐츠가 없다.
- 플레이 화면이 MVP의 모든 실제 시설, 최대 의뢰, 인벤토리, 일꾼 상태를 무스크롤로 수용한다.
- 모든 화면에 정상·로딩·잠금·부족·저장 오류 중 해당하는 상태가 정의된다.
- 레드+점멸이 과열 임박에만 사용된다.
- 화면별 읽기 상태와 명령 의도가 `TECH-MVP-001`이 구현 가능한 수준으로 나열된다.
- `ART-MVP-001`, `AUD-MVP-001`의 에셋 요구와 `QA-MVP-001`의 검증 지점이 추적 가능하다.
