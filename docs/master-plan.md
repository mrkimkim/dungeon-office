# 던전 오피스 — 프로젝트 마스터 플랜 v0.1

> 성격: 이 문서는 **전 영역 통합 실행 계획**이다. 기획 문서 10종(GDD + 하위 9종)이 정의한 *무엇을 만드는가*를, *언제·어떤 순서로·누가·얼마에* 만드는가로 번역한다. 세부 사양은 각 하위 문서가 소유하고, 본 문서는 그것들을 하나의 일정·의존성·게이트로 엮는 상위 조율 문서다. 충돌 시 사양은 하위 문서가, 일정·우선순위는 본 문서가 이긴다.
>

## 0. 전제와 상태

### 0.1 확정된 것 (프리프로덕션 완료)
- 게임명: **던전 오피스 (Dungeon Office)**
- 배포: 무료 다운로드 + 거래처 1 체험 + 본편 언락 IAP($4.99 / ₩6,900), 광고 없음, 완전 오프라인
- 엔진: Godot 4.4 + 타입드 GDScript, 20Hz 고정 틱 결정론
- 콘텐츠 규모: 거래처 8 · 라운드 41 · 아이템 41 · 일꾼 2→6 · 시설 9종
- 기획 문서 10종 작성·커밋 완료

### 0.2 계획의 핵심 전제 (사장 확인 요망 — 일정의 축)
1. **1인 풀타임 개발 + 아트·사운드 외주 병행**을 기준으로 잡는다. 플랫폼 문서의 "약 9~10개월"이 이 전제의 산물이다.
   - **파트타임(저녁·주말)이면 달력을 ×2~2.5 하고 출시 윈도를 2027 하반기~2028로 이동**한다. 단계 순서·의존성·게이트는 그대로다.
2. **착수 시점 = 2026년 7월(지금).** 이 가정 하에서 개발 종료가 2027년 4~5월로, 마케팅 문서가 권고한 출시 윈도(2027 3~4월)·한국 인디 지원 캘린더(창구 9기 2월, BIC 2027 봄)와 자연 정렬된다. 이 정렬이 지금 착수의 가장 큰 이점이다.
3. **예산**: 수익 문서(개발 외 현금 지출)와 마케팅 문서(0/300만/1000만 시나리오)의 중간선 — 아트·사운드 외주에 집중 투자, 유료 UA는 최소. §7 롤업 참조.

### 0.3 이 계획이 답하는 질문
- 무엇을 먼저 만드는가(임계 경로), 무엇을 병행하는가(워크스트림), 언제 멈추고 판단하는가(게이트), 무엇이 아직 문서로 없는가(§8).

## 2. 워크스트림 — 누가 무엇을 언제

7개 워크스트림이 위상도 위에서 병행한다. "본인"은 사장(개발자), 나머지는 외주/구매.

| 스트림 | 담당 | M0 | M1 | M2 | M3 |
|--------|------|----|----|----|----|
| **A. 시뮬 코어·시스템** | 본인 | 결정론 코어·틱·세이브·5라운드 | 전체 루프(상점·지도·결과)·IAP 언락 | 기능 동결·버그 | 안정화만 |
| **B. 콘텐츠 데이터(JSON)** | 본인 | 임시 5라운드 | 거래처 1 정식 데이터 | **8사·41라운드 전량 author** | 밸런스 2차 반영 |
| **C. 밸런스 튜닝** | 본인+봇 | C-3/C-4 관측 | 거래처 1 실측 튜닝 | 1차 패스(전 라운드) | 2차 패스(베타 피드백) |
| **D. 아트** | 외주+본인 | 그레이박스(도형) | **핵심 에셋 발주·수령**(일꾼·시설·UI) | 나머지 에셋(거래처 8·연출) | 폴ish·아이콘·스토어 아트 |
| **E. 사운드** | 외주+라이브러리 | 무음/임시음 | **메인 테마+핵심 SFX 15종 발주** | 나머지 곡·징글·SFX | 믹싱·검수 |
| **F. QA·현지화** | 본인+테스터 | 골든 리플레이 인프라 | 테스터 5인 루프 | CI 회귀 상시화 | 디바이스 매트릭스·한/영 로컬라이즈·베타 |
| **G. 비즈니스·마케팅** | 본인 | (없음) | 스토어 계정·창구9기/BIC 접수 준비 | 스토어 페이지·소재 은행 촬영 | **D-90 빌드업 실행**·심사 제출 |

### 병행의 원칙
- **아트·사운드는 M1 착수와 동시에 발주한다**(외주 리드타임 때문). M0에서 도형·무음으로 재미를 먼저 검증하고, 재미 게이트 통과 후에야 에셋에 현금을 태운다 — M0 이전 발주 금지.
- **콘텐츠 데이터(B)와 밸런스(C)는 M2의 주작업**이다. M2에서 개발자의 시간 대부분이 여기 간다(코드는 M1에서 거의 완성).
- **마케팅(G)은 M3에 집중**되지만, 스토어 계정 개설·인디 지원 접수는 리드타임 때문에 M1~M2에 미리 시작한다.

## 3. 임계 경로와 의존성

**임계 경로**(이게 늦으면 출시가 늦는다): `시뮬 코어(M0) → 전체 루프+IAP(M1) → 41라운드 데이터 author(M2) → 밸런스 2차+심사(M3)`.

핵심 의존성 (선행 → 후행):
- 재미 게이트 통과 → 아트·사운드 현금 집행 (M0 → M1)
- 시뮬 이벤트 API 확정 → 아트 애니메이션·사운드 바인딩 (M1 초, 뷰 연동 전 고정)
- 아이템·스테이지·밸런스 JSON 스키마 확정 → 41라운드 author (M1 → M2)
- 밸런스 봇(`max_score`/`bot_player`) 가동 → 커트라인 검증 (M0 인프라 → M2 상시)
- M1 슬라이스 빌드 → 창구9기/BIC 접수 데모·마케팅 소재 (M1 → M2 마케팅)
- 리뷰 빌드(M3 초) → 인플루언서 키·베타 (D-30 마케팅)
- 한/영 현지화 완료 → 애플 피처링 가점 + 심사 (M3)

**병목 주의**: 개발자 1인이 A·B·C·F·G를 동시에 지는 M2~M3이 가장 위험하다. 완화 — M2에서 콘텐츠 author에만 집중하도록 코드(A)를 M1에 최대한 끝내고, 마케팅(G)의 반복 작업(숏폼)은 소재 은행을 M2에 미리 촬영해 M3엔 재편집만 한다.

## 4. 마일스톤별 상세 백로그

### M0 — 프로토타입 (6주) · 산출물 = 판단
플랫폼 §8 기준. **목표는 완성이 아니라 답이다.**
- 시뮬 코어: `step()` 순수함수, 20Hz 틱, PCG32, 의뢰 생성기, 과열·인내·납품 매칭
- 최소 뷰: 도형 스프라이트, 탭 조작(운반·착수·수납·납품), 유휴 일꾼 표시
- 임시 데이터 5라운드(일꾼 2명 구간 포함), 일시정지·중단 저장
- 골든 리플레이 인프라 가동
- **측정**: 부록 C-1~C-5, 특히 **C-3(콤보 없는 병렬 운영 자생력)·C-4(2일꾼 답답함)**. 본인+지인 3~5명 플레이 관측.

### M1 — 버티컬 슬라이스 (10주) · 거래처 1곳 출시 품질
- 전체 씬 루프: 타이틀→가도 지도→상점/장부→라운드(브리핑→플레이→결과)→복귀
- IAP 본편 언락(비소모성 + 구매 복원 + 오프라인 영수증 로컬 저장), 페이월 = "계약 체결식" 연출
- 거래처 1(새싹 원정대, 라운드 5개) 정식 데이터·튜닝
- **아트 발주**: 일꾼 공용 바디+머리 6종, 시설 4계열 실루엣, 핵심 UI, 과열·납품·판정 연출
- **사운드 발주**: 메인 가도 테마 1곡 + 플레이 1곡(4스템) + 핵심 SFX 15종
- 세이브 전체 왕복, 온보딩(스테이지 1 튜토리얼)
- **게이트**: 외부 테스터 5인+ 설명 없이 완주, 최소 사양 실기 60fps, 라운드 2~4분 실측

### M2 — 알파 (14주) · 콘텐츠 전량 + 기능 동결
- **41라운드 데이터 author**(거래처 2~8), 입장 요건 자동 도출 검증 통과
- 밸런스 1차 패스: `bot_player` 시드 100개 회귀, 커트라인·별 게이트·강화 할증 검증식(플랫폼 §7.5) 전 라운드 통과
- 나머지 아트(거래처 8사 로고·팀·던전 배경, 시설 등급업, 전 아이콘) + 나머지 사운드(플레이 3곡·징글 8·SFX 잔량)
- 특수 규칙 6유형, 부품·조립(다인 작업) 라운드, 단골 특별 의뢰
- 기능 동결 — 이후 신규 시스템 금지, 데이터·버그·밸런스만
- **게이트**: 처음부터 끝까지 완주 가능, CI 전 검증 통과, 크래시 없는 1시간 세션

### M3 — 베타·출시 준비 (8주)
- 밸런스 2차 패스(외부 베타 피드백 반영), 리뷰 빌드 D-30까지 완성
- 디바이스 매트릭스 QA(Android 8+ 저가~플래그십, iOS 15+), 세이브 마이그레이션 리허설
- **한/영 현지화 완성**(번역 CSV), 애플 피처링 노미네이션 갱신 제출
- 스토어 심사 제출·대응(등급분류 GRAC/IARC, 개인정보처리방침, 오픈소스 고지)
- **게이트**: 크래시프리 스모크, 양 스토어 승인

## 5. 결정 게이트 — 멈추고 판단하는 지점

### 5.1 M0 재미 게이트 (프로젝트에서 가장 중요한 순간)
- **질문**: 콤보를 없앤 지금, 일꾼 2명 구간의 스케줄링이 그 자체로 재미있는가(C-3/C-4)?
- **통과**: M1로 진행, 아트·사운드 현금 집행 개시.
- **불통과**: 아트에 돈 쓰기 전에 GDD를 개정한다. 후보 — 콤보 재도입(부록 B), 시작 일꾼 3명, 라운드 단축, 인벤토리 압박 상향. **여기서 멈추는 것이 계획의 목적이다** — 재미없는 게임을 예쁘게 만드는 낭비를 M0가 막는다.

### 5.2 M1 슬라이스 게이트
- 무설명 완주 실패 → 온보딩(UI·스테이지1) 재설계 후 재테스트. 콘텐츠 확장(M2) 진입 금지.

### 5.3 M2 콘텐츠 게이트
- 밸런스 봇이 "권장 설비로 ★1 불가" 또는 "3성 수학적 불가" 라운드를 찾으면 → 그 라운드 데이터 수정. 기능 추가로 해결 금지(동결 위반).

### 5.4 출시 Go/No-Go
- 크래시프리 미달 또는 스토어 미승인 → 출시 연기. 마케팅 D-데이는 빌드 확정 후에만 확정.

## 6. 아직 문서로 없는 것 — 언제 만드는가

지난 정리에서 짚은 미작성 문서들의 착수 시점을 계획에 편입한다.

| 문서 | 착수 | 이유 |
|------|------|------|
| **QA·플레이테스트 계획** | M0 직전 | 부록 C 7개 질문이 그대로 테스트 항목. M0의 관측 프로토콜부터 필요 |
| **로컬라이제이션 계획** | M1 초 | 문자열을 처음부터 키로 분리(플랫폼 §4.1). 텍스트 확정 전 규칙만 잡으면 됨 |
| **출시 체크리스트** | M2 중 | 등급분류·개인정보처리방침·스토어 자산·심사 요건. M3 실행의 준비물 |
| **제작 일정 상세(주 단위)** | 각 마일스톤 시작 시 | 본 문서의 마일스톤을 주 단위 태스크로 분해. 진행하며 갱신 |
| **라이브 운영 로드맵** | M3 | 콘텐츠 업데이트를 마케팅 비트로(마케팅 §8). 출시 전 골격만 |
