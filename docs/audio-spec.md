## 1. 목적과 소유 범위

이 문서는 거래처 1곳·5개 라운드 MVP에서 실제로 재생하는 음악, SFX, 믹싱 규칙과 햅틱 패턴의 전수를 정의한다. 오디오와 햅틱은 장식이 아니라 화면을 보지 않는 순간에도 작업 완료, 과열, 마감, 거절을 구분하게 하는 정보 채널이다.

이 문서가 Source of Truth인 항목은 다음과 같다.

- MVP 음악·징글·스팅어의 실제 목록과 상태별 재생 의도
- 게임·UI 이벤트와 SFX의 전수 매핑
- 정보음 우선순위, 동시 재생, 덕킹, 반복 피로 방지 규칙
- 햅틱 패턴과 무음·미지원 기기 대체 조건
- 오디오 소스의 납품 포맷, 파일명, 용량·라우드니스 예산

이 문서는 다음을 소유하지 않는다.

- 게임 이벤트의 발생 조건과 타이밍 값
- 화면의 시각 피드백과 설정 레이아웃
- 엔진 오디오 노드·버스 코드와 플랫폼 API 구현
- 외주 비용, MVP 카탈로그 밖 거래처 음악, 마케팅 사운드
- 현재 무료 MVP의 결제·페이월·구매 복원 소리

## 2. 오디오 방향과 불변 원칙

MVP의 음악 방향은 **동화 판타지 워크숍 팝**이다. 어쿠스틱 포크 기반에 마림바, 피치카토 현, 통기타, 작은 아코디언과 밝은 모루 퍼커션을 사용한다. 다크 판타지, 공포 드론, 순수 칩튠, 웅장한 전투 오케스트라는 사용하지 않는다.

1. **정보음은 고정한다.** 작업 완료, 과열, 마감처럼 학습해야 하는 소리는 음색과 음정의 의미를 바꾸지 않는다.
2. **확인음은 제한적으로 변주한다.** 탭·운반·망치질처럼 반복되는 소리는 라운드로빈으로 피로를 줄인다.
3. **실패를 처벌하지 않는다.** 거친 버저, 비명, 장송곡, 아이템 파괴음은 없다.
4. **강화는 확정의 소리다.** 룰렛·드럼롤·심박 상승·성공 여부를 기다리는 공백을 사용하지 않는다.
5. **긴장은 마감에서 온다.** MVP는 적응형 음악 레이어를 만들지 않고, 고정 BGM 위의 카운트다운 정보음과 시각 피드백으로 마감을 알린다. 과열은 별도 정보음으로 구분한다.
6. **시뮬레이션과 분리한다.** 오디오 라운드로빈과 피치 변주는 표현 계층에만 있고 게임 결과·리플레이 상태를 바꾸지 않는다.
7. **무음도 정식 지원한다.** 모든 핵심 정보음은 `UI-MVP-001`의 시각 신호가 있고, 가능한 기기에서는 구분 가능한 햅틱을 병행한다.

## 3. 우선순위와 오디오 버스 계약

### 3.1 우선순위

| 등급 | 의미 | 예 | 규칙 |
|---|---|---|---|
| `P0_INFO` | 놓치면 아이템·점수 기회를 잃는 정보 | 과열 경고, 작업 완료, 마감 카운트다운 | 항상 들림, 고정 음색, BGM·질감 덕킹 |
| `P1_ACTION` | 사용자 행동과 중요한 결과 확인 | 운반, 착수, 납품, 강화, 별 | 낮은 지연, 제한적 변주 |
| `P2_TEXTURE` | 공간과 작업의 질감 | 불, 망치, 합성 루프 | 언제든 줄이거나 생략 가능 |
| `P3_MUSIC` | 정서와 마감 밀도 | 메타·플레이 BGM | P0를 가리지 않음 |

### 3.2 논리 버스

```text
Master
  ├─ BGM
  ├─ SFX_INFO
  ├─ SFX_ACTION
  ├─ SFX_TEXTURE
  └─ UI
```

`UI-SCR-SETTINGS`의 음악 슬라이더는 `BGM`, 효과음 슬라이더는 나머지 네 SFX/UI 버스를 제어한다. Master 제어는 OS에 맡기며 별도 MVP 설정을 만들지 않는다.

### 3.3 덕킹과 보이스

- `P0_INFO` 발화 중 BGM은 6dB, `P2_TEXTURE`는 9dB 낮춘다.
- 덕킹 attack은 즉시성을 우선하고 release는 경고 끝을 가리지 않도록 부드럽게 복귀한다. 구체 ms 값은 기술 튜닝 데이터가 소유한다.
- 전체 동시 보이스 상한은 20이다. `P0_INFO`는 보이스 스틸 대상이 아니다.
- 같은 계열 작업 루프는 시설 수만큼 복제하지 않고 계열당 한 보이스를 사용한다. MVP에는 각 작업 시설이 하나뿐이어도 이 규칙을 유지한다.
- 과열 경고가 중첩되면 가장 위험한 단계 하나를 재생하고 나머지는 시각 방향 신호에 맡긴다.
- 동일 확인 SFX의 50ms 이내 중복 발화를 합친다. 탭 연속 입력이 클리핑을 만들지 않아야 한다.

## 4. MVP 음악 에셋 전수

### 4.1 목록

| 음악 ID | 파일 구성 | 길이 목표 | 사용처 |
|---|---|---:|---|
| `MUS_META_MVP` | 스테레오 루프 1 | 60~90초 | 타이틀, 지도, 상점, 설정. 화면별 볼륨만 다르게 사용 |
| `MUS_PLAY_MVP` | 스테레오 루프 1 | 70~90초 | 브리핑 저음량과 플레이 전체에 동일 파일 사용 |
| `JIN_CLIENT_SPROUT` | 모노 또는 좁은 스테레오 원샷 | 4~6초 | 새싹 원정대 첫 브리핑·계약 리콜 |
| `STG_RESULT_0` | 원샷 | 1.5~2초 | 0성 후퇴 |
| `STG_RESULT_1` | 원샷 | 약 2초 | 1성 신승 |
| `STG_RESULT_2` | 원샷 | 2~3초 | 2성 안정 공략 |
| `STG_RESULT_3` | 원샷 | 3~4초 | 3성 개선 행진 |
| `STG_CONTRACT` | 원샷 | 3~5초 | R5 첫 계약 체결 |

실제 풀 트랙은 메타 1곡과 플레이 1곡뿐이다. 적응형 스템과 결과 전용 루프를 만들지 않는다. 결과 스팅어 종료 후에는 `MUS_META_MVP`를 낮은 볼륨으로 사용한다.

### 4.2 플레이 음악 재생 상태

상태 전환 조건의 정확한 수치는 `BAL-MVP-001`이 소유한다.

| 상태 | 재생 | 전환 의도 |
|---|---|---|
| 브리핑 | `MUS_PLAY_MVP` 저음량 | 시간이 멈춘 계획 |
| 플레이 | `MUS_PLAY_MVP` 정상 볼륨 | 한 곡의 고정 작업 리듬 |
| 마감 임박 | 같은 곡 유지 | 음악 상태를 바꾸지 않고 시각 경고만 시작 |
| 최후 카운트다운 | 같은 곡을 약하게 덕킹 | P0 카운트다운이 앞으로 나옴 |
| 기한 도래 | 즉시 페이드 또는 음악적 마침 | 출정 신호와 결과 스팅어로 전환 |
| 일시정지 | 같은 재생 위치에서 볼륨 감소 | 차분한 관찰 공간, 시간 정지 인식 |
| 전화·시스템 오디오 인터럽트 | 모든 버스 즉시 정지 | 오디오 포커스가 돌아와도 자동 재생하지 않고 사용자 재개 뒤 같은 BGM 위치와 현재 작업 상태에서 복원 |

- 마감 임박에서 템포와 조성을 바꾸지 않는다.
- 재개 때 음악이 처음부터 다시 시작되지 않고 정지 전 음악 위치를 이어간다.
- 부팅·데이터 오류에서 별도 긴장 음악을 만들지 않는다. 메타 음악을 줄이거나 무음으로 처리한다.

| `worker_base__enhance` 대타격 프레임 | `SFX_ENHANCE_P1` 주 타격 | 기준 렌더 1프레임 이내 |
| `fac_furnace__fire` 점화 프레임 | `SFX_START_FURNACE` 점화 | 1프레임 이내 |
| `VFX_DELIVERY_STAMP` 도장 접촉 | `SFX_DELIVERY` 도장 | 1프레임 이내 |
| `VFX_STAR_STAMP` 접촉 | `SFX_STAR_STAMP` | 1프레임 이내 |
| 계약 도장 접촉 | `SFX_CONTRACT_STAMP` + `STG_CONTRACT` 마침표 | 1프레임 이내 |

저프레임 기기에서도 오디오는 시뮬레이션 이벤트를 놓치지 않아야 한다. 프레임 드랍이 있으면 애니메이션 프레임을 건너뛰더라도 이벤트 원샷은 한 번만 재생한다.

## 7. 햅틱 명세

### 7.1 원칙

- 햅틱은 설정에서 켬/끔만 제공한다.
- 지원하지 않는 기기에서는 햅틱 명령을 조용히 무시하고 시각·오디오 의미를 유지한다.
- 일상 탭과 모든 망치 타격에는 햅틱을 넣지 않는다. 정보 가치가 있는 사건에만 사용한다.
- 강도는 플랫폼 편차를 고려해 약·강 두 단계와 짧은 패턴으로 제한한다.

### 7.2 실제 패턴

| 햅틱 ID | 이벤트 | 패턴 | 시각·무음 대체 |
|---|---|---|---|
| `HAP_COMPLETE` | 작업 완료 | 약 단발 | 체크·트윙클 |
| `HAP_OVERHEAT_WARN` | 과열 유예 | 약 2연 | 모래시계·역방향 링 |
| `HAP_OVERHEAT_DANGER` | 과열 임박 | 강 3연, 경고 주기 반복 | 레드 점멸·`!`·연기 |
| `HAP_OVERHEAT_LOSS` | 산출물 소실 | 저강도 250ms | 연기·재·빈 시설 |
| `HAP_NO_WORKER` | 일손 없음 | 약 2연, 간격 짧음 | 앰버 윤곽·빈 일꾼·흔들림 |
| `HAP_DELIVERY` | 납품 | 약 단발 | 도장·점수 이동 |
| `HAP_ENHANCE_P1` | `+1` 확정 | 강 단발 | 보라 충격 링·`+1` 뱃지 |
| `HAP_DEADLINE` | `BAL-MVP-001`이 정한 최후 카운트다운 | 카운트다운 틱에 약 단발, 최대 3회 | 숫자 확대·카운트다운 |
| `HAP_STAR_3` | 3성 판정 | 약-약-강 상승 | 별 3개 스탬프 |
| `HAP_CONTRACT` | 첫 계약 | 강 단발 + 약 2연 | 계약 도장·봉랍 |
| `HAP_SETTINGS_PREVIEW` | 설정에서 햅틱 켜기 | 약 단발 1회 | 토글의 켬 상태와 짧은 시각 펄스 |

효과음 볼륨을 0으로 해도 햅틱 설정은 독립적으로 유지한다. 기기 무음 스위치를 감지해 설정을 자동 변경하지 않는다.

## 8. 반복 피로와 변주

- `SFX_UI_PRESS`, `SFX_ITEM_MOVE`, `LOOP_CRAFT`는 각각 3~4개 원본을 순차 라운드로빈한다.
- `P0_INFO`, 결과 스팅어, 강화 확정, 계약 도장은 변주하지 않는다.
- 피치 변주는 확인·질감음에만 ±30 cent 범위로 적용한다.
- 표현 변주 인덱스는 시뮬레이션 RNG를 소비하지 않는다. 오디오 계층의 단순 순환 카운터를 사용한다.
- 작업 루프는 30초 연속 재생에서도 어택이 날카롭게 누적되지 않아야 한다.
- 마감 틱은 `BAL-MVP-001`의 구간만 재생하고 일시정지 중 멈춘다. 재개 시 남은 초와 다시 동기화한다.

## 9. 납품·파일 규격

### 9.1 원본

- 음악: 48kHz, 24bit, stereo WAV
- SFX: 48kHz, 24bit WAV. 공간 방향성이 필요 없는 UI·정보음은 mono
- 루프: 시작·끝 zero crossing, sample-accurate loop marker 제공
- 무음 패딩: 입력 반응 원샷 앞에는 불필요한 선행 무음 5ms 이상 금지
- peak: clipping 없음, 원본 headroom 최소 3dB

### 9.2 런타임 권장

- 음악: stereo Vorbis streaming
- 짧은 `P0_INFO`와 탭 확인음: 저지연 메모리 상주 형식
- 기타 SFX: mono 압축 또는 플랫폼 적합 형식
- MVP 오디오 설치 용량 목표 12MB 이하: 음악 8MB, SFX 3MB, 예비 1MB

구체 코덱과 임포트 플래그는 `TECH-MVP-001`이 소유하되 위 지연·루프·용량 요구를 만족해야 한다.

### 9.3 경로와 이름

```text
audio/mvp/source/
  music/  stingers/  sfx/  haptics/

audio/mvp/runtime/
  music/  stingers/  sfx/
```

파일명은 이 문서의 대문자 ID를 소문자 snake_case로 변환한다. 라운드로빈은 `_01`, `_02` 접미사를 쓴다. 예: `sfx_item_move_01.wav`.

## 10. 실제 에셋 수량

| 분류 | 수량 | 비고 |
|---|---:|---|
| 풀 음악 작곡 | 2곡 | 메타 1, 플레이 1 |
| 거래처 징글 | 1 | 새싹 원정대만 |
| 결과·계약 스팅어 | 5 | 0~3성 + 계약 |
| SFX 이벤트 | 39 | §5의 ID 전수 |
| SFX 원본 파일 | 약 48~54 | 라운드로빈·루프 변주 포함 |
| 햅틱 패턴 | 11 | §7.2 전수 |

이 수량에는 MVP 카탈로그와 화면 이벤트 밖의 사운드가 포함되지 않는다.

## 11. 명시적 비범위

- MVP 카탈로그 밖 거래처·시설·아이템·강화 SFX와 음악
- 보이스오버, 대사 음성, 전투·몬스터 사운드
- 현재 무료 MVP의 결제·페이월·구매 복원·광고 사운드
- 클라우드·계정·업적·알림 사운드
- 3D 위치 오디오와 고급 오디오 미들웨어 전용 기능
- 모노 다운믹스, 무음 부스트 같은 출시판 후보 설정

후속 유료 스테이지가 승인되기 전에는 결제·복원 이벤트용 무음 자리나 SFX를 선제 제작하지 않는다. 승인될 경우 `UX-MVP-001`, `UI-MVP-001`을 먼저 개정한 뒤 이 문서에 필요한 오디오 피드백을 추가한다.

## 12. 검증 방법

- 최소 사양 기기 스피커에서 플레이 음악과 작업 루프가 모두 재생되는 중에도 `SFX_OVERHEAT_DANGER`를 즉시 식별한다.
- 화면을 보지 않고 작업 완료, 과열 유예, 과열 임박, 일손 없음, 인벤토리 만석을 구분한다.
- 소리와 햅틱을 모두 끄고도 `UI-MVP-001`의 시각 신호만으로 같은 상태를 구분한다.
- 강화 착수부터 완료까지 도박·확률 성공으로 오해할 만한 소리나 지연이 없다.
- 일시정지 중 카운트다운·작업 루프가 멈추고 재개 후 남은 상태와 맞는다.
- 전화·시스템 오디오 인터럽트가 끝나도 일시정지 중 자동 재생하지 않으며, 사용자 재개 후 BGM·작업 루프가 중복되지 않는다.
- 10회 연속 재도전에서 탭·망치·카운트다운 소리의 피로도를 외부 테스터가 평가한다.
- SFX 이벤트 로그와 실제 재생 로그를 비교해 이벤트당 원샷이 0회 또는 2회가 되는 버그가 없다.
- 햅틱 지원 Android 실기와 햅틱 미지원 Android 실기에서 각각 설정 켬/끔, 조용한 명령 무시와 시각·오디오 폴백을 검증한다.
- 런타임 오디오 폴더에 §10 외의 미사용·미래 에셋이 없다.

## 14. 완료 조건

이 문서는 다음을 모두 만족할 때 `Review`로 전환한다.

- `UI-MVP-001`의 모든 오디오·햅틱 요구가 §4~7의 에셋 또는 명시적 무음 처리에 매핑된다.
- `CNT-MVP-ROUND-001`의 R1~R5와 `+1` 강화까지 필요한 이벤트만 포함된다.
- 음악 2곡, 거래처 징글 1개, 0~3성·계약 스팅어, SFX 39개, 햅틱 패턴 11개가 전수 표와 파일 manifest에서 일치한다.
- 정보음 우선순위, 덕킹, 동시 재생, 반복 변주의 규칙이 구현 가능한 형태로 정의된다.
- 무음·햅틱 미지원·최소 스피커의 대체와 검증 방법이 있다.
- 현재 무료 MVP에 없는 결제와 후속 콘텐츠용 오디오가 포함되지 않으며, 후속 유료 스테이지 승인 시 문서를 먼저 개정한다.
