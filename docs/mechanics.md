# 던전 오피스 (Dungeon Office)
## 코어 메커닉스 명세 — 라운드 안의 규칙 (mechanics.md)

> 구 GDD(v1.9 코어 시스템 명세)에서 **라운드 안(in-round)의 코어 메커닉스**를 추출·재구성한 모듈이다. 진행 구조·계약·판정/별·정산·영속성 등 라운드 밖의 규칙은 자매 모듈 progression.md가 담당한다.

---

## 1. 문서 안내

### 1.1 이 문서의 성격

이 문서는 **라운드 안의 규칙 — "손의 시간"**을 다룬다. 납품기한까지 흐르는 시간 위에서 일꾼·불·시설·인벤토리를 굴려 장비를 만들어 납품하는, 그 한 판의 작동 방식만을 규정한다. 배경지식 없는 독자가 이 문서만으로 **라운드가 어떻게 굴러가는가**를 완전히 이해하는 것을 목표로 한다.

### 1.2 문서 체계

```
[상위: GDD] ── 컨셉·재미의 축·UX·설계 원칙(헌법)
   ├── [이 문서: mechanics.md] ── 라운드 안의 규칙 (손의 시간)
   ├── [자매: progression.md]  ── 라운드 밖의 규칙 (머리의 시간)
   │                              진행 구조·계약·판정/별·정산·영속성·상점
   ├── 아이템·레시피 설계서 (items)
   ├── 스테이지·라운드 설계서 (stages)
   ├── 밸런스·경제 수치서 (balance)
   ├── UI·화면 설계서 (ui-ux)
   └── 아트·테마 가이드
```

- **충돌 시 GDD가 이긴다.** 이 문서와 GDD가 어긋나면 GDD를 먼저 개정한다. progression.md와의 경계가 모호한 항목도 GDD의 설계 원칙(GDD §3, 이 문서에서는 "GDD 원칙 N"으로 인용)이 최종 심판이다.
- 본 문서에 등장하는 구체적 명칭·숫자는 **설명용 예시**이며 확정값이 아니다. 확정은 하위 문서의 몫이다. 예외 — 이 문서가 "구조적 확정"으로 명시한 값(계열 상한 §5.2, 일꾼 수 범위 §4)은 코어 명세의 일부다.
- 화면 구성·레이아웃·연출 등 **화면 표현**은 ui-ux.md의 몫이다. 이 문서는 "무엇을 하는가(추상 조작·규칙)"만 규정한다.

### 1.3 다루는 것 / 다루지 않는 것

라운드 안은 손의 시간, 라운드 사이는 머리의 시간이다. 이 문서는 앞의 절반을 맡는다.

| 이 문서(mechanics.md)가 다룬다 | progression.md가 다룬다 |
|--------------------------------|--------------------------|
| 조작 모델 — 운반·착수·수납·파기 (§3) | 진행 구조 — 가도·스테이지·라운드·순차 해금 |
| 일꾼·동시성·다인 작업 (§4) | 브리핑·라운드의 흐름·입장 요건 |
| 대장간 시설과 계열 상한제 (§5) | 판정 — 총점 → 공략 결과 → 별 |
| 제작 파이프라인 — 제련·제작·강화 (§6) | 정산 — 별 등급별 골드, 재플레이 지급 |
| 인벤토리 (§7) / 주의력과 과열 (§8) | 계약·장부·누적 별 게이트 |
| 의뢰의 도착·매칭·철회 (§9) | 영속성(무엇이 남고 리셋되는가)·상점 |
| 강화 (§10) / 일시정지·세션 시간 (§11) | 재도전·수익 모델 |

- 수치(시간·배율·가격·커트라인·용량)는 모두 **밸런스 문서**의 몫이다 — 이 문서는 노브의 존재와 방향만 정한다.
- 재료·장비·레시피의 목록과 해금 배치는 **items·stages 설계서**의 몫이다.

### 1.4 이 문서가 지키는 라운드 안의 재미

이 문서가 규정하는 규칙은 두 가지 쾌감을 위해 존재한다. 하위 문서는 이 둘을 훼손하지 않는 선에서 자유롭다.

- **일손 굴리기** — 일꾼들과 불로 여러 타이머를 저글링하는 스케줄링의 쾌감. 과열 직전의 회수, 무엇을 만들고 무엇을 흘려보낼지의 트리아지. 실력은 손놀림이 아니라 **일의 순서를 짜는 머리**에서 나오며, 동시성의 상한은 언제나 일꾼 수다.
- **마감의 드라마** — 예고된 대형 강화 의뢰라는 미니 프로젝트를 기한 전에 완주하는 클러치. 이 게임에 주사위는 없다 — 같은 판단은 언제나 같은 결과를 낳고, 심장을 뛰게 하는 것은 주사위가 아니라 마감이다.

## 2. 핵심 용어 (라운드 안)

라운드 안에서 손이 다루는 대상과 행위의 용어다. 진행·거래 용어는 라운드 밖의 개념이므로 여기서 정의하지 않는다:

> **progression.md 참조** — 거래처, 스테이지, 라운드, 계약, 브리핑, 납품기한, 공략 결과/별, 장부(도감), 골드(G). (골드는 일꾼 고용·대장간 성장에 쓰는 영구 재화로, 라운드 정산으로 얻는다.)

### 사람과 공간
| 용어 | 정의 |
|------|------|
| **일꾼** | 실제로 일하는 캐릭터. **작업 하나가 일꾼 하나를 점유** — 일꾼 수 = 동시 작업 수의 상한. 고용은 영구. 시작 2명, 상한 6명 (§4) |
| **대장간** | 작업장. 일꾼·시설·시설 수·시설 등급이 영구 유지되는, 성장이 쌓이는 곳 (영속성은 progression.md) |
| **시설(스테이션)** | 대장간의 기능 지점. 변환·합성·제작·강화 4계열, 종류 수에 영구 상한 (§5.2) |
| **인벤토리** | 라운드 안의 공용 보관함. **무엇이든 넣을 수 있다.** 용량 유한, 라운드 종료 시 소멸 (§7) |
| **쓰레기통** | 기본 제공 시설. 아이템을 자발적으로 파기하는 유일한 수단 (운반과 같은 탭-탭, 즉시·일꾼 불필요) |

### 물건과 행위
| 용어 | 정의 |
|------|------|
| **재료 / 장비 / 부품** | 원료(원자재·가공재) / 납품 대상 완성물(무기·방어구·장신구) / 조립용 중간 제품(특수 의뢰 전용) |
| **제련** | 재료로 새 재료를 만들기. **변환**(1→1)과 **합성**(2→1) 두 규칙뿐 (§6.1) |
| **제작** | 재료로 장비 만들기. 분류별 제작대에서 일꾼이 수행 (§6.2) |
| **강화** | 장비의 단계를 +1 올리는 결정적 공정. 강화석 1개를 소모하며, **실패는 없다** (§10) |
| **강화 단계 (+N)** | 장비의 강화 횟수. 일부 의뢰는 최소 단계를 요구하며, 그만큼 높은 점수를 건다 |
| **납품** | 장비를 납품 카운터에 올려 의뢰를 완료하기. 점수를 얻는 유일한 방법 (매칭 §9) |
| **과열** | 변환 완료 후 제때 회수하지 않으면 재료가 소실되는 것 — **이 게임의 유일한 비자발적 손실원** (§8) |
| **강화석** | 강화 1회의 연료. 라운드 안에서 제련으로 제작하며, 라운드 종료 시 소멸 |

### 의뢰
| 용어 | 정의 |
|------|------|
| **의뢰** | 장비 요청 1건. 순차적으로 밀려오며 고르거나 거절할 수 없다. 점수는 카드에 적힌 값이다 (§9) |
| **인내 시간** | 의뢰별 제한 시간. 다하면 철회된다. 철회의 대가는 그 점수를 벌 기회의 상실뿐이다 |

## 3. 조작 모델

### 3.1 아바타 없음, 사장의 손가락

조종하는 캐릭터는 없다. 화면 속에서 뛰는 것은 일꾼들이고, 플레이어는 탭으로 지시하는 사장이다. 모든 조작은 탭이다 — 홀드도, 연타도, 타이밍 판정도 없다. 손맛은 제스처가 아니라 피드백(일꾼의 망치질 애니메이션·소리)에 담는다. 구체적 화면 표현·연출은 ui-ux.md의 몫이다.

### 3.2 조작 문법 — 무엇을 하는가

아래는 **추상 조작**의 규정이다. 화면상의 탭 UX·레이아웃·표현은 ui-ux.md를 참조한다.

| 조작 | 하는 일 | 규칙 |
|------|---------|------|
| **운반** | 재료·장비를 출발지에서 목적지로 옮긴다 | 모든 이동. 즉시 처리, 개수 제한 없음 — 연출상 일꾼이 나르지만 규칙상 시간 비용은 없다 |
| **착수** | 재료가 놓인 시설의 작업을 시작한다 | 유휴 일꾼 1명이 배정되어 작업 시작. 이후 진행은 자동 (불은 예외 — §4) |
| **수납** | 완성 산출물을 인벤토리로 넣는다 | 산출물이 인벤토리로 들어가고 시설이 비워진다 (원탭) |
| **파기** | 아이템을 쓰레기통에 버린다 | 운반과 같은 탭-탭, 즉시·일꾼 불필요. 자발적 파기의 유일 수단 (§6.3) |
| **일시정지** | 시간을 멈춘다 | §11 |

## 4. 일꾼 — 동시성의 화폐

이 게임의 핵심 자원은 시간이 아니라 **일손**이다.

- **작업 하나 = 일꾼 하나.** 합성·제작·강화는 시작 시 유휴 일꾼 1명을 배정받아 작업 시간 동안 점유하고, 끝나면 풀려난다.
- **불은 일꾼이 아니다.** 용광로(변환)는 재료를 넣으면 혼자 탄다 — 일꾼을 쓰지 않는다. 대신 과열이 있다. 일꾼-시간이 드는 작업(동기)과 불-시간만 드는 작업(비동기)의 교차가 스케줄링 깊이의 원천이다.
- 동시 작업 수 = **min(유휴 일꾼, 해당 시설의 빈자리)**. "다음 일꾼이냐, 다음 시설이냐"는 서로 다른 병목을 푸는 별개의 구매 선택이다.
- **다인 작업**: 작업의 일꾼 요구 수는 1이 기본이나, 특수 작업(예: 부품의 최종 조립)은 2명 이상을 요구할 수 있다 — 스테이지 설계서가 쓸 수 있는 규칙 노브다.
  - 단, 시작 인원이 2명이므로 **다인 작업 의뢰는 일꾼 3명 이상을 전제하는 구간부터** 배치한다 (2명 구간의 2인 작업은 대장간 전체를 세운다 — 스테이지 배치 제약, stages §0).
- **대기열은 없다.** 일꾼이 전부 바쁠 때의 착수 탭은 거절된다. 예약이 아니라 **지금 시킬 수 있는 일을 고르는 것**이 이 게임의 판단이다.
- **진행 중인 작업은 취소할 수 없다.** 착수가 곧 확정이다 — 대기열 없음과 짝을 이루어, 판단의 무게를 착수의 순간에 싣는다.
- 유휴 일꾼 수는 항상 표시된다 (표현은 ui-ux.md).
- 일꾼은 골드로 고용하며 영구히 유지된다 (고용 행위 자체는 라운드 밖 — progression.md).
  - **시작 2명, 상한 6명 — 계열 상한제(§5.2)와 같은 급의 구조적 확정이다.**
  - 동시성 2~6이 이 게임이 감당하는 복잡도의 전 범위이며, UI·콘텐츠·밸런스는 이 범위에 커밋한다 (고용 가격과 증가 곡선은 밸런스 문서).
  - 현재 일꾼은 수만 다르다 — 개성·전문화는 보류 (GDD 부록 — 보류 개념).

## 5. 대장간과 시설 — 계열 상한제

### 5.1 시설 구성

- **기본 제공**: 재료 공급함(원자재 무한), 변환 시설 1, 무기 제작대 1, 납품 카운터, **쓰레기통**(파기 — 운반과 같은 탭-탭, 즉시·일꾼 불필요), 설치 슬롯 약간.
- **골드로 사는 구매 축**은 다음과 같다 (구매 행위·가격·영속성·상점의 규칙은 라운드 밖 — progression.md):
  - 나머지 시설의 **신규 설치**
  - 같은 시설의 **추가 설치**(수) — 동시 작업의 빈자리를 늘린다 (§4)
  - 시설 **레벨업**(등급)
  - **설치 슬롯** 확장
  - **인벤토리** 확장 (§7)
  - **도구**(패시브)
  - **일꾼** 고용 (§4)
- 어떤 시설을 언제 여는가(해금 배치)는 items·stages 설계서의 몫이다. 이 문서가 정하는 것은 **구매 축의 종류**와 **계열 상한**뿐이다.

### 5.2 계열 상한제

모든 시설은 네 계열에 속하고, 각 계열의 종류 수에는 **영구 상한**이 있다.

| 계열 | 규칙 | 상한 |
|------|------|------|
| 변환 | 재료 1 → 재료 1. 불이 일하고(일꾼 불필요) 과열이 있다 | 최대 3종 |
| 합성 | 재료 2 → 재료 1. 일꾼 점유 | 최대 3종 |
| 제작 | 재료 → 장비. 일꾼 점유. 장비 분류(무기·방어구·장신구)별 전용 제작대 | 최대 3종 — **확정** |
| 강화 | 장비 +1 단계. 결정론. 일꾼 점유 | **단 1종** |

- 게임의 시설 분류학은 영원히 10종 이내다 (변환 3 · 합성 3 · 제작 3 · 강화 1 = 최대 10종 — GDD 원칙 6). 콘텐츠가 늘어도 배울 "새 기계"는 없다 — 늘어나는 것은 재료·레시피·의뢰(데이터)뿐이다.
- 제작 상한이 이미 차 있다는 것은 **장비 분류가 영원히 셋**이라는 커밋이다.
- 강화가 단일 시설인 이유: 강화는 공정의 정점이며, 정점으로 가는 문은 하나여야 배우기 쉽다.

## 6. 제작 문법 — 세 가지 동사

모든 생산은 **제련 → 제작 → (강화) → 납품**의 파이프라인이다.

```
공급함 ──▶ [제련: 변환·합성] ──▶ [제작] ──▶ [강화 (의뢰 요구 시)] ──▶ [납품]
원자재        가공재 생성          장비 생성      단계 상승             점수
```

### 6.1 두 개의 생산 규칙

재료 생성 규칙은 아래 둘뿐이며, 콘텐츠가 아무리 늘어도 이 위에서만 늘어난다.

- **변환**: 재료 1개 → (변환 시설) → 다른 재료 1개. 자동 진행 + 과열.
- **합성**: 재료 2개 → (합성 시설) → 새 재료 1개. 일꾼 점유.

### 6.2 제작과 부품

- 제작은 장비 분류에 대응하는 제작대에서 한다.
- **이행 가능성은 입장에서 보장된다**: 라운드는 이행 불가능한 의뢰를 품은 채 시작될 수 없으므로(입장 요건 — progression.md), 라운드 안의 모든 의뢰는 정의상 이행 가능하다. 어떤 설비가 어떤 의뢰를 여는지는 라운드의 입장 요건으로 표현된다.
- **부품과 조립**: 제작의 산출물은 **완제품(장비)과 부품** 두 가지다.
  - 여러 부품을 각각 제작한 뒤 최종 조립해 완성하는 대형 장비가 존재할 수 있으며, **조립 역시 제작이다** — 부품을 재료로 삼아 해당 분류의 제작대에서 수행하고, 다인 작업(§4)의 대표 용례다.
  - 통상의 제작은 1단계로 끝나며, 부품·조립은 특수 의뢰 전용이다 (배치는 스테이지 설계서 재량).

### 6.3 손실의 원칙

- 레시피에 없는 조합은 시설이 받지 않는다 (아무 일도 일어나지 않음). 잘못 만든 물건은 존재하지 않는다.
- **쓰레기통은 정리의 도구다.** 과잉 생산·빗나간 투기로 자리를 차지한 아이템은 쓰레기통으로 자발적으로 파기한다 — 이것은 손실이 아니라 선택이다.
- **비자발적 손실의 출처는 과열 하나뿐이다.** 강화에 실패란 없고(§10), 의뢰 철회는 기회의 상실이지 물건의 상실이 아니다. 억울한 손실이 없어야 실패가 분노가 아니라 학습이 된다. (과열의 자리는 §8.)

## 7. 인벤토리

- 라운드 안의 공용 보관함. **무엇이든 넣을 수 있다** — 원자재부터 부품·강화석·완성 장비까지. 시설 위의 산출물은 원탭으로 수납되어 시설이 비워지고, 꺼내 쓰기는 탭-탭이다.
- **존재 이유**: 산출물이 시설을 막는 병목(§8)과 "미리 만들어 두기"라는 계획 플레이를 양립시키는 완충 공간이다.
  - 브리핑(예고, progression.md) — 인벤토리(비축) — 결정론(§10)의 삼각이 **"예측하고 준비하는" 플레이를 공식화**한다.
- **용량은 유한**하고, 확장은 구매 축이다 (수치·가격은 밸런스 문서). 무한이면 병목 압박이 소멸하므로 유한성 자체가 시스템이다.
- **라운드 종료 시 소멸**한다. 라운드 간 이월은 없다 — 매 라운드는 빈손에서 시작하는 공평한 스프린트다. (라운드마다 리셋되는 것 전체의 목록은 progression.md.)

## 8. 주의력의 경제와 과열

플레이어의 눈은 항상 세 개의 시계를 오간다.

- **용광로(변환)**: 완료 벨 후 짧은 과열 유예 — 유일하게 "돌아와야 하는" 시설. 긴장의 제1원천.
- **시설 출구**: 산출물을 치우기 전까지 그 시설은 막혀 있다. 방치의 벌은 소실이 아니라 병목.
- **의뢰 게이지**: 인내가 마르면 의뢰가 철회된다 — 그 점수를 벌 기회가 사라진다 (§9).

난이도는 이 세 시계의 속도와 일꾼 대비 일감의 비율로 조절한다 (값은 밸런스 문서).

**과열은 이 게임의 유일한 비자발적 손실원이다.** 세 시계 중 물건을 실제로 잃게 하는 것은 용광로의 과열 하나뿐이다.

- 시설 출구의 방치는 병목(기회 지연)이지 소실이 아니다.
- 의뢰 게이지의 고갈은 기회의 상실이지 물건의 상실이 아니다.
- 강화에는 실패가 없고(§10), 레시피 밖 조합은 아무 일도 일으키지 않으며(§6.3), 파기는 자발적 선택이다.

억울한 손실이 없어야 실패가 분노가 아니라 학습이 된다 (GDD 원칙 3). 손실의 원칙 전문은 §6.3.

## 9. 의뢰

*판정·정산(총점 → 별 → 골드)은 라운드 밖의 규칙이다 — **progression.md**를 참조한다. 이 절은 라운드 안에서 의뢰가 어떻게 도착·매칭·소멸하는가만 다룬다.*

- 의뢰는 정해진 칸에 **순차적으로** 도착한다. 고르거나 거절할 수 없다 — 다만 무엇을 만들고 무엇을 흘려보낼지는 전적으로 플레이어의 선택이다.
- 의뢰 카드의 구성:
  - 요청 장비
  - **요구 강화 단계** (+0이 기본)
  - 점수 (카드에 적힌 값)
  - 인내 게이지
- **의뢰의 점수는 카드에 적힌 값이다.** 기본은 장비 기본점이며, 강화 요구 의뢰는 추가 공정에 비례한 할증이 붙는다. 특별 의뢰(단골 재방문, 이벤트 등)는 그 위에 추가 할증을 실을 수 있다 — 스테이지 설계 재량이며, 이 조항이 단골·이벤트 콘텐츠의 시스템적 근거다.
- 납품 카운터에 장비를 올리면 **같은 장비이고 요구 단계가 장비의 단계 이하인 의뢰** 중 최고점과 자동 매칭된다. 초과 강화분에 대한 추가 보상은 없다 — 투기적 선강화의 대가는 스스로 진다 (물건을 잃는 것은 아니므로 손실 원칙과 충돌하지 않는다).
- 인내가 다한 의뢰는 철회되고 새 의뢰가 온다. **철회의 대가는 그 점수를 벌 기회의 상실뿐이다** — 별도 감점도, 그 밖의 벌칙도 없다. 의뢰를 버리는 것은 처벌이 아니라 전략이다.
- 의뢰의 등장 속도·구성·던전 테마에 따른 비중은 라운드 설계의 제1 노브다 (스테이지 설계서 + 밸런스 문서).
  - 이행 불가능한 의뢰는 입장 요건(progression.md) 덕에 애초에 등장하지 않는다 — 죽은 카드는 없다.
  - 공정이 깊은 의뢰(강화·부품)일수록 인내와 예고가 그 깊이를 감당할 수 있어야 한다 — 브리핑 예고와 긴 인내는 대형 의뢰의 의무 사양이다.

## 10. 강화 — 가장 깊은 공정

### 10.1 규칙

완성 장비를 강화 시설에 올리고 강화석 1개를 소모하면, 장비의 단계가 **+1 오른다. 그게 전부다** — 확률도, 실패도, 파괴도 없다. 일꾼 1명을 점유하는 보통의 작업일 뿐이다.

- 강화 1회(패스) = 강화석 1개 + 일꾼 작업 1회 = 단계 +1. 상위 단계는 패스를 거듭해 도달한다 (+2 = 패스 2회 + 강화석 2개).
- 단계 상한은 소수로 유지한다 (값은 밸런스 문서).
- **강화가 존재하는 이유는 의뢰가 요구하기 때문이다.** "+2 마강검" 의뢰는 강화석 2개의 제련 라인과 강화 패스 2회가 걸린 미니 프로젝트이며, 그 노동만큼 높은 점수를 건다 (GDD 원칙 5).

### 10.2 설계 의도

- 강화는 도박이 아니라 **파이프라인의 가장 깊은 층**이다. 저가 의뢰가 한 줄짜리 공정이라면, 고단계 강화 의뢰는 제련-합성-제작-강화가 전부 맞물리는 종합 시험이다.
- 결정론의 힘: 실패가 없으므로 **모자랄 수 있는 것은 오직 시간과 준비뿐**이다.
  - 클러치의 긴장은 주사위가 아니라 마감에서 나오고, 브리핑의 예고 — 인벤토리의 비축(§7) — 강화의 결정론이 삼각을 이루어 "예측하고 준비하는" 실력을 보상한다.
- 강화 요구 의뢰의 비율·단계 분포는 후반으로 갈수록 깊어지는 난이도 사다리의 주축이다 (스테이지·밸런스 문서).

## 11. 일시정지와 세션 시간

- **일시정지는 언제든, 조건 없이.** 시간·타이머·진행이 전부 멈추고, 재개하면 그 순간부터. 앱 이탈 시 자동 정지·저장.
- **일시정지 중 화면 관찰과 계획은 허용이다.** 정지하고 판을 읽는 것은 치팅이 아니라 플레이 스타일이다 (오버쿡드 등 선례).
  - 이 게임의 실력은 순간 감시가 아니라 **무엇을 언제 시키는가의 판단**에 있으므로, 정지-계획은 실력의 우회가 아니라 행사다.
  - 단, **정지 중 조작(탭 지시)은 불가능하다** — 실행은 언제나 흐르는 시간 위에서만 한다.
- 라운드는 한 세션 안에 완결되는 길이로 설계한다 (목표 길이는 밸런스 문서). 에너지·입장권 게이트는 없다.

## 12. 아이템 설계서에 부과하는 제약

구체적 재료·장비·레시피는 아이템·레시피 설계서(items)의 몫이되, 다음을 지켜야 한다:

1. 생산 규칙은 변환(1→1)과 합성(2→1)만 사용한다.
2. 장비 분류는 무기·방어구·장신구 셋으로 고정한다.
3. 장비의 기본 점수는 대략 "투입 노동 시간 × 공정 깊이"에 비례시킨다.
4. **용처가 여럿인 전략 자원을 정확히 1종** 둔다 (강화석 연료·상위 합성·고가 장비가 한 재료를 두고 경쟁하는 구조). 나머지 재료는 용처가 단순한 벌크로 남겨, 판단 피로를 한곳에 모은다.
5. 기본 제공 제작대(무기) 라인만으로 저가부터 고가까지의 점수 사다리가 완성되어야 한다 — 구매는 시장 확장이지 통행료가 아니다.
6. **강화석 레시피와 강화 시설의 해금은 동기화한다** (한쪽만 열린 상태 금지 — 스테이지 설계서와 공동 책임).

## 13. 부록: 미결 사항 (메커니즘) — 검증으로 답할 질문

라운드 안의 메커니즘에 대해 플레이 검증으로 답할 열린 질문들이다.

1. **투기 생산의 템포** — 브리핑 예고 → 선제 양산 → 인벤토리 비축이 의도된 플레이로 공식화된 지금, 인벤토리 용량 노브(§7)만으로 라운드의 긴장이 유지되는지.
2. **회생 불가 라운드의 체감** — 조기 정산이 없는 구조(progression.md)에서, 이미 실패가 확실한 라운드를 기한까지 두는 것이 지루함이 되는지 (완화 수단: 짧은 라운드 길이, 일시정지 재시작).
3. **병렬 운영의 자생력** — 콤보 없이(GDD 부록 B) 점수 합산만으로 "대형 라인 + 소형 납품 병행"이 자연 발생하는지, 아니면 대형 의뢰 집중이 지배 전략이 되어 라운드 중반이 단조로워지는지 (커트라인 설계와 공동 검증 — progression.md·밸런스 문서).
4. **대기열 없음 + 취소 불가** — 시작 일꾼이 2명으로 확정된 지금(§4), 초반 구간이 답답함으로 느껴지는 구간이 있는지.
5. **강화 요구 의뢰의 성립** — 예고·인내·할증의 조합이 공정 깊이를 실제로 감당하는지 (§10; 밸런스 문서와 공동 검증).

> 부록 C의 나머지 두 질문 — **입장 요건 벽의 체감**과 **누적 별 게이트의 체감** — 은 진행 게이트에 관한 것으로 **progression.md**가 다룬다.

## 하위 문서에 대한 위임·제약 요약

이 문서는 라운드 안의 규칙을 확정하고, 그 아래 네 하위 문서에 다음을 위임·제약한다. **items**(아이템·레시피 설계서)에는 재료·장비·부품 목록과 레시피·기본 점수를 위임하되 §12의 6개 조항과 계열 상한(§5.2)·손실의 원칙(§6.3)을 지키게 한다. **stages**(스테이지·라운드 설계서)에는 의뢰 구성·던전 테마·다인 작업(§4)·부품 조립(§6.2)의 배치를 위임하되 새 조작을 발명하지 않고 데이터로만 변주하게 하며, 다인 작업은 일꾼 3명 이상 구간부터 배치하게 한다. **balance**(밸런스·경제 수치서)에는 모든 수치 — 작업 시간·과열 유예(§8)·인내·라운드 길이(§11)·의뢰 동시 칸 수·강화 단계 상한(§10)·인벤토리 용량(§7)·일꾼 고용 곡선(범위 2~6은 §4에서 확정) — 를 위임하되 깊은 공정이 항상 값을 하도록(GDD 원칙 5) 제약한다. **ui**(UI·화면 설계서)에는 조작(§3)의 화면 표현·연출과 유휴 일꾼 수 상시 표시·의뢰의 요구 단계 식별을 위임한다. 라운드 밖의 규칙(진행·계약·판정/별·정산·영속성·상점)은 전부 자매 모듈 progression.md의 몫이다.
