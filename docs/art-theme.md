## 1. 비주얼 스타일 — 확정

### 1.1 결정: **2D 픽셀아트 (HD 픽셀, 정수배 스케일)**

- **결정**: 전 게임 픽셀아트. 캐릭터 2등신 SD, 일꾼 기준 캔버스 48×48px(실작화 높이 약 40px), 대장간 타일 그리드 32px, 아이템 아이콘 원판 32px(표시 시 2배 확대 기본). 렌더링은 정수배 스케일 + 니어리스트 필터만 허용 — 흐릿한 픽셀은 금지.
- **근거 1 (제작량)**: 이 게임의 손맛은 조작이 아니라 반복 재생되는 피드백 애니메이션이다(§5.1). 저해상도 프레임 애니메이션은 1인이 감당 가능한 유일한 "많은 모션" 노선이며, 팔레트 스왑·파츠 교체로 일꾼 6인 변형과 스킨 DLC(§11.2)가 거의 공짜로 나온다.
- **근거 2 (가독성·톤)**: 픽셀아트는 강제된 단순화 덕에 세로 소형 화면에서 실루엣이 뭉개지지 않고, SD 픽셀은 "귀엽고 폭넓게"와 자연스럽게 합치한다. 도트 특유의 레트로 공예 감성이 "대장간 수공업"이라는 소재와도 맞는다.
- **기각한 대안**:
  - 플랫 벡터 — 스켈레탈 리깅(Spine 등) 학습·제작 비용이 크고, 모바일 광고형 캐주얼 게임의 기시감이 유료 프리미엄 포지셔닝과 충돌한다.
  - 손그림(수채·크레용) — 낱장 단가가 가장 높고, 시설 상태 4종×10종 같은 파생 변형마다 다시 그려야 해 1인 개발과 상극이다.
  - 3D/2.5D — 파이프라인 전체가 과잉. 논외.

### 1.2 레퍼런스 게임 (5)

| 레퍼런스 | 가져올 것 |
|---|---|
| **카이로소프트 시리즈** (게임발전국 등) | 세로 화면 SD 픽셀 경영물의 밀도와 친근함, "작은 사람들이 바쁘게 일하는" 화면의 매력 |
| **데이브 더 다이버** | 픽셀 캐릭터 + 모던 클린 UI의 하이브리드 — 본작 UI의 기준 |
| **스타듀 밸리** | 따뜻한 팔레트, 1인 제작 픽셀아트의 품질 상한선 |
| **오버쿡드** | 상태 색 언어(경고·완료·진행)의 명료함, 정지-계획 플레이의 선례(§3.3) |
| **굿 피자, 그레잇 피자** | 세로 한 손 가게 운영물의 톤, 단골 손님 연출의 정서 |

### 1.3 스타일 규칙

- 외곽선: 캐릭터·아이템은 짙은 웜 브라운(순검정 금지) 1px 아웃라인. 배경·바닥은 아웃라인 없이 면으로 — 전경 가독성을 배경과의 문법 차이로 확보한다.
- 픽셀 밀도 혼용 금지: 동일 씬 안에서 확대 배율이 다른 스프라이트를 섞지 않는다.
- UI 텍스트·숫자는 픽셀 폰트를 쓰되 판정·점수 등 정보성 숫자는 가독 우선의 큰 사이즈로 — "레트로 감성이 정보를 가려서는 안 된다"가 원칙.

## 2. 컬러·톤 시스템

### 2.1 마스터 팔레트

- **결정**: 마스터 팔레트 총 40색 내외로 제한 운용(Lospec 계열 웜톤 팔레트 기반 커스텀). 기조는 **"화덕의 주황 × 무쇠의 청회색"** — 따뜻한 앰버·오렌지(불·완성·보상)와 차분한 청회·네이비(철·돌·바닥)의 보색 대비가 대장간의 정체성이다.
- 근거: 제한 팔레트는 1인 제작에서 화면 전체의 통일감을 자동으로 보장하고, 스킨 DLC의 팔레트 스왑 기반이 된다.
- 기각: 무제한 풀컬러 — 에셋이 늘수록 톤이 흩어지고, 상태 색 언어(2.2)의 배타성이 무너진다.

### 2.2 상태 색 언어 — 배타적 예약

비자발적 손실의 출처는 과열 하나뿐이므로(원칙 3), **경고색의 위계도 하나여야 한다.**

| 상태 | 색 | 규칙 |
|---|---|---|
| **과열 경고** | **순수 레드 (#E8382F 계열) + 점멸** | **게임 전체에서 레드+점멸 조합은 과열 전용으로 예약.** 다른 어떤 UI·연출도 이 조합을 쓸 수 없다. 게임 유일의 손실원 = 게임 유일의 최상급 신호 |
| 의뢰 인내 게이지 | 청록 → 앰버 → **회색으로 식음** | 레드에 닿지 않는다. 철회는 손실이 아니라 기회의 소멸(§9.1)이므로 "위험"이 아니라 "식어감"으로 그린다 |
| 강화 +N | 자수정 퍼플 (+금색 상한 강조) | 강화·강화석·강화 시설을 관통하는 계열색. 공정의 정점(§10)에 왕가의 색을 준다 |
| 작업 진행 | 스카이 블루 진행 링 | 안심·자동 진행의 색 |
| 완료·수납 대기 | 골드 옐로 반짝 | 보상 예감의 색. 단, 점멸이 아닌 부드러운 트윙클(점멸은 과열 전용) |
| 유휴 일꾼 | 화이트 계열 하이라이트 | "쓸 수 있는 손"의 색 |

- 색약 대응: 모든 상태 신호는 **색 + 형태(아이콘·점멸·모션)의 이중 코딩**을 의무로 한다. 과열은 레드 점멸에 더해 연기 이펙트+벨 아이콘이 항상 동반된다.

### 2.3 라운드 안 vs 라운드 사이 — 톤 대비

- **라운드 안 (손의 시간)**: 고채도·고대비. 화덕 불빛이 살아 있는 웜 라이팅, 배경은 살짝 어둡게 눌러 전경(일꾼·시설·산출물)을 띄운다. 긴장은 색온도로 만들고 채도로 유지한다.
- **라운드 사이 (머리의 시간)**: 저채도 파스텔, 해 질 녘·가게 마감 후의 정서. 상점·장부·가도 지도는 종이 질감의 아이보리 바탕 — "장부를 넘기는 사장의 시간"이라는 서사와 합치.
- 근거: 시간 압박의 유무(§3.1)를 플레이어가 색으로 먼저 체감하게 한다. 브리핑 화면은 "라운드 사이" 톤으로 시작해 개시 탭과 동시에 웜톤으로 점화 — 개시의 손맛을 톤 전환으로 강화.

## 3. 캐릭터

### 3.1 일꾼 (최대 6명, §6)

- **컨셉**: "대장간 견습 도깨비" — 작고 둥근 2등신, 큰 손(연장을 쥐는 손이 매력 포인트), 종족적 잔혹함 없는 순한 인상. 근거: 인간 아이로 그리면 '아동 노동' 인상의 리스크가 있고, 몬스터로 그리면 귀여움 제약과 충돌 — 도깨비/코볼트풍 요정 일꾼이 안전하고 세계관(던전 경제)과 맞는다.
- **6인 즉시 구분 규칙 — 실루엣과 색의 이중 코딩**:
  1. **머리 실루엣 6종 고정**: 두건 / 뿔 하나 / 곱슬머리 / 납작모자 / 양갈래 / 대머리+수염. 흑백에서도 구분되는 것이 통과 기준.
  2. **앞치마 색 6색 고정**: 레드브라운 / 블루 / 그린 / 옐로 / 퍼플 / 화이트. 상태 색 언어(2.2)와 혼동되지 않도록 채도를 한 단 낮춘 "의류 톤"으로.
  3. 몸통·모션은 전원 공용 베이스 1종 — 변형은 머리 파츠+팔레트 스왑만. 제작비의 핵심 절약 지점이자 스킨 DLC의 구조.
- **상태별 시각 구분** (표현은 UI 문서와 협의, 아트 문법은 본 문서 소관):
  - **유휴**: 제자리에서 좌우 두리번 + 발끝 까딱. 흰 하이라이트 림 라이트. "일 주세요"의 몸짓이되 재촉하는 느낌은 금지 — 죄책감 유발은 톤 위반.
  - **작업**: 시설별 전용 모션(4.2). 망치질이 대표 — 8프레임, 타격 프레임에 스쿼시&스트레치와 불꽃 파티클. 게임 손맛의 심장이므로 전 모션 중 최고 품질로.
  - **운반**: 물건을 머리 위로 번쩍 들고 종종걸음. 규칙상 즉시 이동(§5.2)이므로 실제로는 0.3초 내외의 연출용 잔상 대시 — 규칙과 연출의 괴리를 속도로 봉합한다.
- 기각: 일꾼별 고유 바디·고유 모션 — 개성 시스템 자체가 보류(부록 B)인데 아트만 앞서갈 이유가 없고, 제작량이 6배가 된다.

### 3.2 레이드 팀 (거래처당 1팀 이상, 브리핑·결과 화면 등장)

- **컨셉**: 팀 = 3~4인 파티의 단체 스프라이트 1장 + 표정·포즈 차분. 화면 등장은 브리핑(출정 전)과 원정 리포트(귀환 후)뿐이므로 인게임 워크 사이클은 만들지 않는다.
- **제작 규칙**: 공용 바디 3종(전사형/민첩형/로브형) × 파츠·팔레트 조합으로 8팀을 생성. 팀당 신규 작화는 머리·상징 소품 정도로 억제.
- **팀 개성은 실루엣 소품 하나로**: 팀마다 "그 팀임을 알아보는 물건" 1개를 정한다(예: 초대형 방패, 낡은 깃발, 랜턴 지팡이). 근거: 소형 화면에서 얼굴보다 소품 실루엣이 먼저 읽힌다.
- **표정 원칙 (원칙 10)**: 실패 귀환도 다친 곳 없이 — 그을음·헝클어진 머리·시무룩까지만 허용. 붕대·부상·유혈·공포는 전면 금지. 실패의 최대 강도는 "소풍을 망친 얼굴"이다.

### 3.3 거래처 8사 — 마스코트·로고 컨셉 (스테이지·라운드 설계서 v0.1과 동기화 완료)

각 거래처는 **로고(장부·가도 지도용) + 팀 컬러 + 팀 상징 소품**의 3점 세트로 정의한다. 순서·명칭은 스테이지·라운드 설계서(A-2)의 확정을 따른다 (가안 대비 2·4번 교환 — 무쇠 망치단 = T2 방어구 티어, 홍염 상단 = T4 강화·화산 티어):

| # | 거래처 | 로고 모티프 | 팀 컬러 | 팀 개성 한 줄 |
|---|---|---|---|---|
| 1 | 새싹 원정대 | 새싹 돋은 투구 | 그린 | 의욕만 앞서는 신출내기들 — 튜토리얼 스테이지의 얼굴 |
| 2 | 무쇠 망치단 | 교차한 망치 | 스틸 그레이 | 방어구를 사랑하는 우직한 드워프풍 — 폐광산 전문 |
| 3 | 달빛 길드 | 초승달 랜턴 | 인디고 | 야간 던전 전문, 조용하고 신비로운 톤 |
| 4 | 홍염 상단 | 불꽃 문양 주판 | 스칼렛 | 화산 전문, 장사꾼 기질 — 강화 의뢰(T4)의 첫 큰손 |
| 5 | 바람개비 용병단 | 바람개비 깃발 | 스카이 | 속도전 전문, 인내 짧은 의뢰의 서사적 근거 |
| 6 | 서리지기 원정회 | 눈꽃 방패 | 아이스 블루 | 설산 전문, 두꺼운 방한구 실루엣 |
| 7 | 금술 협회 | 저울과 반지 | 골드 | 장신구·강화 의뢰의 큰손, 격식 있는 톤 |
| 8 | 왕립 원정단 | 왕관 문장 | 로열 퍼플 | 최종 거래처 — 최고 난도 강화 의뢰의 주인 |

- 기각: 거래처별 완전 개별 아트 세트(전용 배경·전용 NPC 군중) — 등장 빈도 대비 과투자. 개성은 로고·컬러·소품·리포트 문구로 낸다.

### 3.4 사장(플레이어)

- **화면에 등장하지 않는다**(§5.1). 다만 존재감은 남긴다 — 계약서의 **도장**, 장부의 필적, 원정 리포트 하단의 서명란. "보이지 않는 손"의 정체성을 소품으로만 구현한다. 근거: 아바타 부재를 결핍이 아니라 연출로 전환.

## 4. 공간과 시설

### 4.1 대장간 공간 컨셉

- **시점**: 3/4 준-탑다운(오버쿡드식 부감). 근거: 세로 무스크롤 한 화면에 시설 10종 이하+일꾼 6명을 겹침 없이 배치하려면 바닥 평면이 넓게 보이는 시점이 필수. 사이드뷰는 겹침, 순수 탑다운은 시설 실루엣 표현력이 죽는다 — 기각.
- **공간 서사**: "가도변의 오래된 대장간". 돌바닥 + 목조 골조 + 벽의 주문서·공구걸이. 화면 위쪽이 카운터·거리(의뢰가 오는 곳), 아래쪽이 화덕 안쪽(공정이 깊어지는 곳) — 공정의 깊이(§8)가 화면의 깊이와 일치하게. (구체 레이아웃 좌표는 UI 문서 소관.)
- **성장의 가시화**: 시설·일꾼이 늘수록 화면이 물리적으로 붐빈다(§1.3의 재미 축 2). 빈 설치 슬롯은 분필 표시된 바닥 자국으로 — "다음엔 무엇을 들일까"(§7.3)를 항상 눈에 보이게.

### 4.2 시설 4계열의 시각 언어 — 실루엣 문법

**계열은 실루엣만으로, 종은 색·소품으로 구분한다.** (계열 상한 3·3·3·1, §7.2)

| 계열 | 실루엣 문법 | 시각 모티프 | 일꾼 모션 |
|---|---|---|---|
| **변환 (3종)** | **둥근 통형** — 배가 불룩한 가마·솥 | 불이 스스로 타는 아궁이 창, 굴뚝 연기 | 없음(불이 일한다, §6) — 투입 시 뚜껑 여닫힘만 |
| **합성 (3종 상한 / 현재 2종, 1자리 공석)** | **V자/깔때기형** — 입구 둘이 하나로 모임 | 두 재료가 만나는 절구·혼합조 | 젓기/찧기 6프레임 |
| **제작 (3종)** | **수평 작업대형** — 낮고 긴 테이블 + 수직 상징물 | 무기=모루, 방어구=재봉·판금대, 장신구=세공대 | **망치질 8프레임(간판 모션)**, 재봉·세공 변형 |
| **강화 (1종)** | **수직 첨탑형** — 유일하게 세로로 높은 제단 | 자수정 크리스탈, 보라 빛기둥 | 담금질 대타격 1회(느리고 묵직하게) |

- 근거: 소형 화면에서 플레이어의 시선은 색보다 덩어리 형태를 먼저 읽는다. "둥글면 불, V면 합침, 낮으면 만들기, 높으면 정점"의 한 줄 문법으로 학습 비용을 없앤다.
- 강화 시설이 유일하게 수직으로 높은 것은 의도된 특권 — 공정의 정점(§10.2)이 화면에서도 정점이다.
- 쓰레기통·공급함·납품 카운터·인벤토리는 계열 문법 밖의 "가구"로, 채도를 낮춰 시설과 위계를 분리한다.

### 4.3 시설 상태 표현 (공용 오버레이 시스템)

상태 표현은 시설별 개별 작화가 아니라 **공용 오버레이 4종**으로 통일한다 — 제작비 절약이자 학습 일관성.

| 상태 | 표현 |
|---|---|
| 비어있음 | 기본 스프라이트, 저채도. 변환 시설은 불씨만 잔잔히 |
| 작업 중 | 스카이 블루 진행 링 + 일꾼 모션 + 계열별 소형 파티클(불꽃·김·톱밥) |
| 산출물 대기 | 시설 위 산출물 아이콘이 통통 바운스 + 골드 트윙클. 시설이 막혀 있음(§5.4)을 "물건이 자리를 차지한 그림"으로 직관화 |
| **과열 임박** | **레드 점멸 림 + 검은 연기 증가 + 벨 아이콘** — 2.2의 예약 신호. 화면 내 최우선 시인성 보장 |

- 과열 소실의 순간: 뻥 하는 연기구름 + 까맣게 탄 재 부스러기. 유쾌하지만 명확한 상실감 — "탄 토스트"의 정서. 잔혹하지 않되 아프게.

### 4.4 시설 등급업의 시각 변화

- **결정**: 실루엣 불변 + 장식 가산. 등급 2=구리 테두리·소품 추가, 등급 3=금장·전용 파티클(불티·보라 미광). 근거: 실루엣이 바뀌면 계열 문법(4.2)이 깨진다. 등급은 "같은 기계가 좋아진 것"으로 읽혀야 한다.
- 기각: 등급별 전체 리드로 — 10종×3등급=30벌 작화는 1인 개발 예산 초과이며 문법도 해친다.

## 5. 아이템 아이콘 규칙

- **분류 식별 — 배경 태그 문법**: 아이콘 셀의 바탕 형태로 분류를 고정한다.
  - 재료 = 원형 바탕 / 장비 = 방패꼴(오각) 바탕 / 부품 = 톱니형 바탕 / 강화석 = 바탕 없는 보라 크리스탈 고유형(전략 자원 1종의 특권, §8.4-4).
  - 근거: 실물 실루엣만으로는 "은괴 vs 은반지"급 혼동이 남는다. 바탕 형태가 1차 분류를, 실물 그림이 2차 식별을 맡는다.
- **장비 3분류 자세 규칙**: 무기=45° 사선, 방어구=정면 대칭, 장신구=중앙 소형+여백. 분류가 영원히 셋(§7.2)이므로 자세 문법도 영구 고정.
- **가독성 기준**: 원판 32px, 최소 표시 32px(1배)에서 식별 가능해야 통과. 디테일은 "3색 덩어리+하이라이트 1점"까지만 — 아이콘은 그림이 아니라 글자다.
- **강화 ◆핍 뱃지 (상한 +3, 화면 전체 공유 — UI §4.2와 동일 정의·상호 참조)**:
  - 위치 우상단 고정, 자수정 퍼플 바탕 위에 **◆ 핍을 단계 수만큼(1~3개) 얹는다** — 핍 개수가 1차 신호(카운트 비교·색약 대응). +3만 금테+미광 — 상한 도달의 명예.
  - 좁은 자리에서는 흰 픽셀 숫자 "+N"을 작게 병기할 수 있다(보조 신호). **핍이 주(主), 숫자가 종(從).**
  - 의뢰 카드의 요구 단계 표기와 장비 아이템의 현재 단계는 **완전히 동일한 뱃지**를 쓴다 — "장비의 현재 단계와 의뢰의 요구 단계가 즉시 식별"(A-4 / UI §4.2)을 같은 그림 문법으로 보장.
  - 기각: 별·화살표 등 별도 심볼 — 별은 판정(§3.2)에 예약된 언어다. 혼용 금지.
- **왁스 인장 마크 (대형·예고 의뢰 표식 — UI §4.2/§5가 의존)**:
  - 대형·예고 의뢰 카드와 브리핑 예고에 붙는 **봉랍(왁스 씰) 스탬프** 스프라이트. "격식 있는 큰 계약이 온다"를 한눈에 알리는 표식이다.
  - 계약 체결 연출의 왁스 실링(§6.3)과 조형 계열은 같되 용도가 다르다 — 이쪽은 카드·브리핑에 상시 붙는 작은 마크, 저쪽은 스테이지 클리어 1회의 대형 연출. 봉랍 색은 거래처 팀 컬러(§3.3)를 흡수해 "누구의 큰 의뢰인지"를 함께 전한다.
  - 색약 대응: 마크의 유무(형태)가 1차 신호이므로 색에 의존하지 않는다.

## 6. 연출 (VFX·애니메이션)

### 6.1 인게임 연출

- **과열 3막**: ①완료 벨(골드 트윙클+"띵") → ②유예 구간(레드 점멸+연기 짙어짐 — 게임 유일의 레드 점멸) → ③소실(연기 뻥+탄 재+시설이 잠시 시무룩하게 김 빠짐). 손실을 아프게, 그러나 웃으며 배우게.
- **납품 순간**: 카운터에서 장비가 상자에 포장→리본→의뢰 카드에 "납품 완료" 도장 쾅+점수 숫자가 총점 게이지로 날아가 박힌다. 근거: 점수는 카드에 적힌 값의 합(§3.2)이므로, 연출도 "카드 값이 게이지에 쌓이는 그림"으로 시스템을 그대로 시각화한다.
- **강화 연출**: 담금질 대타격 1회 → 보라 링 파동 → +N 뱃지가 그 자리에서 스탬프. **도박식 연출(룰렛·두근거림 연타) 전면 금지** — 강화에 실패가 없다는 결정론(§10)을 연출 톤부터 약속한다.

### 6.2 별 판정 (0~3성) — 실패는 귀엽게, 성공은 성대하게

| 판정 | 연출 컨셉 |
|---|---|
| 별 0 (공략 실패) | 팀이 던전 입구에서 그을음 묻은 채 터덜터덜 되돌아 나온다. 전원 무사, 막내가 뒤돌아 던전에 혀 내밀기. 화면은 짧고, "다시" 버튼이 가장 크다(§11.1) |
| ★ (신승) | 땀 닦으며 깃발 하나 꽂고 브이. 소박한 팡파레 |
| ★★ (안정 공략) | 전리품 수레를 끌고 씩씩한 귀환 행렬 |
| ★★★ (완벽 공략 — 개선 행진) | 색종이+군중 환호+팀 무등 행진. 이때 **팀원들이 든 장비가 반짝인다 — 내가 납품한 바로 그 장비다**(6.4와 연결) |

- 별은 하늘에서 쾅쾅 박히는 금별 3칸으로 통일(장부·지도에서도 동일 그림). 실패 시 회색 빈 별 3칸 — 벌이 아니라 "다음에 채울 칸"으로 보이게.

### 6.3 계약 체결

- 스테이지 클리어 시: 장부가 펼쳐지고 → 거래처 로고가 인장으로 찍히고 → 왁스 실링 위에 **사장의 도장**(3.4) 쾅 → 장부 한 페이지가 채워진다. 3성 승급(§4)은 같은 페이지에 금테가 둘리는 후속 연출. 근거: 계약의 가치가 진행+수집(§4)이므로, 연출도 "장부가 채워지는 기쁨"에 집중한다.
- 여기의 **왁스 실링(1회성 대형 연출)** 과 대형·예고 의뢰 카드의 **왁스 인장 마크(§5의 상시 표식 스프라이트)** 는 조형 계열은 같되 별개 에셋이다 — 용도·크기·등장 빈도가 다르다.

### 6.4 시그니처 연출 — "원정 리포트 화면" (용어는 UI §6.1과 공동)

**원칙 10의 무게("내 솜씨가 남의 모험을 결정한다")를 싣는 자리는 여기다.**

- **용어 정의**: 결과 화면 **전체**를 "원정 리포트 화면"이라 부르고, 그 안의 서술 텍스트는 "리포트 문장 카드"로 구분한다(UI §6.1과 동일). 화면과 그 안의 문장 카드는 다른 층위다 — 혼동 금지.
- **형식**: 원정 리포트 화면은 점수판이 아니라 **거래처가 보내온 한 장의 리포트 문서**다. 종이 질감 바탕 위에:
  1. **장착 매핑 컷** — 납품한 장비 아이콘들이 팀원 스프라이트에게 하나씩 날아가 **장착되는** 짧은 시퀀스. "내가 만든 +1 지팡이를 저 로브 입은 아이가 쥔다." 이 **약 3초(장착 → 던전 진입)** 가 이 게임의 시그니처이며, UI §6.1의 1단계와 길이·구성이 일치한다.
  2. **원정 실루엣 파노라마** — 무장을 마친 팀이 던전 입구로 걸어 들어가는 옆모습 실루엣 1컷(공용 배경+팀 팔레트, 판정별 조명 차등). 위 1과 이어지는 하나의 시그니처 시퀀스다.
  3. **판정 연출**(6.2) → **리포트 문장 카드**(UI §6.1-6): 납품 내역에서 파생되는 짧은 서술("귀사의 +1 지팡이 덕에 용암 골렘을 넘었습니다" / 실패 시 "방어구가 부족해 이번엔 물러났습니다 — 다음 납품을 기다리겠습니다"). 결정론이므로 문장도 납품 데이터의 함수다 — 거짓 드라마 금지.
  4. 하단에 점수 합산 내역과 정산 골드, 사장 서명란.
- 근거: "실패는 가볍게"는 연출 길이(실패=짧게)로, "운명은 무겁게"는 연출 밀도(장착 매핑=정성껏)로 나눠 싣는다. 무게는 잔혹함이 아니라 **인과의 가시화**에서 나온다.
- 기각: 던전 내부 전투 컷신 — 제작비 폭증에 전투 묘사는 톤 리스크. 모험은 실루엣과 문장으로 충분히 상상시킨다.

## 7. 사운드 톤 (A-5 범위 — 방향만 확정)

- **망치질 타격음이 게임의 대표음**(§5.1) — 금속성이되 통통 튀는 밝은 질감. 타격 프레임과 1프레임 오차 없이 동기화하는 것을 품질 기준으로.
- 과열 벨은 유일하게 불안한 음색(반음 하강), 완료 벨은 밝은 상승음 — 청각도 색 언어(2.2)와 같은 배타 문법.
- BGM: 라운드 안=경쾌한 워크송 풍(템포는 고정 — 기한 임박 가속 연출은 밸런스 문서와 협의 전 보류), 라운드 사이=어쿠스틱 소품. 전 트랙 루프 1~2분의 소편성으로 외주·라이브러리 조달.

### 8.2 제작 도구

- 스프라이트·애니메이션: **Aseprite** (팔레트 스왑·태그 애니메이션 파이프라인의 표준).
- 팔레트 관리: Lospec 기반 커스텀 마스터 팔레트 1파일을 단일 출처로.
- 폰트: 한글 픽셀 폰트는 **Galmuri 등 SIL OFL 라이선스 폰트** 채용(상용 유료 게임 사용 가능 여부를 채용 전 라이선스 재확인).
- VFX: 엔진 내장 파티클 + 스프라이트 시트 혼용.

### 8.3 조달 전략 — 자체 / 외주 / 에셋스토어

| 구분 | 대상 | 이유 |
|---|---|---|
| **자체 제작 (핵심)** | 일꾼 전 모션, 시설 10종, 상태·과열 표현, 아이템 아이콘, 원정 리포트 연출 | 게임의 정체성이자 반복 수정이 가장 잦은 곳 — 외주 왕복이 오히려 비싸다 |
