## 1. 목적과 범위

이 문서는 MVP에서 플레이 결과가 골드로 바뀌고, 골드가 강화 설비 세트 구매로 이어지는 최소 경제 루프를 정의한다.

MVP 경제가 검증할 것은 하나다.

> 라운드 플레이 → 결과 보상 → 영구 설비 구매 → 더 깊은 라운드 진입의 연결이 이해되고 만족스러운가?

## 2. 명시적 비범위

- 가격, 결제, 광고, 체험판, IAP 등 상업 모델
- 일꾼 고용, 시설 증설·등급업, 도구, 설치 슬롯, 인벤토리 확장
- 복수 재화, 소모성 재화, 재화 교환
- 반복 수입, 방치 보상, 계약 정기 수입
- 골드·가격의 실제 숫자: `BAL-MVP-001`
- 상점 화면과 구매 피드백: `UX-MVP-001`, `UI-MVP-001`

## 3. 재화

MVP 재화는 `GOLD` 한 종류뿐이다.

- 정수이며 음수가 될 수 없다.
- 라운드 사이에 영구 유지된다.
- 라운드 플레이 결과 외의 획득 경로는 없다.
- 구매 외의 감소 경로는 없다.
- 현금 또는 외부 재화와 연결되지 않는다.

## 4. 보상 구조

### 4.1 완료 보상

- 제한 시간 종료 후 `SYS-PROG-001`이 확정한 별 판정에 따라 골드를 지급한다.
- 별이 높을수록 완료 보상이 커야 한다.
- 같은 별 결과의 재플레이는 같은 골드를 지급한다.
- 공략 실패, 재시작, 중도 이탈에는 골드를 지급하지 않는다.

### 4.2 최초 클리어 보너스

- 라운드를 처음 별 1개 이상으로 완료할 때 한 번만 추가 지급한다.
- 이후 더 높은 별을 얻어도 최초 클리어 보너스를 다시 지급하지 않는다.
- 보너스 지급 여부와 최고 별은 서로 독립된 영속 상태다.

### 4.3 정산 순서

1. 진행 시스템이 별과 최초 클리어 여부를 확정한다.
2. 완료 보상과 최초 클리어 보너스를 계산한다.
3. 골드 잔액을 증가시킨다.
4. 기록·해금·골드 변경을 하나의 영속 처리로 저장한다.
5. 저장 성공 결과를 받은 뒤 사용자에게 최종 정산을 확정 표시한다.

## 5. 강화 설비 세트

MVP 상점 상품은 `SHOP_ENHANCEMENT_KIT` 하나뿐이다.

| 항목 | 규칙 |
|---|---|
| 구성 | 영구 진행 능력 `enhancement_capability_owned` |
| 노출 조건 | R4 최초 클리어 |
| 구매 조건 | 미구매 상태이며 보유 골드가 가격 이상 |
| 구매 결과 | 가격만큼 골드 차감, 능력 플래그 활성화, R5 입장 조건 충족 |
| 반복 구매 | 불가 |
| 환불·판매 | MVP에서 불가 |

능력 플래그는 하나의 상품으로 원자적으로 구매한다. 부분 활성화된 중간 상태는 존재할 수 없다.

## 6. 진행 보장 제약

밸런스 값은 다음 부등식을 반드시 만족해야 한다.

```text
R1~R4를 각각 별 1개로 최초 클리어했을 때의 누적 최소 골드
>= SHOP_ENHANCEMENT_KIT 가격
```

따라서 플레이어는 별 2개·3개 획득이나 반복 파밍 없이도 R5 필수 설비를 살 수 있다. 더 좋은 결과는 구매 시점을 앞당기는 대신 별도 성능 우위를 만들지 않는다. 상품 노출은 R4 최초 클리어 뒤이므로 실제 구매와 R5 해금은 동일 구간에서 일어난다.

## 7. 구매 처리와 예외

- 골드 부족 시 구매를 거절하고 부족량을 반환한다.
- 이미 구매한 상품의 재구매 요청은 상태 변화 없이 `already_owned`를 반환한다.
- 노출 전 구매 요청은 `not_available`을 반환한다.
- 골드 차감과 능력 플래그 변경은 하나의 원자적 영속 처리여야 한다.
- 저장 실패 시 골드와 능력 플래그 모두 구매 전 상태로 남아야 한다.
- 구매가 성공하면 R5 입장 조건을 즉시 다시 평가한다.

## 8. 시스템 인터페이스

| 대상 | 이 문서가 제공하는 것 | 이 문서가 받는 것 |
|---|---|---|
| `SYS-PROG-001` | 강화 능력 소유 여부와 정산 완료 | 별, 최초 클리어 여부, R4 클리어 상태 |
| `BAL-MVP-001` | 보상·가격 파라미터의 의미와 제약 | 별 보상, 보너스, 상품 가격 |
| `UI-MVP-001` | 골드 잔액, 상품 상태, 구매 거절 사유 | 구매 의도 |
| `TECH-DATA-001` | 원자적으로 저장해야 할 경제 변경 | 멱등 트랜잭션과 영속 처리 성공·실패 |
| `QA-MVP-001` | 진행 보장 부등식과 예외 계약 | 회귀 검증 결과 |

## 9. 미결정 사항

| 질문 | MVP 기본값 | 결정 방법 |
|---|---|---|
| 실패에 소액 골드를 줄 것인가? | 지급하지 않는다. | 실패 반복이 재도전 의지를 현저히 낮출 때만 재검토한다. |
| R5 완료 후 남은 골드를 표시할 가치가 있는가? | 표시하되 추가 용처는 만들지 않는다. | MVP 테스트에서 혼란 여부 확인 |

## 10. 검증 방법

- R1~R4를 모두 별 1개로 최초 완료한 저장 상태에서 강화 설비 세트를 즉시 살 수 있어야 한다.
- 같은 결과 재플레이의 지급액이 달라지지 않아야 한다.
- 최초 클리어 보너스는 라운드별 정확히 한 번만 지급되어야 한다.
- 저장 실패를 주입해도 골드만 차감되거나 능력만 생기는 부분 구매가 없어야 한다.
- 경제 경로 어디에도 외부 결제나 시간 대기 조건이 없어야 한다.

## 11. 완료 기준

- 골드의 모든 발생·소비 경로가 열거되어 있다.
- 강화 설비 구매 전후의 상태와 오류가 정의되어 있다.
- 별 1개 진행만으로 R5 진입이 보장된다.
- 실제 숫자는 `BAL-MVP-001`에만 존재한다.
- MVP 범위에 없는 성장 축이나 상업 모델을 포함하지 않는다.
