## 0. 문서 안내

### 0.1 이 문서의 성격

이 문서는 게임의 **코어 시스템 명세**다. 게임을 게임이게 하는 규칙, 시스템, 구조, 설계 원칙만을 다루며, 배경지식 없는 독자가 이 문서만으로 게임의 작동 방식을 완전히 이해하는 것을 목표로 한다.

**다루지 않는 것** — 아래는 각각의 하위 문서와 담당 작업으로 위임된다 (범위와 제약은 부록 A):

| 위임 영역 | 하위 문서 |
|----------|----------|
| 재료·장비·레시피의 목록과 해금 배치 | 아이템·레시피 설계서 |
| 거래처 설정, 라운드 구성, 던전 테마, 라운드 특수 규칙 | 스테이지·라운드 설계서 |
| 모든 수치 — 시간, 배율, 가격, 커트라인, 수입 | 밸런스·경제 수치서 |
| 화면 구성, 레이아웃, 연출 | UI·화면 설계서 |
| 그래픽 테마, 캐릭터, 톤 | 아트·테마 가이드 |

본 문서에 구체적 명칭이나 숫자가 등장하면 그것은 **설명용 예시**이며 확정값이 아니다. 확정은 해당 하위 문서의 몫이다. (예외: 본 문서가 "구조적 확정"으로 명시한 값 — 계열 상한(§7.2), 일꾼 수 범위(§6) — 은 코어 명세의 일부다.)

## 1. 게임 소개

### 1.1 이 게임은 무엇인가

**던전 오피스**는 던전 레이드 회사들을 거래처로 둔 대장간의 백오피스를 운영하는 1인용 모바일 게임이다.

이 세계에는 던전 레이드로 수입을 올리는 회사들이 있다. 던전을 공략하는 것은 그들의 레이드 팀이고, 플레이어는 그들을 무장시키는 납품 업체 — 대장간의 사장이다. 매 라운드, 거래처의 레이드 팀이 출정 준비 의뢰를 들고 온다. 납품기한 — 곧 팀의 출정 시각 — 까지 일꾼들에게 탭으로 지시를 내려 장비를 만들어 납품하고, **팀의 공략 결과는 그 장비에 달린다.** 허술하게 보내면 공략 실패, 완벽하게 갖춰 보내면 별 3개짜리 개선 행진이다.

각 스테이지는 거래처 회사 하나다. 그 거래처의 라운드들을 모두 클리어하면 스테이지 클리어 — **정식 계약이 체결되어 장부에 오른다.** 플레이어의 장기 목표는 가도의 모든 회사를 거래처 장부에 올리는 것이다.

조작하는 캐릭터는 없다. 플레이어는 사장이고, 화면 속에서 뛰는 것은 일꾼들이다. 일꾼 수가 곧 동시에 굴릴 수 있는 일의 수이므로, 실력은 손놀림이 아니라 **일의 순서를 짜는 머리**에서 나온다. 이 게임에 주사위는 없다 — 같은 판단은 언제나 같은 결과를 낳고, 게임의 절정은 **강화 의뢰**: 가장 깊은 공정을 요구하는 묵직한 의뢰를 납품기한 안에 완주해내는 순간이다.

| 항목 | 내용 |
|------|------|
| 플랫폼 | 모바일, 세로 화면, 한 손 조작 |
| 플레이 | 싱글 플레이, 완전 오프라인 지원 |
| 세션 | 라운드 1판 = 수 분, 언제든 일시정지·중단 저장 |
| 진행 구조 | 스테이지(거래처) > 라운드(레벨) — 스토리 모드형 스테이지제 |
| 수익 모델 | **유료 선불(프리미엄)** — 광고 없음, 성능 판매 없음 (§11) |

### 1.2 플레이 장면 미리보기

*등장하는 장비·수치는 설명용 예시다.*

> 브리핑 — *"거래처 '홍염 상단' 3팀 / 목표: 화산 동굴 / 출정까지 150초. 의뢰 경향: 방어구 다수, 종반에 +1 지팡이 대형 의뢰 예정."* 시간이 멈춘 화면 위에서 계획이 선다: 마정석 라인을 미리 돌려 강화석을 준비해 두자. 개시를 탭.
>
> 광석은 용광로에 — 불은 일꾼 없이도 탄다. 일꾼 1호는 모루에, 2호는 합성대에서 강화석. 완성된 강화석은 원탭으로 인벤토리에 넣어 시설을 비운다. 철갑옷 납품 — 점수가 쌓인다. 저가 의뢰 하나가 게이지를 다해 떠나지만, 잃은 것은 그 카드의 점수뿐이다. 버린 것이 아니라 고른 것이다.
>
> 예고된 대형 의뢰 도착: **"+1 마법 지팡이 · 56점."** 지팡이 제작, 그리고 강화 패스 — 실패는 없다. 모자랄 수 있는 것은 오직 시간뿐. 출정 3초 전, 납품.
>
> 결과: **화산 동굴 — 공략 성공! ★★.** *"강화석을 하나 더 준비했다면 +1 반지도 냈을 텐데."* — 한 판 더.

### 1.3 재미의 세 축

1. **일손 굴리기 (라운드 안)** — 일꾼들과 불로 여러 타이머를 저글링하는 스케줄링의 쾌감. 과열 직전의 회수, 무엇을 만들고 무엇을 흘려보낼지의 트리아지.
2. **거래처 키우기 (라운드 사이)** — 골드로 일꾼과 설비를 늘려 붐벼가는 대장간, 계약을 하나씩 따내며 채워지는 장부와 원정 기록.
3. **마감의 드라마 (매 출정의 하이라이트)** — 예고된 대형 강화 의뢰라는 미니 프로젝트를 출정 전에 완주하는 클러치. 심장을 뛰게 하는 것은 주사위가 아니라 마감이다.

## 2. 핵심 용어

### 진행과 거래
| 용어 | 정의 |
|------|------|
| **거래처** | 던전 레이드로 수입을 올리는 회사. 스테이지 하나 = 거래처 하나 |
| **스테이지** | 한 거래처와의 거래 과정. 다수의 라운드로 구성되며, 전 라운드 클리어 = 스테이지 클리어 = **계약 체결** |
| **라운드** | 플레이의 기본 단위인 레벨. 거래처 레이드 팀 하나의 "출정 준비" |
| **계약** | 완전 클리어한 거래처와의 정식 납품 계약. 장부에 등록된다 — 부가 경제 효과는 당분간 없다 (§4) |
| **브리핑** | 라운드 시작 전, 시간이 멈춘 상태로 의뢰 정보를 확인하는 화면. 개시는 플레이어의 탭 |
| **납품기한** | 라운드의 제한 시간. 기한이 도래하면 납품 점수가 **즉시** 합산·평가된다. 서사적으로는 레이드 팀의 출정 시각 — 팀은 그때까지 납품된 장비를 들고 떠난다 |
| **공략 결과 / 별** | 라운드 총점의 서사적 판정 — 공략 실패(별 0) 또는 별 1~3. 별은 정산 골드의 등급(§9.2)이자 다음 거래처 개방의 열쇠(§3.1)다. 커트라인 수치는 밸런스 문서 |
| **장부(도감)** | 거래처·레이드 팀의 계약 현황과 원정 기록 컬렉션 |

### 사람과 공간
| 용어 | 정의 |
|------|------|
| **일꾼** | 실제로 일하는 캐릭터. **작업 하나가 일꾼 하나를 점유** — 일꾼 수 = 동시 작업 수의 상한. 고용은 영구. 시작 2명, 상한 6명 (§6) |
| **대장간** | 작업장. 일꾼·시설·시설 수·시설 등급이 영구 유지되는, 성장이 쌓이는 곳 |
| **시설(스테이션)** | 대장간의 기능 지점. 변환·합성·제작·강화 4계열, 종류 수에 영구 상한 (§7.2) |
| **인벤토리** | 라운드 안의 공용 보관함. **무엇이든 넣을 수 있다.** 용량 유한, 라운드 종료 시 소멸 |
| **쓰레기통** | 기본 제공 시설. 아이템을 자발적으로 파기하는 유일한 수단 (운반과 같은 탭-탭, 즉시·일꾼 불필요) |

### 물건과 행위
| 용어 | 정의 |
|------|------|
| **재료 / 장비 / 부품** | 원료(원자재·가공재) / 납품 대상 완성물(무기·방어구·장신구) / 조립용 중간 제품(특수 의뢰 전용) |
| **제련** | 재료로 새 재료를 만들기. **변환**(1→1)과 **합성**(2→1) 두 규칙뿐 |

## 3. 진행 구조

### 3.1 구조 한눈에

```
가도 지도
 └─ 스테이지 = 거래처 회사 하나
     └─ 라운드 1 … N          ← 각각이 별 0~3개짜리 레벨, 순차 해금
         └─ 마지막 라운드 클리어 = 스테이지 클리어 = 계약 체결
             └─ 누적 별 요건 충족 시 → 다음 거래처 개방
```

- **순차 해금**: 다음 라운드는 이전 라운드의 클리어(★1 이상)로 열린다. 따라서 마지막 라운드의 클리어가 곧 전 라운드 클리어이며, 스테이지 클리어다.
- **다음 거래처의 개방 조건은 스테이지 클리어 + 누적 별 최소 요건이다.** 누적 별은 가도 전체의 라운드별 최고 기록(§3.2)의 합산으로, 부족하면 어느 스테이지의 어느 라운드를 다시 도전해도 좋다 — 막힌 곳이 아니라 잘하는 곳에서 벌 수 있다. 요건 수치는 밸런스 문서의 몫이되, **3성 완주를 강제하지 않는 수준**이어야 한다 (A-3 제약). 별은 게이트(진행)이자 수입(§9.2)으로, 재도전의 표준 동기다.
- 라운드 수, 개방 요건 수치, 해금 곡선은 스테이지·라운드 설계서와 밸런스 문서의 몫이다. 본 문서가 정하는 것은 구조뿐이다: **순차 해금 → 전 라운드 클리어 → 계약, 그리고 누적 별 게이트.**
- **입장 요건**: 라운드에 등장할 의뢰 중 현재 설비로 이행 불가능한 것이 하나라도 있으면, 그 라운드는 **시작할 수 없다.** 부족한 설비는 입장 화면에 표시된다. 시설 구매가 새 라운드를 여는 열쇠이며, 그 덕에 라운드 안의 모든 의뢰는 정의상 이행 가능하다 — 죽은 카드는 존재하지 않는다.
- 라운드 사이에는 시간 제한 없는 영구 상점(§7.3)과 장부가 있다. 라운드 안은 손의 시간, 라운드 사이는 머리의 시간.

### 3.2 라운드의 흐름

1. **브리핑**: 시간이 멈춘 채 의뢰서를 확인한다 — 의뢰인(레이드 팀), 목표 던전, 납품기한, 의뢰 경향(대형·강화 의뢰의 예고 포함), 이 라운드의 특수 규칙. 계획은 공짜고, 실행만 시간을 쓴다. 개시는 플레이어의 탭. **재도전 시 브리핑은 건너뛸 수 있다.**
2. **영업**: 납품기한까지 의뢰가 순차적으로 밀려온다. 만들어 납품한 만큼이 팀의 무장이 된다.
3. **기한과 판정**: 납품기한이 도래하면 팀은 그때까지 납품된 장비를 들고 떠나고, 총점이 **즉시** 합산·평가되어 공략 결과로 번역된다:

| 총점 | 공략 결과 | 별 |
|------|------|-----|
| 실패선 미만 | **공략 실패** — 팀은 아쉬운 얼굴로 후퇴한다 (다치지 않는다, 톤은 가볍게) | 0 |
| 커트라인 1 | 신승 | ★ |
| 커트라인 2 | 안정 공략 | ★★ |
| 커트라인 3 | 완벽 공략 — 개선 행진 | ★★★ |

- **판정은 순수하게 점수로만 한다.** 총점은 납품한 의뢰 카드들에 적힌 값의 합이다 — 그 외의 배율이나 보너스는 없다. (라운드의 특수 규칙이 점수의 산출 방식을 변형하는 것은 허용 — 스테이지 설계서 재량. 브리핑에서 반드시 고지된다.)
- **별은 라운드별 최고 기록으로 저장된다.** 재도전의 결과가 더 낮아도 기록은 내려가지 않는다. 게이트(§3.1)·계약 승급(§4)은 전부 이 기록 위에서 계산된다. 정산 골드(§9.2)만은 그 판의 결과를 따른다.
- 라운드는 중간에 실패로 끊기지 않으며, **평가는 오직 납품기한 도래 시에만 이루어진다.** 조기 정산 수단은 없다 — 마감까지 한 개라도 더 만드는 것이 항상 유효하다. 라운드를 포기하고 싶다면 일시정지 메뉴의 재시작을 쓴다 (평가 없이 처음으로 되돌아가며, 기록에 남지 않는다).
- 별점의 힘은 서사에 있다: 성적표가 아니라 **내 손이 결정한 남의 운명**이다.

### 3.3 일시정지와 세션

- **일시정지는 언제든, 조건 없이.** 시간·타이머·진행이 전부 멈추고, 재개하면 그 순간부터. 앱 이탈 시 자동 정지·저장.
- **일시정지 중 화면 관찰과 계획은 허용이다.** 정지하고 판을 읽는 것은 치팅이 아니라 플레이 스타일이다 (오버쿡드 등 선례). 이 게임의 실력은 순간 감시가 아니라 **무엇을 언제 시키는가의 판단**에 있으므로, 정지-계획은 실력의 우회가 아니라 행사다. 단, 정지 중 조작(탭 지시)은 불가능하다 — 실행은 언제나 흐르는 시간 위에서만 한다.
- 라운드는 한 세션 안에 완결되는 길이로 설계한다 (목표 길이는 밸런스 문서). 에너지·입장권 게이트는 없다.

## 4. 계약

스테이지를 완전히 클리어하면 그 거래처와의 관계가 "공사 중"에서 "계약"으로 바뀐다.

- **계약 체결**: 스테이지의 전 라운드 클리어(= 마지막 라운드 클리어) = 스테이지 클리어 = 계약 체결. 거래처가 장부에 정식 등록되고, 누적 별 요건을 충족했다면 다음 거래처가 열린다 (§3.1).
- **계약 승급**: 그 스테이지의 전 라운드를 3성으로 달성하면 장부에 승급 표시(예: 금테)가 붙는다. 완성욕에 대한 명예 보상이다.
- **부가 경제 효과는 당분간 없다.** 계약에서 정기 수입이 발생하는 시스템은 검토 후 보류되었다 (부록 B) — 도입하더라도 원칙 11("방치 생산은 없다") 아래에서만 한다. 현재 계약의 가치는 **진행**(다음 거래처의 개방)과 **수집**(장부)이다.

## 5. 조작과 인벤토리

### 5.1 아바타 없음, 사장의 손가락

조종하는 캐릭터는 없다. 화면 속에서 뛰는 것은 일꾼들이고, 플레이어는 탭으로 지시하는 사장이다. 모든 조작은 탭이다 — 홀드도, 연타도, 타이밍 판정도 없다. 손맛은 제스처가 아니라 피드백(일꾼의 망치질 애니메이션·소리)에 담는다.

### 5.2 조작 문법

| 조작 | 방법 | 규칙 |
|------|------|------|
| **운반** | 출발지 탭 → 목적지 탭 | 재료·장비의 모든 이동. 즉시 처리, 개수 제한 없음 — 연출상 일꾼이 나르지만 규칙상 시간 비용은 없다 |
| **착수** | 시설 탭 | 재료가 놓인 시설을 탭하면 유휴 일꾼이 배정되어 작업 시작. 이후 진행은 자동 |
| **수납** | 완성 산출물 탭 (원탭) | 산출물이 인벤토리로 들어가고 시설이 비워진다 |
| **일시정지** | 정지 버튼 | §3.3 |

### 5.3 인벤토리

- 라운드 안의 공용 보관함. **무엇이든 넣을 수 있다** — 원자재부터 부품·강화석·완성 장비까지. 시설 위의 산출물은 원탭으로 수납되어 시설이 비워지고, 꺼내 쓰기는 탭-탭이다.
- **존재 이유**: 산출물이 시설을 막는 병목(§5.4)과 "미리 만들어 두기"라는 계획 플레이를 양립시키는 완충 공간이다. 브리핑(예고) — 인벤토리(비축) — 결정론(§10)의 삼각이 **"예측하고 준비하는" 플레이를 공식화**한다.
- **용량은 유한**하고, 확장은 구매 축이다 (수치·가격은 밸런스 문서). 무한이면 병목 압박이 소멸하므로 유한성 자체가 시스템이다.
- **라운드 종료 시 소멸**한다. 라운드 간 이월은 없다 — 매 라운드는 빈손에서 시작하는 공평한 스프린트다.

### 5.4 주의력의 경제

플레이어의 눈은 항상 세 개의 시계를 오간다.

- **용광로(변환)**: 완료 벨 후 짧은 과열 유예 — 유일하게 "돌아와야 하는" 시설. 긴장의 제1원천.
- **시설 출구**: 산출물을 치우기 전까지 그 시설은 막혀 있다. 방치의 벌은 소실이 아니라 병목.
- **의뢰 게이지**: 인내가 마르면 의뢰가 철회된다 — 그 점수를 벌 기회가 사라진다.

난이도는 이 세 시계의 속도와 일꾼 대비 일감의 비율로 조절한다 (값은 밸런스 문서).

## 6. 일꾼 — 동시성의 화폐

이 게임의 핵심 자원은 시간이 아니라 **일손**이다.

- **작업 하나 = 일꾼 하나.** 합성·제작·강화는 시작 시 유휴 일꾼 1명을 배정받아 작업 시간 동안 점유하고, 끝나면 풀려난다.
- **불은 일꾼이 아니다.** 용광로(변환)는 재료를 넣으면 혼자 탄다 — 일꾼을 쓰지 않는다. 대신 과열이 있다. 일꾼-시간이 드는 작업(동기)과 불-시간만 드는 작업(비동기)의 교차가 스케줄링 깊이의 원천이다.
- 동시 작업 수 = **min(유휴 일꾼, 해당 시설의 빈자리)**. "다음 일꾼이냐, 다음 시설이냐"는 서로 다른 병목을 푸는 별개의 구매 선택이다.
- **다인 작업**: 작업의 일꾼 요구 수는 1이 기본이나, 특수 작업(예: 부품의 최종 조립)은 2명 이상을 요구할 수 있다 — 스테이지 설계서가 쓸 수 있는 규칙 노브다. 단, 시작 인원이 2명이므로 **다인 작업 의뢰는 일꾼 3명 이상을 전제하는 구간부터** 배치한다 (2명 구간의 2인 작업은 대장간 전체를 세운다 — A-2 제약).
- **대기열은 없다.** 일꾼이 전부 바쁠 때의 착수 탭은 거절된다. 예약이 아니라 **지금 시킬 수 있는 일을 고르는 것**이 이 게임의 판단이다.
- **진행 중인 작업은 취소할 수 없다.** 착수가 곧 확정이다 — 대기열 없음과 짝을 이루어, 판단의 무게를 착수의 순간에 싣는다.
- 유휴 일꾼 수는 항상 표시된다 (표현은 UI 문서).
- 일꾼은 골드로 고용하며 영구히 유지된다. **시작 2명, 상한 6명 — 계열 상한제(§7.2)와 같은 급의 구조적 확정이다.** 동시성 2~6이 이 게임이 감당하는 복잡도의 전 범위이며, UI·콘텐츠·밸런스는 이 범위에 커밋한다 (고용 가격과 증가 곡선은 밸런스 문서). v1.9의 일꾼은 수만 다르다 — 개성·전문화는 보류 (부록 B).

## 7. 대장간

### 7.1 시설 구성

- **기본 제공**: 재료 공급함(원자재 무한), 변환 시설 1, 무기 제작대 1, 납품 카운터, **쓰레기통**(파기 — 운반과 같은 탭-탭, 즉시·일꾼 불필요), 설치 슬롯 약간.
- **골드로 사는 것**: 나머지 시설의 신규 설치, 같은 시설의 추가 설치(수), 시설 레벨업(등급), 설치 슬롯 확장, 인벤토리 확장, 도구(패시브), 일꾼 고용.
- 어떤 시설을 언제 여는가(해금 배치)는 아이템·스테이지 설계서의 몫이다. 본 문서가 정하는 것은 구매 축의 종류뿐이다.

### 7.2 계열 상한제

모든 시설은 네 계열에 속하고, 각 계열의 종류 수에는 **영구 상한**이 있다.

| 계열 | 규칙 | 상한 |
|------|------|------|
| 변환 | 재료 1 → 재료 1. 불이 일하고(일꾼 불필요) 과열이 있다 | 최대 3종 |
| 합성 | 재료 2 → 재료 1. 일꾼 점유 | 최대 3종 |
| 제작 | 재료 → 장비. 일꾼 점유. 장비 분류(무기·방어구·장신구)별 전용 제작대 | 최대 3종 — **확정** |
| 강화 | 장비 +1 단계. 결정론. 일꾼 점유 | **단 1종** |

- 게임의 시설 분류학은 영원히 10종 이내다. 콘텐츠가 늘어도 배울 "새 기계"는 없다 — 늘어나는 것은 재료·레시피·의뢰(데이터)뿐이다.
- 제작 상한이 이미 차 있다는 것은 **장비 분류가 영원히 셋**이라는 커밋이다.
- 강화가 단일 시설인 이유: 강화는 공정의 정점이며, 정점으로 가는 문은 하나여야 배우기 쉽다.

### 7.3 영속성과 상점

- **영구 유지되는 것**: 일꾼의 수, 시설의 종류, 시설의 수, 시설의 등급, 도구, 골드, 별(라운드별 최고 기록), 계약, 장부.
- **라운드마다 리셋되는 것**: 재료, 장비, 강화석, 인벤토리, 의뢰, 점수.
- 상점은 라운드 사이 언제든, 시간 제한 없이 연다. 솔로 게임의 대장간은 끝내 전부 모으는 컬렉션이며, 성장의 재미는 "무엇을 포기할까"가 아니라 **"다음엔 무엇을 들일까"와 "어떤 순서가 최적일까"**다.
- 시설은 구매 시 설치 위치를 고르고 언제든 무료로 재배치한다. 기능 차이 없는 편의·꾸미기의 층이다.

## 8. 제작 문법 — 세 가지 동사

모든 생산은 **제련 → 제작 → (강화) → 납품**의 파이프라인이다.

```
공급함 ──▶ [제련: 변환·합성] ──▶ [제작] ──▶ [강화 (의뢰 요구 시)] ──▶ [납품]
원자재        가공재 생성          장비 생성      단계 상승             점수
```

### 8.1 두 개의 생산 규칙

재료 생성 규칙은 아래 둘뿐이며, 콘텐츠가 아무리 늘어도 이 위에서만 늘어난다.

- **변환**: 재료 1개 → (변환 시설) → 다른 재료 1개. 자동 진행 + 과열.
- **합성**: 재료 2개 → (합성 시설) → 새 재료 1개. 일꾼 점유.

### 8.2 제작과 부품

- 제작은 장비 분류에 대응하는 제작대에서 한다.
- **이행 가능성은 입장에서 보장된다**: 라운드는 이행 불가능한 의뢰를 품은 채 시작될 수 없으므로(§3.1 입장 요건), 라운드 안의 모든 의뢰는 정의상 이행 가능하다. 어떤 설비가 어떤 의뢰를 여는지는 라운드의 입장 요건으로 표현된다.
- **부품과 조립**: 제작의 산출물은 **완제품(장비)과 부품** 두 가지다. 여러 부품을 각각 제작한 뒤 최종 조립해 완성하는 대형 장비가 존재할 수 있으며, **조립 역시 제작이다** — 부품을 재료로 삼아 해당 분류의 제작대에서 수행하고, 다인 작업(§6)의 대표 용례다. 통상의 제작은 1단계로 끝나며, 부품·조립은 특수 의뢰 전용이다 (배치는 스테이지 설계서 재량).

### 8.3 손실의 원칙

- 레시피에 없는 조합은 시설이 받지 않는다 (아무 일도 일어나지 않음). 잘못 만든 물건은 존재하지 않는다.
- **쓰레기통은 정리의 도구다.** 과잉 생산·빗나간 투기로 자리를 차지한 아이템은 쓰레기통으로 자발적으로 파기한다 — 이것은 손실이 아니라 선택이다.
- **비자발적 손실의 출처는 과열 하나뿐이다.** 강화에 실패란 없고(§10), 의뢰 철회는 기회의 상실이지 물건의 상실이 아니다. 억울한 손실이 없어야 실패가 분노가 아니라 학습이 된다.

### 8.4 아이템 설계서에 부과하는 제약

구체적 재료·장비·레시피는 아이템·레시피 설계서의 몫이되, 다음을 지켜야 한다:

1. 생산 규칙은 변환(1→1)과 합성(2→1)만 사용한다.
2. 장비 분류는 무기·방어구·장신구 셋으로 고정한다.
3. 장비의 기본 점수는 대략 "투입 노동 시간 × 공정 깊이"에 비례시킨다.
4. **용처가 여럿인 전략 자원을 정확히 1종** 둔다 (강화석 연료·상위 합성·고가 장비가 한 재료를 두고 경쟁하는 구조). 나머지 재료는 용처가 단순한 벌크로 남겨, 판단 피로를 한곳에 모은다.
5. 기본 제공 제작대(무기) 라인만으로 저가부터 고가까지의 점수 사다리가 완성되어야 한다 — 구매는 시장 확장이지 통행료가 아니다.
6. **강화석 레시피와 강화 시설의 해금은 동기화한다** (한쪽만 열린 상태 금지 — 스테이지 설계서와 공동 책임).

## 9. 의뢰와 판정

### 9.1 밀려오는 의뢰

- 의뢰는 정해진 칸에 **순차적으로** 도착한다. 고르거나 거절할 수 없다 — 다만 무엇을 만들고 무엇을 흘려보낼지는 전적으로 플레이어의 선택이다.
- 의뢰 카드: 요청 장비, **요구 강화 단계 (+0이 기본)**, 점수, 인내 게이지.
- **의뢰의 점수는 카드에 적힌 값이다.** 기본은 장비 기본점이며, 강화 요구 의뢰는 추가 공정에 비례한 할증이 붙는다. 특별 의뢰(단골 재방문, 이벤트 등)는 그 위에 추가 할증을 실을 수 있다 — 스테이지 설계 재량이며, 이 조항이 단골·이벤트 콘텐츠의 시스템적 근거다.
- 납품 카운터에 장비를 올리면 **같은 장비이고 요구 단계가 장비의 단계 이하인 의뢰** 중 최고점과 자동 매칭된다. 초과 강화분에 대한 추가 보상은 없다 — 투기적 선강화의 대가는 스스로 진다 (물건을 잃는 것은 아니므로 손실 원칙과 충돌하지 않는다).
- 인내가 다한 의뢰는 철회되고 새 의뢰가 온다. **철회의 대가는 그 점수를 벌 기회의 상실뿐이다** — 별도 감점도, 그 밖의 벌칙도 없다. 의뢰를 버리는 것은 처벌이 아니라 전략이다.
- 의뢰의 등장 속도·구성·던전 테마에 따른 비중은 라운드 설계의 제1 노브다 (스테이지 설계서 + 밸런스 문서). 이행 불가능한 의뢰는 입장 요건(§3.1) 덕에 애초에 등장하지 않는다. 공정이 깊은 의뢰(강화·부품)일수록 인내와 예고가 그 깊이를 감당할 수 있어야 한다 — 브리핑 예고와 긴 인내는 대형 의뢰의 의무 사양이다.

### 9.2 판정과 정산

- 판정은 §3.2가 전부다: 납품기한 도래 시점의 총점이 **즉시** 판정된다 — 공략 실패 / ★1 / ★2 / ★3. **순수하게 점수로만, 점수는 카드 값의 합으로만.**
- **라운드 정산은 골드를 지급하며, 액수는 그 판의 공략 결과(별 등급)에 따라 차등이다.** 별이 높을수록 골드가 크다 — 별은 게이트(§3.1)이자 수입이다. **같은 결과의 재플레이에는 동일하게 지급한다 — 파밍을 벌하지 않는다.** 등급별 액수·산식·첫 클리어 보너스·공략 실패 시 지급 여부는 밸런스 문서.
- **설계 노트 — 콤보의 삭제**: 납품 리듬에 따른 팁 배율(콤보)은 v1.9에서 삭제되었다 (부록 B). 병렬 운영의 동기는 점수 합산 그 자체다 — 대형 라인이 도는 동안 노는 일꾼은 벌지 못한 점수이고, 흘려보낸 저가 의뢰는 커트라인에서 모자란 점수다. 이 동기가 배율 없이도 충분히 작동하는지는 부록 C-3의 검증 질문이다.

## 10. 강화 — 가장 깊은 공정

### 10.1 규칙

완성 장비를 강화 시설에 올리고 강화석 1개를 소모하면, 장비의 단계가 **+1 오른다. 그게 전부다** — 확률도, 실패도, 파괴도 없다. 일꾼 1명을 점유하는 보통의 작업일 뿐이다.

- 강화 1회(패스) = 강화석 1개 + 일꾼 작업 1회 = 단계 +1. 상위 단계는 패스를 거듭해 도달한다 (+2 = 패스 2회 + 강화석 2개).
- 단계 상한은 소수로 유지한다 (값은 밸런스 문서).
- **강화가 존재하는 이유는 의뢰가 요구하기 때문이다.** "+2 마강검" 의뢰는 강화석 2개의 제련 라인과 강화 패스 2회가 걸린 미니 프로젝트이며, 그 노동만큼 높은 점수를 건다 (원칙 5).

### 10.2 설계 의도

- 강화는 도박이 아니라 **파이프라인의 가장 깊은 층**이다. 저가 의뢰가 한 줄짜리 공정이라면, 고단계 강화 의뢰는 제련-합성-제작-강화가 전부 맞물리는 종합 시험이다.
- 결정론의 힘: 실패가 없으므로 **모자랄 수 있는 것은 오직 시간과 준비뿐**이다. 클러치의 긴장은 주사위가 아니라 마감에서 나오고, 브리핑의 예고 — 인벤토리의 비축 — 강화의 결정론이 삼각을 이루어 "예측하고 준비하는" 실력을 보상한다.
- 강화 요구 의뢰의 비율·단계 분포는 후반으로 갈수록 깊어지는 난이도 사다리의 주축이다 (스테이지·밸런스 문서).

## 11. 재도전과 수익 모델

### 11.1 실패와 재도전

- 공략 실패의 결과 화면은 짧고 가볍다 — 팀은 무사히 후퇴하고, "다시" 버튼이 가장 크다. 실패에서 재시작까지 즉시 (브리핑 생략 가능, §3.2).
- 에너지·하트·입장권은 없다. 재도전의 유일한 비용은 플레이어의 시간이다.
- 별이 게이트(§3.1)이자 수입(§9.2)이므로, 재도전은 처벌의 반복이 아니라 **더 나은 계획의 재실행**이다 — 결정론(§10) 덕에, 어제보다 나은 판단은 반드시 어제보다 나은 결과가 된다.

### 11.2 수익 모델 — 유료 선불

- **유료 선불(프리미엄) 1회 구매가 전부다.** 광고는 리워드형을 포함해 일절 없다. 에너지도, 확률형도, 성능 판매도 없다.
- **금지선 — 성능에 현금이 닿지 않는다.** 그리고 이 게임에는 확률 판정이 없으므로, 확률형 상품은 금지 이전에 **성립 자체가 불가능하다** — "이 게임은 정직하다"가 정책이 아니라 구조로 보장된다.
- 추가 결제의 여지는 꾸미기(일꾼·대장간 스킨) 한정 DLC까지만 열어 둔다 — 도입 여부 자체가 미정이며, 기본은 "게임 값 한 번".
- 가격, 데모(라이트판) 제공 여부, 구독형 카탈로그(Apple Arcade·Play Pass 류) 입점은 사업 결정으로 본 문서의 범위 밖이다. 어떤 채널에서든 위 금지선은 유지된다. (배포 형태 확정: **무료 다운로드 + 거래처 1 체험 + 본편 언락 비소모성 IAP** — 결제는 여전히 한 번뿐이다. 상세는 수익 모델 문서 §2)
- **구 리워드 광고 접점의 처리**: 정산 보너스 광고는 광고와 함께 소멸했고, **출정 연기(납품기한 연장)는 부록 B로 보류**되었다 — 광고라는 비용이 사라진 채 무료로 남기면 항상 쓰는 지배 전략이 되어, 납품기한을 그만큼 늘린 것과 같아지기 때문이다. 기한의 무게는 기한으로 지킨다.

## 12. 설계 원칙 — 하위 문서 전체에 대한 헌법

수치와 콘텐츠가 바뀌어도 아래는 유지된다. 충돌하면 위가 이긴다.

1. **한 손, 한 화면.** 모든 플레이는 세로 화면 하나, 엄지 하나로 (UI 문서에 대한 제약).
2. **지시는 사장이, 노동은 일꾼이.** 모든 조작은 탭이고, 실력은 반응 속도가 아니라 일의 순서에서 나온다. 동시성의 상한은 언제나 일꾼 수다.
3. **비자발적 손실의 출처는 과열 하나뿐이다.** 쓰레기통 파기는 선택이고, 그 외의 모든 나쁜 결과는 기회의 상실이지 물건의 상실이 아니다.
4. **확률 판정은 없다.** 같은 판단은 언제나 같은 결과를 낳는다. 무작위성은 의뢰 등장의 변주에만 허용되며, 승패를 가르는 주사위는 존재하지 않는다.
5. **보상은 노동에 비례한다.** 더 높은 점수는 언제나 더 깊은 공정의 대가다 — 강화 요구 의뢰가 그 정점이다 (아이템·밸런스 문서에 대한 제약).
6. **계열 상한제.** 변환 3 · 합성 3 · 제작 3 · 강화 1. 확장은 기계가 아니라 데이터로 — 배울 것은 늘지 않고 만들 것만 늘어난다 (아이템·스테이지 문서에 대한 제약).
7. **성장은 사무소에 쌓인다.** 라운드는 매판 리셋되지만 일꾼·설비·계약·장부는 남는다.
8. **선불 한 번, 광고 없음.** 성능에 현금이 닿는 상품은 어떤 형태로도 없다 — 추가 결제는 있더라도 꾸미기까지.
9. **세션의 완결성.** 라운드는 한 자리에서 완결된다. 에너지 게이트 없음, 언제든 정지, 어디서든 복귀.
10. **실패는 가볍게, 운명은 무겁게.** 팀은 죽지 않고 후퇴한다 — 톤은 귀엽게 (아트 문서에 대한 제약). 하지만 "내 솜씨가 남의 모험을 결정한다"는 무게는 결과 연출에 진심으로 싣는다.
11. **방치 생산은 없다.** 일꾼은 라운드 안에서만 일하며, 게임을 꺼둔 시간이 자원을 만들지 않는다. 수입의 주력은 언제나 플레이다 — 훗날 어떤 수동 수입을 도입하더라도 이 원칙 아래에서만 한다.

## 부록 A. 하위 문서 위임 맵

각 하위 문서(담당 에이전트)의 범위와, 본 문서가 부과하는 제약의 요약이다.

### A-1. 아이템·레시피 설계서
- **범위**: 재료·장비·부품의 전체 목록, 레시피, 기본 점수, 스테이지별 해금 배치, 강화석 레시피
- **제약**: §8.4의 6개 조항 + 계열 상한제(§7.2) + 손실의 원칙(§8.3)

### A-2. 스테이지·라운드 설계서
- **범위**: 거래처(회사) 설정과 서사, 스테이지별 라운드 수·구성, 던전 테마와 의뢰 비중, 강화 요구 의뢰의 배치, 특별 의뢰(단골·이벤트) 할증, 라운드 특수 규칙, 거래처 개방의 누적 별 요건 배치(§3.1), 장부·도감 콘텐츠
- **제약**: 구조는 §3.1 고정 (순차 해금 → 전 라운드 클리어 → 계약 + 누적 별 게이트) / 각 라운드는 의뢰 구성에서 도출된 **입장 요건**(필요 설비)을 명시한다(§3.1) / 라운드에 새 조작을 발명하지 않는다 — 변주는 데이터(의뢰 구성·테마·특수 규칙·다인 작업·부품)로만 / **다인 작업 의뢰는 일꾼 3명 이상 전제 구간부터**(§6) / 브리핑 필수 — 대형·강화 의뢰와 특수 규칙은 브리핑에서 고지 / 판정은 점수로만 (특수 규칙의 점수 산출 변형은 허용, §3.2) / 강화석-강화 시설 해금 동기화 (A-1과 공동)

### A-3. 밸런스·경제 수치서
- **범위**: 모든 수치 — 작업 시간, 과열 유예, 인내, 라운드 길이(납품기한), 의뢰 동시 칸 수, 강화 단계 상한, 별 커트라인, 강화 할증 배율, 별 등급별 정산 골드·첫 클리어 보너스, 누적 별 개방 요건, 게임 내 가격 전반, 일꾼 고용 가격·증가 곡선(인원 범위 2~6은 §6에서 확정), 인벤토리 용량
- **제약**: 강화 요구 의뢰의 점수 할증 ≥ 추가 노동(강화석 제련 + 패스)의 기회비용 — 깊은 공정이 항상 값을 해야 한다(원칙 5) / 공정 깊은 의뢰의 인내는 그 깊이를 감당 가능하게(§9.1) / **누적 별 요건은 3성 완주를 강제하지 않는다** — 권장 수준의 플레이로 도달 가능한 별 총량에 대해 여유를 둔다(§3.1) / 권장 수준의 계약 — 각 라운드는 명시된 권장 일꾼·설비 수준을 전제로 튜닝하고, 막힘의 해소는 재플레이·성장이어야 한다

### A-4. UI·화면 설계서
- **범위**: 화면 구성, 레이아웃, 인벤토리·장부·상점의 표현, 브리핑·결과 화면, 연출
- **제약**: 세로 한 손, 플레이 화면 무스크롤(원칙 1) / 유휴 일꾼 수 상시 표시(§6) / **누적 별 현황과 다음 거래처의 개방 요건이 가도 지도에서 항상 확인 가능할 것**(§3.1) / 라운드 입장 요건과 부족 설비의 표시(§3.1) / 의뢰의 요구 강화 단계와 장비의 현재 단계가 즉시 식별될 것 / 실패→재시작 즉시, 재도전 브리핑 생략(§11.1)

### A-5. 아트·테마 가이드
- **범위**: 그래픽 스타일, 캐릭터(일꾼·레이드 팀·거래처), 월드 테마, 사운드 톤
- **제약**: 귀엽고 폭넓게 사랑받는 방향(§1.1) / 실패는 후퇴로, 잔혹 묘사 금지(원칙 10)

## 부록 B. 보류한 개념 — 확장의 예약석

- **콤보** (납품 리듬에 따른 팁 배율): v1.9에서 삭제. 사유 — 별 판정(점수)과 골드 중 무엇에 곱하는지의 이원화 모호성, 초반 2일꾼 구간에서 병렬 리듬이 성립하는지의 불확실성. 재도입한다면 **별 판정과 분리된 골드 보너스로만**, 그리고 일꾼 3명 이상 구간부터
- **출정 연기** (라운드 막바지의 납품기한 연장): 리워드 광고의 삭제(§11)와 함께 보류. 무료로 남기면 항상 쓰는 지배 전략이 되어 기한 연장과 동치가 된다. 재도입 조건 — 실질적 비용(골드 지불 등)과 라운드당 1회 상한을 동반할 것
- **일꾼의 개성** (전문화·레벨·고유 능력): v1.9의 일꾼은 수만 다르다. 강력한 확장 축이지만 "규칙은 단순하게"를 위해 보류
- **라운드 사이 임시 부스트 선택** (3택 1 드래프트): 라운드 내 의사결정 층의 후보
- **일일 의뢰·기간 이벤트** (출시 후 운영 콘텐츠): 구조만 열어두고 v1.9 제외 (§9.1의 특별 의뢰 할증 조항이 시스템적 근거). 유료 게임 전환으로 광고 유인 목적은 소멸 — 도입한다면 순수한 콘텐츠 업데이트로서
- **계약 정기 수입** (클리어한 거래처가 실시간으로 적립하는 골드): 장기 리텐션 훅 후보였으나 복잡도 대비 우선순위로 보류. 도입 시 조건 — 누적 상한 필수, 활성 플레이 수입 미만, 원칙 11 준수
- **변환·합성 계열의 확장석**: 상한(각 3종) 안의 빈자리
- **던전 속성 시스템** (속성 장비 매칭): 던전 테마를 비중 조절에서 속성 보너스로 심화하는 층
- **실유저 비동기 리그**: 서버 필요. 오프라인 원칙과 별개 트랙
- **협동·PvP 모드**: 이 제작 파이프라인 위에 짓는 프랜차이즈 후속작의 자리

## 부록 C. 미결 사항 — 검증으로 답할 질문 (메커니즘 한정)

1. **투기 생산의 템포** — 브리핑 예고 → 선제 양산 → 인벤토리 비축이 의도된 플레이로 공식화된 지금, 인벤토리 용량 노브만으로 라운드의 긴장이 유지되는지
2. **회생 불가 라운드의 체감** — 조기 정산이 없는 구조에서, 이미 실패가 확실한 라운드를 기한까지 두는 것이 지루함이 되는지 (완화 수단: 짧은 라운드 길이, 일시정지 재시작)
3. **병렬 운영의 자생력** — 콤보 없이 점수 합산만으로 "대형 라인 + 소형 납품 병행"이 자연 발생하는지, 아니면 대형 의뢰 집중이 지배 전략이 되어 라운드 중반이 단조로워지는지 (커트라인 설계와 공동 검증)
4. **대기열 없음 + 취소 불가** — 시작 일꾼이 2명으로 확정된 지금(§6), 초반 구간이 답답함으로 느껴지는 구간이 있는지
5. **강화 요구 의뢰의 성립** — 예고·인내·할증의 조합이 공정 깊이를 실제로 감당하는지 (밸런스 문서와 공동 검증)
6. **입장 요건 벽의 체감** — 필요 설비 구매까지의 골드 거리(= 재플레이 횟수)가 진행의 리듬이 되는지, 그라인드 불만이 되는지 (A-3의 "권장 수준의 계약"과 공동 검증)
7. **누적 별 게이트의 체감** — 별이 모자라 다음 거래처가 닫혔을 때, "어느 라운드를 다시 갈까"의 선택이 전략이 되는지 그라인드가 되는지 (A-3의 "3성 강제 금지" 하한과 공동 검증)
