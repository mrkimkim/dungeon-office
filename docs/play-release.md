## 3. 데이터 처리와 오프라인 계약

### 3.1 앱 내부 처리

현재 MVP 앱은 다음 원칙을 모두 만족해야 한다.

- 계정을 만들거나 로그인하지 않는다.
- 이름, 이메일 주소, 전화번호, 연락처, 위치, 광고 ID, 기기 식별자, 사용 행동, 진단·크래시 로그를 개발자에게 수집·전송하지 않는다.
- 광고, 분석, 원격 크래시 수집, 푸시, 원격 설정, 계정, 클라우드 동기화와 결제 SDK를 포함하지 않는다.
- 플레이 기록, 설정과 진행 중 라운드는 Android 앱 전용 로컬 저장소에만 기록한다.
- 런타임은 인터넷 권한이나 HTTP 요청 없이 첫 실행부터 R5 완료, 저장·복구와 법적 고지 열람까지 동작한다.
- 로컬 진단 정보가 필요하면 기기 안에만 두고, 사용자가 직접 내용을 확인·복사한 경우에만 앱 밖으로 나갈 수 있다.

앱 삭제 또는 Android 설정의 `저장공간 및 캐시 > 저장공간 삭제`는 일반적으로 앱 로컬 데이터를 제거한다. 앱은 자체 클라우드 복구를 제공하지 않으므로 삭제 전 진행을 개발자가 복원할 수 있다고 안내하지 않는다.

### 3.2 Android·제조사 백업의 경계

현재 배포 빌드는 Godot export 설정을 통해 최종 Android manifest에 `android:allowBackup="false"`를 지정한다. 이는 앱 데이터의 Android 자동 백업을 허용하지 않도록 요청하는 현재 구현 경계이며, 별도의 `dataExtractionRules`나 `fullBackupContent` XML을 커스텀하지 않는다. `TECH-MVP-001`이 export·manifest 설정을 소유하고 `QA-MVP-001`이 최종 AAB의 `allowBackup=false`를 검증한다.

다만 Android 12 이상에서의 기기 간 전송과 Android 또는 기기 제조사가 제공하는 전체 기기 백업·복원·복제 도구는 앱 밖의 운영체제 기능이며 제조사별로 동작이 다를 수 있다. 제조사 구현과 사용자 계정 설정에 따라 로컬 앱 데이터가 처리될 가능성을 개발자가 완전히 통제한다고 보장하지 않는다. 이 경우에도 던전 오피스 앱이나 개발자 서버로 데이터가 전송되는 것은 아니며, 해당 플랫폼·백업 제공자의 정책이 적용된다.

### 3.3 지원 이메일의 경계

앱의 `정보·법적 고지` 화면은 지원 이메일을 표시하고 복사할 수 있게 한다. 사용자가 선택하면 OS의 메일 앱 작성 화면을 열 수 있지만 앱이 자동으로 메일을 보내거나 첨부물을 추가하지 않는다. 메일 앱이 없거나 오프라인이어도 게임과 고지 열람은 차단되지 않는다.

사용자가 직접 지원 메일을 보내면 이메일 주소, 본문과 사용자가 선택한 첨부물은 사용자의 메일 서비스와 Gmail을 통해 개발자에게 전달된다. 개발자는 이를 문의 확인과 회신에만 사용하며 판매하거나 광고 목적으로 공유하지 않는다. 지원 필요 또는 법적 보관 필요가 끝나기 전까지 메일 계정에 보관될 수 있고, 삭제 요청은 같은 지원 이메일로 받는다. 메일 제공자의 전송·보관에는 해당 제공자의 개인정보처리방침이 적용된다.

### 3.4 GitHub Pages의 경계

공개 지원·개인정보·오픈소스 페이지는 GitHub Pages가 호스팅한다. 프로젝트는 별도 분석 스크립트, 광고, 문의 폼이나 자체 쿠키를 추가하지 않는다. 다만 사용자가 공개 URL에 접속하면 GitHub가 IP 주소, 브라우저·요청 정보와 서비스 로그를 GitHub의 개인정보처리방침에 따라 처리할 수 있다. 이는 설치된 앱의 데이터 수집이나 개발자 서버 전송이 아니며 Google Play Data safety의 앱 동작 선언과 구분한다.

## 4. 개인정보처리방침과 Data safety 계약

### 4.1 공개·앱 내부 원문

`site/privacy/index.md`가 개인정보처리방침의 유일한 원문이다.

- GitHub Pages는 같은 원문을 공개 URL로 제공한다.
- 앱은 파일을 리소스에 포함해 오프라인으로 표시한다.
- 앱 표시 내용, Play Console URL과 Data safety 답변은 원문의 현재 승인 버전과 일치해야 한다.
- 원문에는 시행일, 앱 식별, 데이터 처리 범위, 로컬 저장·삭제, OS 백업 경계, 지원 메일 경계와 연락처가 있어야 한다.
- 개인정보 처리 방식이 바뀌는 코드나 SDK는 원문과 Data safety 검토가 승인되기 전에 병합·출시하지 않는다.

### 4.2 현재 MVP의 Data safety 기준 답변

| 질문 범주 | 현재 답변 기준 | 검증 증거 |
|---|---|---|
| 앱 또는 포함 SDK가 사용자 데이터를 수집하는가 | 아니요 | 의존성 목록, 권한·네트워크 정적 검사, 오프라인 실기 |
| 사용자 데이터를 제3자와 공유하는가 | 아니요 | SDK·엔드포인트 0건, 네트워크 캡처 0건 |
| 계정을 생성하는가 | 아니요 | 화면·코드·저장 스키마 검사 |
| 광고를 포함하는가 | 아니요 | 의존성·manifest·화면 검사 |
| 앱 내부 결제를 포함하는가 | 아니요 | Billing 의존성·상품 ID·구매 경로 0건 |
| 데이터 삭제 요청 경로 | 앱 로컬 데이터는 Android 앱 데이터 삭제 또는 앱 제거. 자발적 지원 메일은 지원 이메일로 삭제 요청 | 앱 내부 고지와 공개 정책 대조 |

Play Console 질문 문구나 분류가 바뀌면 당시 양식의 의미를 다시 확인하되, “앱이 무엇을 실제로 하는가”라는 위 계약은 바꾸지 않는다. 내부 테스트 트랙이 제출 의무에서 제외되더라도 공개 출시 전에 Data safety와 개인정보 URL을 완성한다.

## 5. 앱 내부 정보·법적 고지 요구

앱은 설정에서 진입 가능한 단일 `정보·법적 고지` 화면을 제공한다. 화면은 네트워크 없이 다음을 표시한다.

1. 앱 이름과 설치된 버전
2. 개인정보처리방침 전문
3. 오픈소스 라이선스 전문
4. 지원 이메일 `2020promoking@gmail.com`과 복사 행동
5. 선택적 `메일 앱 열기` 행동
6. 공개 지원·개인정보 URL의 텍스트와 복사 행동

앱 안의 웹뷰, 원격 페이지 로드, 연결 필수 상태는 금지한다. 외부 메일 앱 또는 브라우저 전환은 사용자의 명시적 탭으로만 시도하며, 실패 시 주소를 그대로 복사할 수 있어야 한다.

## 7. 미래 유료 스테이지의 변경 게이트

`BUS-MVP-001`이 허용 가능성만 남긴 비소모성 후속 스테이지 상품은 현재 앱의 기능이 아니다. 실제 추진 승인이 난 경우 구현 전에 다음을 수행한다.

1. `BUS-MVP-001`에 상품 범위·가격·무료 콘텐츠 불변 조건을 승인한다.
2. 이 문서에서 Billing을 통한 데이터 처리, Play와의 공유, 구매 지원·환불 경계와 Data safety 영향을 재평가한다.
3. `UX-MVP-001`과 `UI-MVP-001`에 구매·대기·취소·실패·복원·오프라인 entitlement 경로를 먼저 승인한다.
4. `TECH-MVP-001`에 Billing·권한 검증·오프라인 권리 캐시·서명 검증을 명세한다.
5. `QA-MVP-001`과 Play Console 제출을 변경한 뒤에만 상품을 등록·출시한다.

현재 MVP에는 위 미래 상태의 코드, 데이터 필드, SDK, 상품 ID, 아이콘, 화면 또는 문구를 미리 만들지 않는다.

## 8. 정상·예외·경계 상태

### 정상 상태

- 사용자는 설치 후 네트워크 없이 게임과 앱 내부 고지를 모두 이용한다.
- 공개 GitHub Pages의 정책과 앱에 포함된 정책의 버전·내용이 일치한다.
- Google Play의 Data safety, 가격, 등록 정보와 최종 AAB가 같은 제품 경계를 설명한다.

### 예외 및 경계 상태

- GitHub Pages 장애·오프라인: 앱 내부 원문은 계속 열리고 게임을 차단하지 않는다.
- 외부 메일 앱 없음·전송 실패: 이메일 복사는 가능하고 게임 상태는 유지된다.
- 외부 브라우저 없음: URL 복사는 가능하고 로컬 원문은 계속 표시된다.
- 사용자가 지원 메일을 발송함: 앱 내부 수집이 아니라 §3.3의 자발적 외부 연락으로 처리한다.
- 백업·복제 도구 사용: §3.2의 플랫폼 처리 경계로 설명하며 자체 클라우드 복원을 약속하지 않는다.
- Data safety와 빌드 동작이 다름: P0 개인정보 결함으로 공개를 차단한다.

## 9. 다른 시스템과의 인터페이스

- `MVP-001`: Android 단독·오프라인 R1~R5라는 출시 범위를 제공한다.
- `BUS-MVP-001`: 무료 획득, 현재 Billing 없음과 미래 유료 콘텐츠의 승인 경계를 제공한다.
- `UX-MVP-001`: 고지 열람, 주소 복사, 외부 앱 전환 실패의 사용자 결과를 정의한다.
- `UI-MVP-001`: canonical 원문을 표시하는 화면과 상태를 정의한다.
- `ART-MVP-001`: 필수 Play 이미지와 실제 화면 캡처 계약을 정의한다.
- `TECH-MVP-001`: application ID, 서명, `allowBackup=false`, 네트워크 금지와 법적 원문 번들을 구현한다.
- `QA-MVP-001`: 최종 AAB·Play Console·GitHub Pages·앱 내부 고지의 일치를 출시 게이트로 검증한다.

## 10. 검증 방법

- 공개 URL 세 곳이 로그인·권한 요청 없이 HTTPS 200으로 열리고 모바일에서 읽히는지 검사한다.
- 앱을 기내 모드로 실행해 개인정보처리방침·라이선스·지원 주소와 설치 버전을 모두 확인한다.
- 앱 원문 파일 해시와 Pages 빌드에 사용된 같은 Git 커밋의 원문 해시를 비교한다.
- 최종 AAB의 권한, manifest, SDK 의존성, 문자열과 네트워크 캡처를 §3·§4와 대조한다.
- Play Console Data safety 제출 사본, 개인정보 URL과 스토어 등록 정보를 릴리스 기록에 보관한다.
- Play Console의 계정·대시보드에서 공개 신청 전에 필요한 실기기 인증, 테스트 인원·기간과 프로덕션 액세스 상태를 확인해 계정별 요구를 기록한다.
- 필수 이미지의 크기·형식·내용이 §6과 실제 후보 빌드에 일치하는지 검사한다.
- 지원 이메일 복사, 메일 앱 있음·없음, 외부 브라우저 있음·없음 상태를 실기에서 검증한다.

## 11. 완료 조건

- 지원 이메일과 공개 URL이 §2의 값으로 동결되어 있다.
- canonical 개인정보·오픈소스 Markdown 원문 묶음이 앱 빌드와 GitHub Pages에서 같은 내용으로 제공된다.
- 최종 AAB에 원격 데이터 수집·공유·광고·분석·계정·Billing·인터넷 권한이 없다는 증거가 있다.
- Play Console의 Data safety와 개인정보처리방침이 최종 AAB 동작과 일치한다.
- 앱 내부 정보·법적 고지 화면의 정상·오프라인·외부 앱 실패 경로가 통과한다.
- 아이콘, 기능 그래픽과 실제 후보 스크린샷을 포함한 필수 등록 산출물이 승인되었다.
- 개발자 표시명이 릴리스 기록에 확정되고 공개 지원 페이지의 표시와 충돌하지 않는다.
- 현재 개발자 계정에 적용되는 신원·기기·테스트와 프로덕션 액세스 과제가 완료되어 있다.
