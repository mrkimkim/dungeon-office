## 1. 목적과 소유 범위

이 문서는 거래처 1곳·5개 라운드 MVP에서 실제로 화면에 등장하는 아트만 제작하기 위한 스타일·상태 언어·에셋 전수·납품 규격을 정의한다. MVP 밖 거래처나 장비 카탈로그를 미리 제작하지 않는다.

이 문서가 Source of Truth인 항목은 다음과 같다.

- MVP의 비주얼 스타일과 상태 색·형태 언어
- 캐릭터, 거래처, 시설, 아이템, UI, VFX의 실제 MVP 에셋 전수
- 스프라이트 크기, 프레임, 파일 형식, 명명·아틀라스 규칙
- 각 화면·게임 이벤트가 요구하는 아트 상태
- 아트 에셋의 제작 완료·검수 기준

이 문서는 다음을 소유하지 않는다.

- 화면의 위치와 레이아웃
- 시설·아이템·레시피·라운드 내용과 해금
- 게임 상태 전이와 밸런스 수치
- 애니메이션 발생 조건·엔진 스케줄링, 사운드·햅틱, 엔진 임포트 코드
- 외주 예산과 별도 캠페인용 키 비주얼·트레일러 에셋

## 2. MVP 비주얼 방향

### 2.1 스타일

- **2D HD 픽셀아트 + 모던 픽셀 UI**로 통일한다.
- 캐릭터는 둥글고 읽기 쉬운 2등신 SD다. 얼굴 세부보다 실루엣과 몸짓을 우선한다.
- 대장간은 따뜻한 화덕 주황과 차분한 무쇠 청회색을 기본 대비로 사용한다.
- 전경 캐릭터·아이템·시설은 짙은 웜 브라운 1px 외곽선을 사용하고 순검정 외곽선은 피한다.
- 배경은 외곽선을 줄이고 명도·채도를 낮춰 인터랙티브 대상을 우선시한다.
- 픽셀 스프라이트는 nearest filtering과 pixel snap을 사용한다. 흐린 보간과 서로 다른 픽셀 밀도 혼용은 금지한다.

### 2.2 기준 픽셀 체계

| 대상 | 원본 규격 | 표시 원칙 |
|---|---:|---|
| 기준 게임 캔버스 | `360 × 640px` | 기준 렌더 `1080 × 1920`에서 정확히 3배. 긴 기기는 배경·안전 여백 확장 |
| 환경 타일 | `32 × 32px` | 같은 씬에서 동일 배율 |
| 시설 원판 | 최대 `64 × 64px` | 3×2 시설 셀 안에서 실루엣 중심 정렬 |
| 일꾼 프레임 | `48 × 48px` | 실제 몸 높이 약 38~42px |
| 아이템 원판 | `32 × 32px` | UI에서 정수배 표시, 축소 시 24px 미만 금지 |
| 상태·기능 아이콘 | `16 × 16px`, `24 × 24px` | 용도별 한 규격만 사용 |
| 거래처 팀 단체 | 최대 `160 × 96px` | 브리핑·결과 전용 |
| UI 9-slice 코너 | 8px 또는 12px | 비정수 확대 금지 |

`360 × 640`은 픽셀 제작과 기준 렌더의 계약이다. `UI-MVP-001`의 반응형 레이아웃은 긴 기기에서 월드 스프라이트를 늘이지 않고 타일·배경 여백으로 대응한다.

### 2.3 팔레트와 상태 언어

MVP 마스터 팔레트는 UI 중간색을 포함해 40색 이내로 유지한다. 아래 의미색은 배타적으로 사용한다.

| 상태 | 색·형태 | 사용 규칙 |
|---|---|---|
| 과열 임박 | 레드 계열 + 점멸 림 + `!` + 짙은 연기 | **레드+점멸의 유일한 사용처** |
| 과열 유예 | 앰버 + 역방향 링 + 모래시계 | 임박 전 단계, 레드 점멸 금지 |
| 작업 중 | 스카이 블루 + 정방향 링 | 안전하게 진행 중 |
| 완료·수납 대기 | 골드 옐로 + 체크 + 트윙클 | 점멸 대신 느린 반짝임 |
| 강화·강화석·`+1` | 자수정 퍼플 + ◆ 한 개 + `+1` 보조 글자 | 별 심볼 사용 금지 |
| 일손 없음 | 앰버 윤곽 + 빈 일꾼 형태 + 좌우 흔들림 | 레드 점멸 금지 |
| 골드 부족 | 동전 윤곽 + 부족분 + 앰버 | 위험·손실 신호처럼 표현하지 않음 |
| 의뢰 철회 | 청록이 회색으로 식음 + 카드 이탈 | 레드, 파손, 감점 파편 금지 |
| 저장 경고 | 남색 파일 형태 + 앰버 `!` | 과열과 다른 실루엣·점멸 없음 |

색 보조 모드에서는 과열에 사선, 완료에 점, 강화에 마름모 패턴을 더한다. 색을 완전히 제거한 캡처에서도 각 상태가 구분되어야 한다.

## 3. MVP 콘텐츠 아트 계약

### 3.1 실제 시설 7종

| 콘텐츠 ID | 화면 위치 | 아트 역할 | 필요한 상태·파생 에셋 |
|---|---|---|---|
| `FAC_SUPPLY` | 시설 격자 | 원자재 공급함 | 열린 선반 본체, 원자재 아이콘 1개·3개 배치용 소켓, 공급 가능 하이라이트 |
| `FAC_FURNACE` | 시설 격자 | 무인 변환·과열의 중심 | 기본, 불 4f, 산출물 받침, 연기 소켓 |
| `FAC_WEAPON_BENCH` | 시설 격자 | 단검·철검 제작대 | 기본, 재료 놓임, 일꾼 작업 소켓 |
| `FAC_SYNTH_BENCH` | 시설 격자 | 강화석 합성대 | 기본, 재료 2개 놓임, 작업 용기 4f |
| `FAC_ENHANCE_ANVIL` | 시설 격자 | 결정론적 `+1` 강화 | 기본, 보라 코어 4f, 타격 플래시 소켓 |
| `FAC_TRASH` | 시설 격자 상단 모서리 | 자발적 파기 | 기본, 뚜껑 3f, 유효 목적지 표시 |
| `FAC_DELIVERY` | 하단 고정 영역 | 의뢰 자동 매칭 납품 | 기본, 상자 닫힘 3f, 도장 소켓 |

시설은 상태별 전체 스프라이트를 중복 제작하지 않는다. 기본 본체 위에 §5의 공용 진행 링, 완료, 과열, 선택 오버레이를 조합한다. 구매 전 `FAC_SYNTH_BENCH`와 `FAC_ENHANCE_ANVIL` 위치가 필요하다면 같은 본체의 단색 분필 실루엣 1장을 사용한다.

### 3.2 실제 아이템 8종

아래 아이콘만 제작한다. 출시판 후보 아이템이나 사용하지 않는 부품 아이콘은 만들지 않는다.

| 콘텐츠 ID | 분류 | 아이콘 핵심 실루엣 | 바탕 형태 |
|---|---|---|---|
| `MAT_IRON_ORE` | 원자재 | 모난 회청색 광석 3덩이 | 원형 재료 태그 |
| `MAT_WOOD` | 원자재 | 짧은 통나무 2개 | 원형 재료 태그 |
| `MAT_MANA_SHARD` | 원자재·강화 경로 | 작은 보라 결정 조각 | 원형 재료 태그 + 보라 중심 |
| `MAT_IRON_INGOT` | 가공재 | 사다리꼴 철 주괴 | 원형 재료 태그 |
| `MAT_CHARCOAL` | 가공재 | 검은 목탄 3조각과 주황 끝 | 원형 재료 태그 |
| `MAT_ENHANCEMENT_STONE` | 가공재·합성 결과 | 큰 자수정 결정 1개 | 고유 보라 결정 태그 |
| `EQ_DAGGER` | 장비 | 45도 단검 | 방패꼴 장비 태그 |
| `EQ_IRON_SWORD` | 장비 | 45도 철검 | 방패꼴 장비 태그 |

`+1` 철검은 별도 아이콘이 아니다. `EQ_IRON_SWORD` 위에 공용 `BADGE_ENHANCE_P1`을 우상단에 합성한다. 같은 뱃지를 의뢰 카드의 요구 단계와 인벤토리 장비의 현재 단계에 사용한다.

### 3.3 일꾼 2명

MVP 일꾼은 같은 공용 바디를 사용하는 대장간 견습 도깨비 2명이다.

| 에셋 | 수량 | 내용 |
|---|---:|---|
| 공용 바디 시트 | 1 | 유휴 4f, 걷기 6f, 운반 6f, 망치질 8f, 합성 6f, 강화 타격 8f |
| 머리 파츠 | 2 | 두건, 작은 뿔. 흑백 실루엣으로 구분 |
| 앞치마 팔레트 | 2 | 청록, 머스터드. 상태 의미색보다 채도 낮게 |
| 유휴 벤치 미니어처 | 공용 조합 | 실제 캐릭터 축소가 아니라 같은 48px 원판을 UI가 크롭해 사용 |

운반은 게임 규칙상 즉시이므로 이동 애니메이션 전체가 0.3초 안에 끝날 수 있게 잔상이 읽히는 자세로 만든다. 망치질의 접촉 프레임과 강화 대타격 프레임에는 VFX·SFX 동기용 프레임 마커를 둔다.

### 3.4 거래처 1곳과 원정 팀

MVP의 단일 거래처는 `CNT-MVP-ROUND-001`의 새싹 원정대다.

| 에셋 | 수량 | 상태 |
|---|---:|---|
| 거래처 로고·인장 | 1세트 | 새싹 돋은 투구 로고, 소형 지도용·대형 계약용 파생 |
| 팀 단체 스프라이트 | 1시트 | 브리핑 대기, 0성 후퇴, 1성 신승, 2성 귀환, 3성 개선 행진 |
| 원정 진입 실루엣 | 1 | 결과 장착 매핑 뒤 던전으로 향하는 공용 측면 컷 |
| 계약 도장 | 1 | 사장 도장과 거래처 봉랍을 분리 레이어로 제공 |

팀은 3인 공용 파티로 구성하고 상징 소품은 새싹 달린 배낭 하나로 한정한다. 실패 상태에서도 부상·붕대·유혈은 없고 그을음과 시무룩한 자세까지만 허용한다.

| 결과 | 빈 별, 채운 별, 신규 기록 광택, 골드 코인, 계약 인장 |

| 상태 | 자물쇠, 설비 부족 공구, 보유 체크, 골드 부족, 수납 화살표, 저장 중, 저장 경고, 복구 파일 |
| 라운드 | 노드 기본, 현재, 잠김, 설비 부족, 완료, 중단 중 오버레이 |
| 의뢰 | 기본 카드, 예고 봉랍, `+1` 뱃지, 납품 완료 도장 |
| 튜토리얼 | 스포트라이트 마스크, 말풍선 9-slice, 손가락 탭 4f, 펄스 링 |

UI 아이콘은 실제 기능만 제작한다. 현재 무료 MVP에 없는 결제, 복원, 광고, 장부, 다른 거래처, 계정, 클라우드 아이콘은 MVP 아틀라스에 포함하지 않는다. 후속 유료 스테이지가 승인되기 전에는 결제 아이콘이나 CTA용 에셋을 선제 제작하지 않는다.

## 5. VFX·상태 오버레이 전수

| VFX ID | 프레임·형식 | 사용 이벤트 | 완료 상태가 전달할 의미 |
|---|---|---|---|
| `VFX_SELECT_RING` | 1 + 알파 펄스 | 아이템 선택 | 현재 출발지 |
| `VFX_VALID_TARGET` | 1 + 스케일 호흡 | 유효 목적지 | 다음 탭 가능 |
| `VFX_COMPLETE_SPARK` | 4f | 작업 완료 | 회수·수납 가능 |
| `VFX_OVERHEAT_STEAM` | 4f loop | 과열 유예 | 돌아와야 함 |
| `VFX_OVERHEAT_DANGER` | 림 2f + `!` | 과열 임박 | 유일한 비자발적 손실 직전 |
| `VFX_OVERHEAT_LOSS` | 연기 6f + 재 3f | 산출물 소실 | 해당 산출물만 사라짐 |
| `VFX_DELIVERY_ARC` | 4f trail | 장비 납품 | 장비가 의뢰로 이동 |
| `VFX_DELIVERY_STAMP` | 4f | 의뢰 완료 | 카드 점수가 합산됨 |
| `VFX_ENHANCE_IMPACT` | 플래시 2f + 링 6f | `+1` 완료 | 실패 없는 단계 상승 |
| `VFX_SCORE_FLY` | 숫자 배경 1 | 납품 점수 | 카드 값이 총점으로 이동 |
| `VFX_STAR_STAMP` | 별당 5f | 판정 | 획득 별 |
| `VFX_CONFETTI_SMALL` | 파티클 소스 2 | 3성·계약 | MVP 최고 성취 |
| `VFX_PURCHASE` | 코인 4f + 설치 먼지 4f | 설비 구매 | 골드 거래와 설치 완료 |
| `VFX_REJECT` | 코드 흔들림 + 앰버 윤곽 | 무효 탭·일손 없음·만석 | 행동 불가, 손실 없음 |

룰렛, 확률 게이지, 실패 직전 페이크, 아이템 파괴 파편은 강화 VFX에 사용하지 않는다.

## 6. 화면 연출 계약

### 6.1 브리핑에서 플레이로

- 브리핑은 아이보리 종이와 낮은 채도다.
- `개시` 뒤 대장간 화면의 화덕이 켜지며 웜톤으로 전환한다.
- 전환은 정보 확인을 방해하지 않으며 플레이 가능 시점보다 길어지지 않는다.

### 6.2 결과와 계약

1. 납품한 장비 아이콘이 팀 단체 스프라이트의 해당 캐릭터 쪽으로 이동한다.
2. 원정 진입 실루엣을 짧게 보여준다.
3. 결과별 팀 포즈와 별 스탬프를 보여준다.
4. R5 성공 시 같은 리포트 위에 거래처 인장과 사장 도장을 찍는다.

납품하지 않은 장비를 팀이 들고 있는 그림을 만들지 않는다. 장착 매핑은 실제 납품 데이터에 따라 아이콘 레이어를 조합한다.

0성은 가장 짧고 가볍게, 3성과 계약은 가장 풍성하게 표현한다. 어떤 결과에도 던전 내부 전투 컷, 부상, 죽음은 없다.

## 7. 파일·아틀라스 규격

### 7.1 소스와 출력

- 원본: Aseprite `.aseprite`, 레이어·태그·프레임 마커 유지
- 런타임: 투명 PNG 스프라이트 시트와 JSON 또는 엔진 리소스 프레임 메타데이터
- 색 공간: sRGB, 프리멀티플라이드 알파 금지
- 필터: nearest, mipmap 기본 끔. 축소가 필요한 정적 배경만 별도 검증 후 허용
- 피벗: 시설은 바닥 중앙, 캐릭터는 발 중앙, 아이템은 중앙, VFX는 사건 중심
- 여백: 스프라이트 시트 셀 사이 2px 이상 extrusion

픽셀 애니메이션의 프레임 cadence는 이 문서가 소유한다. 각 Aseprite 태그에는 아래 FPS와 `audio_marker` 프레임을 메타데이터로 저장하며, 엔진이 임의로 클립 길이를 늘이거나 줄이지 않는다.

| 클립 계열 | 재생 FPS | 반복 |
|---|---:|---|
| 일꾼 유휴·걷기·제작·합성·강화 | 12 | 유휴·걷기·작업은 loop, 강화 완료 타격은 one-shot |
| 용광로 불·합성 용기·강화 코어 같은 시설 ambient | 8 | loop |
| UI 탭·도장·VFX·결과 스탬프 | 12 | one-shot, 명시된 loop만 반복 |

오디오가 맞아야 하는 접촉 프레임은 §3.3과 §5의 마커 이름으로 고정한다. `AUD-MVP-001`은 파일 시작 추정이 아니라 이 마커를 소비하며, 표시 프레임 드롭이 있어도 같은 마커 이벤트를 중복 발화하지 않는다.

### 7.2 경로와 이름

```text
art/mvp/source/
  branding/  characters/  facilities/  items/  ui/  vfx/  backgrounds/

art/mvp/runtime/
  branding/
  atlas_characters.png
  atlas_facilities.png
  atlas_items.png
  atlas_ui.png
  atlas_vfx.png
  backgrounds/
```

파일명은 소문자 snake_case, 애니메이션 태그는 `<content_id>__<state>` 형식을 사용한다. 예: `fac_furnace__fire`, `worker_base__craft`, `eq_iron_sword__icon`.

### 7.3 아틀라스 예산

| 아틀라스 | 최대 크기 | 비고 |
|---|---:|---|
| 캐릭터 | 1024×1024 | 일꾼 2변형 + 팀 1세트 |
| 시설·아이템 | 1024×1024 | 실제 7시설·8아이템만 |
| UI·VFX | 1024×1024 | 9-slice extrusion 확인 |

여백을 채우기 위한 미래 에셋 선제 제작은 금지한다.

## 8. 제작 순서

1. 마스터 팔레트, 상태 언어 샘플, 앱 인장과 로고
2. `FAC_FURNACE`, 일꾼 망치질, `EQ_IRON_SWORD`로 플레이 화면 가독성 스파이크
3. 실제 시설 7종과 아이템 8종
4. 일꾼 전체 모션과 두 파츠
5. 플레이 UI·상태 VFX
6. 지도·상점·브리핑·설정·복구 UI
7. 새싹 원정대 팀, 결과 0~3성, 계약 연출
8. 전체 아틀라스·기기 캡처 검수
9. 승인 후보 빌드에서 `PLAY_PHONE_SHOTS_MVP` 캡처 후 `PLAY_ICON_512`, `PLAY_FEATURE_MVP`와 함께 스토어 제출 검수

스타일 샘플이 실기기에서 통과하기 전에 팀 결과 포즈나 배경을 대량 제작하지 않는다.

## 9. 명시적 비범위

- MVP 카탈로그 밖 거래처·재료·장비·시설·일꾼 에셋
- 자유 배치 모드, 장부, 업적, 스킨, 장식
- 현재 무료 MVP의 결제·페이월·구매 복원 에셋
- `REL-PLAY-001`의 필수 Google Play 아이콘·기능 그래픽·스크린샷 밖의 캠페인 키 비주얼, 장식용 스크린샷 프레임, 트레일러 컷
- 던전 내부 전투와 적 캐릭터

## 10. 검증 방법

- 기준 360×640 캡처를 3배 정수 확대했을 때 모든 픽셀 경계가 선명하다.
- `APP_ICON_MVP`가 Android adaptive icon 마스크에서도 핵심 인장이 잘리지 않고 작은 런처 크기에서 식별된다.
- `PLAY_ICON_512`와 `PLAY_FEATURE_MVP`의 픽셀 크기·형식·파일 크기가 §4.0에 일치한다.
- Google Play 스크린샷이 승인된 Android 후보 빌드의 실제 화면·수치·콘텐츠와 일치한다.
- 휴대전화 스크린샷이 최소 2장이고 각각 §4.0의 크기·비율 제한을 만족하며, 미래 콘텐츠·결제·가짜 UI가 없다.
- 390×844 비율 기기에서 시설·아이템을 비정수 확대하지 않고 배경 확장으로 대응한다.
- 아이템 아이콘 8종을 32px 원판에서 이름 없이 구분할 수 있다.
- 시설 6개 격자를 흑백 실루엣으로 보고 역할 계열을 구분할 수 있다.
- 일꾼 2명을 흑백과 색약 시뮬레이션 모두에서 구분할 수 있다.
- 레드+점멸 검색 결과가 과열 임박 에셋과 상태 정의에만 존재한다.
- 효과음과 햅틱을 끈 상태에서도 작업 완료, 과열 유예·임박·소실, 일손 없음, `+1`을 구분할 수 있다.
- 결과 화면의 장착 아이콘이 실제 납품 목록과 일치한다.
- 아틀라스 manifest의 콘텐츠 ID 집합이 `CNT-ITEM-001`과 `CNT-MVP-ROUND-001`의 MVP 집합과 정확히 일치한다.

## 12. 완료 조건

이 문서는 다음을 모두 만족할 때 `Review`로 전환한다.

- 시설 7종, 아이템 8종, 일꾼 2명, 거래처·팀 1세트가 manifest에 모두 있고 그 밖의 콘텐츠 에셋은 없다.
- `UI-MVP-001`의 모든 화면·컴포넌트·상태가 공통 UI 또는 전용 에셋에 매핑된다.
- 레드+점멸, `+1`, 별, 저장 경고의 의미가 서로 충돌하지 않는다.
- 각 애니메이션과 VFX에 원본 규격, 프레임, FPS, 동기 마커, 피벗, 재사용 규칙이 있다.
- 기준·긴 화면과 색약·무음 조건의 검증 방법이 정의된다.
- 현재 무료 MVP에 없는 결제, 후속 콘텐츠, 별도 마케팅 에셋이 포함되지 않으며 후속 유료 스테이지 승인 전에는 예약 에셋을 만들지 않는다.
- `REL-PLAY-001`이 요구하는 512px 앱 아이콘, 1024×500 기능 그래픽과 실제 후보 스크린샷 최소 2장이 제출 가능 상태다.
