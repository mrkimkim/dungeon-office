## 1. 목적과 소유 범위

이 문서는 거래처 1곳·5개 라운드 MVP에서 실제로 화면에 등장하는 아트만 제작하기 위한 스타일·상태 언어·에셋 전수·납품 규격을 정의한다. MVP 밖 거래처나 장비 카탈로그를 미리 제작하지 않는다.

이 문서가 Source of Truth인 항목은 다음과 같다.

- MVP의 비주얼 스타일과 상태 색·형태 언어
- 캐릭터, 거래처, 시설, 아이템, UI, VFX의 실제 MVP 에셋 전수
- 스프라이트 크기, 프레임, 파일 형식, 명명·아틀라스 규칙
- 각 화면·게임 이벤트가 요구하는 아트 상태
- 아트 에셋의 제작 완료·검수 기준

이 문서는 다음을 소유하지 않는다.

- 화면의 위치와 레이아웃
- 시설·아이템·레시피·라운드 내용과 해금
- 게임 상태 전이와 밸런스 수치
- 애니메이션 발생 조건·엔진 스케줄링, 사운드·햅틱, 엔진 임포트 코드
- 외주 예산과 별도 캠페인용 키 비주얼·트레일러 에셋

## 2. MVP 비주얼 방향

### 2.1 스타일 — 던전 배달 대장간

- **캐주얼 의사 3D로 보이는 2D 렌더 아트**로 통일한다. 런타임은 `Control`과 `Texture2D` 기반이며 `Node3D`, 실시간 3D 카메라·조명·물리 재질을 요구하지 않는다.
- 대장간을 손으로 빚은 장난감 디오라마처럼 표현한다. 시설과 소품은 둥글고 큼직한 실루엣, 완만한 모서리와 과장된 기능 파츠를 사용한다.
- 캐릭터는 둥글고 읽기 쉬운 2등신 SD다. 얼굴 세부보다 실루엣과 몸짓을 우선한다.
- 대장간은 따뜻한 화덕 주황·도색 목재와 차분한 무쇠 청회색을 기본 대비로 사용한다. 표면은 무광 점토, 칠한 나무, 에나멜 금속처럼 단순한 덩어리와 부드러운 명암으로 구분한다.
- 전경 캐릭터·아이템·시설은 순검정 외곽선 대신 짙은 웜 브라운 림과 베이크된 접촉 그림자를 사용한다. 배경은 명도·채도와 세부 밀도를 낮춰 인터랙티브 대상을 우선시한다.
- 빠르게 읽히는 캐주얼 타임 매니지먼트의 가독성만 참고한다. **Overcooked의 캐릭터, 의상, 주방, 로고, UI 배치, 소품 조형과 그 밖의 고유 표현을 모사하지 않는다.**

### 2.2 기준 렌더와 런타임 규격

고해상도 무손실 원본에서 아래 런타임 PNG를 한 번 내려 쓴다. 원본이 런타임 규격보다 작아서는 안 되며, 런타임에서 저해상도 이미지를 확대해 의사 3D 명암을 만들지 않는다.

| 대상 | 런타임 규격 | 표시 원칙 |
|---|---:|---|
| 기준 게임 캔버스 | `360 × 640` 논리 단위 | 긴 기기는 안전 여백과 배경을 확장하고 핵심 배치를 비균일 확대하지 않음 |
| 타이틀 배경 1차 패스 | `720 × 1080px`, RGB PNG | 비율을 유지한 cover 배치. 로고·버튼 가독 영역에 핵심 소품을 두지 않음 |
| 시설 원판 | `384 × 384px`, RGBA PNG | 3×2 시설 셀 안에서 종횡비 유지, 바닥 중앙 피벗, 셀 밖 그림자 침범 금지 |
| 아이템 원판 | `256 × 256px`, RGBA PNG | 중앙 피벗, 24 논리 단위 미만 표시 금지, 이름 없이도 실루엣 구분 |
| 일꾼 원판 | `384 × 384px` 이하, RGBA PNG | 발 또는 작업대 접점 중앙 피벗, 시설보다 앞에 읽히게 배치 |
| 상태·기능 아이콘 | 벡터 또는 고해상도 RGBA PNG | 용도별 형태와 안전 여백을 통일하고 색 하나에 의존하지 않음 |
| 거래처 팀 단체 | `512 × 320px` 이하, RGBA PNG | 브리핑·결과 전용, 전달 장비를 합성할 소켓 보존 |
| UI 패널·버튼 | Godot Theme·9-slice | 최소 탭 영역과 큰 글씨에서 모서리·그림자가 찌그러지지 않음 |

모든 painted sprite는 linear filtering을 사용하고 종횡비를 보존한다. `360 × 640`은 화면 배치의 논리 계약이지 정수배 픽셀 확대 계약이 아니다. `UI-MVP-001`의 반응형 레이아웃은 긴 기기에서 시설 위치와 탭 영역을 유지하고 배경·안전 여백으로 대응한다.

### 2.3 카메라·조명·재질 계약

- 시설·아이템·일꾼은 윗면, 앞면, 오른쪽 면이 보이는 같은 **3/4 직교 시점**을 사용한다. 기준은 약 30~35도 하향의 대각 시점이며 원근 수렴을 넣지 않는다.
- 키라이트는 화면 좌상단의 따뜻한 빛, 낮은 채도의 보조광은 우측, 부드러운 접촉 그림자는 우하단으로 고정한다.
- 같은 분류의 에셋은 바닥 면적과 높이 비율을 맞춘다. 개별 파일에서 카메라 방향·광원·그림자 길이를 임의로 바꾸지 않는다.
- 목재는 넓은 판과 완만한 모서리, 무쇠는 청회색 큰 면과 제한된 하이라이트, 화덕은 주황·골드 중심으로 표현한다. 사실적 녹·기름때·날카로운 파손·미세 PBR 질감은 사용하지 않는다.
- 정상 화염에 위험 레드를 사용하지 않는다. 베이크된 명암이 §2.4의 상태색과 경쟁하면 기본 물성의 채도를 낮춘다.

### 2.4 팔레트와 상태 언어

베이크된 의사 3D 명암에는 연속 톤을 허용하므로 전체 색 수를 제한하지 않는다. 대신 기본 물성 팔레트와 아래 의미색의 색상 계열을 고정하고, 상태색은 배타적으로 사용한다.

| 상태 | 색·형태 | 사용 규칙 |
|---|---|---|
| 과열 임박 | 레드 계열 + 점멸 림 + `!` + 짙은 연기 | **레드+점멸의 유일한 사용처** |
| 과열 유예 | 앰버 + 역방향 링 + 모래시계 | 임박 전 단계, 레드 점멸 금지 |
| 작업 중 | 스카이 블루 + 정방향 링 | 안전하게 진행 중 |
| 완료·수납 대기 | 골드 옐로 + 체크 + 트윙클 | 점멸 대신 느린 반짝임 |
| 강화·강화석·`+1` | 자수정 퍼플 + ◆ 한 개 + `+1` 보조 글자 | 별 심볼 사용 금지 |
| 일손 없음 | 앰버 윤곽 + 빈 일꾼 형태 + 좌우 흔들림 | 레드 점멸 금지 |
| 골드 부족 | 동전 윤곽 + 부족분 + 앰버 | 위험·손실 신호처럼 표현하지 않음 |
| 의뢰 철회 | 청록이 회색으로 식음 + 카드 이탈 | 레드, 파손, 감점 파편 금지 |
| 저장 경고 | 남색 파일 형태 + 앰버 `!` | 과열과 다른 실루엣·점멸 없음 |

색 보조 모드에서는 과열에 사선, 완료에 점, 강화에 마름모 패턴을 더한다. 색을 완전히 제거한 캡처에서도 각 상태가 구분되어야 한다.

## 3. MVP 콘텐츠 아트 계약

### 3.0 현재 1차 아트 패스

현재 저장소에 포함한 첫 표현 패스는 테마와 콘텐츠 식별성을 실기에서 검증하기 위한 **정적 런타임 PNG 16개**다. 최종 캐릭터 애니메이션·상태 VFX·스토어 제출 에셋 완료를 의미하지 않는다.

| 묶음 | 수량 | 런타임 경로와 포함 항목 |
|---|---:|---|
| 타이틀 배경 | 1 | `art/mvp/runtime/backgrounds/bg_title_forge.png` |
| 시설 | 7 | `art/mvp/runtime/facilities/`의 공급함·용광로·무기 제작대·합성 작업대·강화 모루·쓰레기통·납품대 |
| 아이템 | 8 | `art/mvp/runtime/items/`의 원자재 3종·가공재 3종·장비 2종 |

시설과 아이템 파일명은 §3.1~§3.2의 콘텐츠 ID를 소문자 snake_case로 투영한다. 이 에셋은 게임 규칙이나 콘텐츠 정의를 소유하지 않으며, UI는 ID→표현 매핑으로만 참조한다. 누락 에셋 때문에 시뮬레이션·저장 데이터를 바꾸거나 미래 콘텐츠용 대체 아이콘을 추가하지 않는다.

검토한 외부 CC0 후보는 현재 1차 런타임 패스에 포함하지 않았다. 따라서 이 16개 파일 때문에 추가되는 제3자 에셋 고지 의무는 없다. 이후 외부 에셋·폰트를 실제 번들하면 출처·버전·라이선스·수정 여부를 기록하고 `REL-PLAY-001`의 앱 내부·Pages 고지를 같은 변경에서 갱신한다.

### 3.3 일꾼 2명

MVP 일꾼은 같은 공용 바디를 사용하는 대장간 견습 도깨비 2명이다.

| 에셋 | 수량 | 내용 |
|---|---:|---|
| 공용 바디 시트 | 1 | 유휴 4f, 걷기 6f, 운반 6f, 망치질 8f, 합성 6f, 강화 타격 8f |
| 머리 파츠 | 2 | 두건, 작은 뿔. 흑백 실루엣으로 구분 |
| 앞치마 팔레트 | 2 | 청록, 머스터드. 상태 의미색보다 채도 낮게 |
| 유휴 벤치 미니어처 | 공용 조합 | 실제 캐릭터 축소가 아니라 같은 일꾼 원판을 UI가 크롭해 사용 |

운반은 게임 규칙상 즉시이므로 이동 애니메이션 전체가 0.3초 안에 끝날 수 있게 잔상이 읽히는 자세로 만든다. 망치질의 접촉 프레임과 강화 대타격 프레임에는 VFX·SFX 동기용 프레임 마커를 둔다.

### 3.4 거래처 1곳과 원정 팀

MVP의 단일 거래처는 `CNT-MVP-ROUND-001`의 새싹 원정대다.

| 에셋 | 수량 | 상태 |
|---|---:|---|

팀은 3인 공용 파티로 구성하고 상징 소품은 새싹 달린 배낭 하나로 한정한다. 실패 상태에서도 부상·붕대·유혈은 없고 그을음과 시무룩한 자세까지만 허용한다.

## 6. 화면 연출 계약

### 6.1 브리핑에서 플레이로

- 브리핑은 아이보리 종이와 낮은 채도다.
- `개시` 뒤 대장간 화면의 화덕이 켜지며 웜톤으로 전환한다.
- 전환은 정보 확인을 방해하지 않으며 플레이 가능 시점보다 길어지지 않는다.

### 6.2 결과와 계약

1. 납품한 장비 아이콘이 팀 단체 스프라이트의 해당 캐릭터 쪽으로 이동한다.
2. 원정 진입 실루엣을 짧게 보여준다.
3. 결과별 팀 포즈와 별 스탬프를 보여준다.
4. R5 성공 시 같은 리포트 위에 거래처 인장과 사장 도장을 찍는다.

납품하지 않은 장비를 팀이 들고 있는 그림을 만들지 않는다. 장착 매핑은 실제 납품 데이터에 따라 아이콘 레이어를 조합한다.

0성은 가장 짧고 가볍게, 3성과 계약은 가장 풍성하게 표현한다. 어떤 결과에도 던전 내부 전투 컷, 부상, 죽음은 없다.

## 7. 파일·아틀라스 규격

### 7.1 소스와 출력

- 원본: 런타임 규격 이상의 무손실 PNG 또는 레이어가 보존된 편집 원본. 생성·편집 이력과 공통 카메라·광원 기준을 함께 보관한다.
- 1차 런타임: 배경은 RGB PNG, 시설·아이템은 개별 RGBA PNG. 시설은 `384×384`, 아이템은 `256×256`으로 내보낸다.
- 후속 애니메이션: 투명 PNG 프레임 시트 또는 프레임 시퀀스와 JSON·엔진 리소스 메타데이터를 사용한다.
- 색 공간: sRGB, 프리멀티플라이드 알파 금지
- 필터: linear. UI·아이템·시설 mipmap은 기본 끄고, 축소가 큰 정적 배경만 실기 검증 후 허용한다.
- 피벗: 시설은 바닥 중앙, 캐릭터는 발 중앙, 아이템은 중앙, VFX는 사건 중심
- 여백: 개별 PNG는 그림자와 하이라이트가 잘리지 않는 투명 안전 여백을 둔다. 아틀라스 전환 시 셀 사이 2px 이상 extrusion을 적용한다.

현재 §3.0의 1차 패스는 정적 에셋이며 아래 애니메이션 프레임을 포함하지 않는다. 후속 애니메이션을 추가하면 클립 이름, 아래 FPS와 `audio_marker` 프레임을 메타데이터로 저장하고 엔진이 임의로 클립 길이를 늘이거나 줄이지 않는다.

| 클립 계열 | 재생 FPS | 반복 |
|---|---:|---|
| 일꾼 유휴·걷기·제작·합성·강화 | 12 | 유휴·걷기·작업은 loop, 강화 완료 타격은 one-shot |
| 용광로 불·합성 용기·강화 코어 같은 시설 ambient | 8 | loop |
| UI 탭·도장·VFX·결과 스탬프 | 12 | one-shot, 명시된 loop만 반복 |

오디오가 맞아야 하는 접촉 프레임은 §3.3과 §5의 마커 이름으로 고정한다. `AUD-MVP-001`은 파일 시작 추정이 아니라 이 마커를 소비하며, 표시 프레임 드롭이 있어도 같은 마커 이벤트를 중복 발화하지 않는다.

### 7.2 경로와 이름

```text
art/mvp/source/
  branding/  characters/  facilities/  items/  ui/  vfx/  backgrounds/

art/mvp/runtime/
  branding/  backgrounds/  characters/  facilities/  items/  ui/  vfx/
```

파일명은 소문자 snake_case를 사용한다. 정적 콘텐츠 파일은 소문자 콘텐츠 ID와 같은 이름을 사용한다. 예: `fac_furnace.png`, `eq_iron_sword.png`. 후속 애니메이션 클립 이름은 `<content_id>__<state>` 형식이다. 예: `fac_furnace__fire`, `worker_base__craft`.

### 7.3 개별 PNG와 아틀라스 전환

§3.0의 1차 패스는 검수와 교체가 쉬운 개별 PNG를 런타임 정본으로 사용한다. 카메라·조명·실루엣과 표시 크기가 실기에서 고정되기 전에는 빈 공간을 줄이기 위한 아틀라스 패킹을 하지 않는다.

아틀라스 전환은 다음 조건을 모두 만족한 뒤 같은 표현만 기계적으로 패킹하는 최적화 작업으로 수행한다.

1. 시설 7종·아이템 8종의 개별 PNG가 §10의 기기·가독성 검증을 통과한다.
2. 아틀라스 전후 개별 콘텐츠의 표시 크기, 피벗, 색, 알파와 linear filtering 결과가 같다.
3. ID→영역 manifest가 `CNT-ITEM-001`의 실제 집합과 정확히 일치한다.
4. 메모리·로드 시간 개선이 `QA-MVP-001` 측정으로 확인된다.

| 아틀라스 | 최대 크기 | 비고 |
|---|---:|---|
| 캐릭터 | 1024×1024 | 일꾼 2변형 + 팀 1세트 |
| 시설·아이템 | 1024×1024 | 실제 7시설·8아이템만 |
| UI·VFX | 1024×1024 | 9-slice extrusion 확인 |

아틀라스는 개별 파일을 삭제하거나 콘텐츠 ID를 바꾸는 근거가 아니다. 여백을 채우기 위한 미래 에셋 선제 제작은 금지한다.

## 10. 검증 방법

- §3.0의 타이틀 1장·시설 7종·아이템 8종이 정확한 경로·파일 수·색상 모드와 `720×1080`·`384×384`·`256×256` 규격을 가진다.
- 시설·아이템은 투명 가장자리의 검은 halo, 잘린 접촉 그림자, nearest 계단 현상 없이 linear filtering으로 표시된다.
- 360×640과 390×844 캡처에서 타이틀 배경은 종횡비를 유지하고 시설·아이템은 셀과 탭 영역을 밀어내거나 가리지 않는다.
- 시설 7종과 아이템 8종의 카메라 방향, 좌상단 키라이트, 우하단 접촉 그림자와 재질 단순화가 같은 세트로 보인다.
- `APP_ICON_MVP`가 Android adaptive icon 마스크에서도 핵심 인장이 잘리지 않고 작은 런처 크기에서 식별된다.
- `PLAY_ICON_512`와 `PLAY_FEATURE_MVP`의 픽셀 크기·형식·파일 크기가 §4.0에 일치한다.
- Google Play 스크린샷이 승인된 Android 후보 빌드의 실제 화면·수치·콘텐츠와 일치한다.
- 휴대전화 스크린샷이 최소 2장이고 각각 §4.0의 크기·비율 제한을 만족하며, 미래 콘텐츠·결제·가짜 UI가 없다.
- 아이템 아이콘 8종을 최소 표시 크기에서 이름 없이 구분할 수 있다.
- 시설 6개 격자를 흑백 실루엣으로 보고 역할 계열을 구분할 수 있다.
- 일꾼 2명을 흑백과 색약 시뮬레이션 모두에서 구분할 수 있다.
- 레드+점멸 검색 결과가 과열 임박 에셋과 상태 정의에만 존재한다.
- 효과음과 햅틱을 끈 상태에서도 작업 완료, 과열 유예·임박·소실, 일손 없음, `+1`을 구분할 수 있다.
- 결과 화면의 장착 아이콘이 실제 납품 목록과 일치한다.
- 개별 PNG의 콘텐츠 ID 집합이 `CNT-ITEM-001`과 `CNT-MVP-ROUND-001`의 MVP 집합과 정확히 일치하며, 아틀라스를 만든 경우 manifest도 같은 집합이다.
- 현재 런타임 번들에 외부 CC0 후보나 출처 불명의 대체 에셋이 없고, 특정 상용 게임의 고유 표현을 모사하지 않았음을 리뷰한다.

## 12. 완료 조건

이 문서는 다음을 모두 만족할 때 `Review`로 전환한다.

- 시설 7종, 아이템 8종, 일꾼 2명, 거래처·팀 1세트가 manifest에 모두 있고 그 밖의 콘텐츠 에셋은 없다.
- `UI-MVP-001`의 모든 화면·컴포넌트·상태가 공통 UI 또는 전용 에셋에 매핑된다.
- 모든 시설·아이템·캐릭터가 같은 3/4 직교 카메라, 광원, 재질 문법과 linear filtering을 사용한다.
- 레드+점멸, `+1`, 별, 저장 경고의 의미가 서로 충돌하지 않는다.
- 각 애니메이션과 VFX에 원본 규격, 프레임, FPS, 동기 마커, 피벗, 재사용 규칙이 있다.
- 기준·긴 화면과 색약·무음 조건의 검증 방법이 정의된다.
- 개별 PNG를 유지하거나 §7.3 조건을 통과한 아틀라스로 전환하며, 패킹을 위해 ID·표시·피벗을 바꾸지 않는다.
- 현재 무료 MVP에 없는 결제, 후속 콘텐츠, 별도 마케팅 에셋이 포함되지 않으며 후속 유료 스테이지 승인 전에는 예약 에셋을 만들지 않는다.
- 외부 에셋·폰트를 번들한 경우 필요한 배포 권한과 고지가 `REL-PLAY-001`에 반영되고, 포함하지 않은 후보를 고지 대상으로 오인하지 않는다.
- `REL-PLAY-001`이 요구하는 512px 앱 아이콘, 1024×500 기능 그래픽과 실제 후보 스크린샷 최소 2장이 제출 가능 상태다.
