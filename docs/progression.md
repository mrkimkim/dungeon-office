## 1. 목적과 범위

이 문서는 라운드 진입 전과 라운드 결과 수신 후에 MVP 진행 상태가 어떻게 바뀌는지를 정의한다. MVP의 장기 단위는 거래처 한 곳과 그 거래처의 라운드 다섯 개다.

이 문서가 답해야 하는 질문은 다음과 같다.

- 어떤 순서로 라운드가 열리는가?
- 라운드 점수는 실패와 별 1~3개로 어떻게 번역되는가?
- 재도전이 기록과 진행을 어떻게 바꾸는가?
- 어떤 조건에서 MVP 거래처와 계약하는가?
- 무엇이 라운드 사이에 남는가?

## 2. 명시적 비범위

- 라운드 안의 작업·의뢰·점수 누적: `SYS-CORE-001`
- 거래처와 라운드의 실제 콘텐츠: `CNT-MVP-ROUND-001`
- 커트라인 숫자: `BAL-MVP-001`
- 골드 지급액과 설비 구매: `SYS-ECON-001`, `BAL-MVP-001`
- 저장 파일·복구·마이그레이션: `TECH-DATA-001`
- 화면 전이와 결과 연출: `UX-MVP-001`, `UI-MVP-001`
- 복수 거래처, 누적 별 게이트, 계약 승급, 도감 확장
- 상업 모델

## 3. 진행 단위

| ID | 정의 |
|---|---|
| `CLIENT_MVP_01` | MVP에서 유일한 거래처다. |
| `R1`~`R5` | 거래처의 다섯 라운드이며 고정된 순서로 진행한다. |
| 클리어 | 해당 라운드에서 별 1개 이상을 받은 상태다. |
| 계약 | `R1`~`R5`를 모두 클리어한 상태다. MVP의 최종 진행 목표다. |

## 4. 해금 규칙

### 4.1 순차 해금

- 새 프로필은 `R1`만 입장할 수 있다.
- `Rn`을 처음 클리어하면 `R(n+1)`이 열린다.
- `R5`에는 다음 두 조건이 모두 필요하다.
  1. `R4` 클리어
  2. 영구 상태 `enhancement_capability_owned = true`
- `enhancement_capability_owned`는 게임 내 골드 구매로 얻는 진행 능력이며 상업 구매 권한이 아니다. 미래의 유료 스테이지 접근 판정에 재사용하지 않는다.
- 이미 열린 라운드는 언제든 다시 플레이할 수 있다.
- 별 누계나 재화 소비를 라운드 해금 조건으로 사용하지 않는다.

### 4.2 입장 검증

입장 요청은 다음 순서로 검증한다.

1. 라운드가 순차 진행으로 열렸는가?
2. 해당 라운드가 요구하는 영구 능력 플래그가 있는가?
3. 중단 저장이 있다면 새로 시작할지 이어 할지 UX에서 결정되었는가?

조건을 충족하지 못하면 입장은 거절되고 구체 사유를 반환한다. `R5`의 강화 능력 부족 외에는 MVP에 영구 능력 입장 게이트가 없다.

## 5. 라운드 결과 판정

`SYS-CORE-001`이 제한 시간 종료 결과를 전달하면 라운드별 세 커트라인과 비교한다. 커트라인 값은 `BAL-MVP-001`이 소유한다.

| 조건 | 판정 | 별 |
|---|---|:---:|
| 점수 < `cutline_1` | 공략 실패 | 0 |
| `cutline_1` ≤ 점수 < `cutline_2` | 신승 | 1 |
| `cutline_2` ≤ 점수 < `cutline_3` | 안정 공략 | 2 |
| `cutline_3` ≤ 점수 | 완벽 공략 | 3 |

- 평가는 제한 시간 종료 시 한 번만 한다.
- 조기 성공·조기 실패로 라운드를 종료하지 않는다.
- 재시작이나 중도 이탈에는 결과 판정이 없다.
- 별 이외의 콤보·등급·보조 판정은 없다.

## 6. 기록 규칙

각 라운드는 다음 영속 기록을 가진다.

- 최고 점수
- 최고 별
- 최초 클리어 여부
- 플레이 완료 횟수

새 결과가 기존 결과보다 낮아도 최고 점수와 최고 별은 내려가지 않는다. 같은 별에서 점수가 더 높으면 최고 점수만 갱신한다. 골드 정산은 최고 기록이 아니라 방금 끝난 판정으로 계산하며 세부는 `SYS-ECON-001`이 소유한다.

## 7. 재도전과 재시작

- 실패와 성공 모두 같은 라운드를 즉시 재도전할 수 있다.
- 재도전에는 에너지, 입장권, 골드 비용이 없다.
- 재도전은 해당 라운드의 초기 상태에서 시작한다.
- 의뢰는 전량 스크립트이므로 재도전해도 동일한 순서와 시점으로 등장한다.
- 메뉴의 재시작은 현재 판을 폐기하고 초기 상태로 돌아간다. 완료 횟수·점수·별·골드에 영향을 주지 않는다.
- 결과가 나온 재도전은 평소와 동일하게 기록과 골드 정산을 처리한다.

## 8. 계약과 MVP 종료 상태

- `R5`를 별 1개 이상으로 처음 완료하면 `CLIENT_MVP_01`과 계약한다.
- 계약은 영구 상태이며 취소되거나 강등되지 않는다.
- 계약 시 MVP 완료 상태를 기록한다.
- 계약 뒤에도 R1~R5 재도전과 기록 향상은 가능하다.
- 계약은 추가 골드 생산이나 패시브 효과를 만들지 않는다.
- MVP에는 계약 후 새 거래처나 새 라운드가 없다.

## 9. 영속 상태와 라운드 리셋

### 9.1 라운드 사이에 남는 상태

- 열린 라운드
- 라운드별 최고 점수·최고 별·최초 클리어·완료 횟수
- 계약 및 MVP 완료 여부
- 골드와 `enhancement_capability_owned` 상태 (`SYS-ECON-001`)
- 사용자 설정과 중단 저장의 존재 여부 (`TECH-DATA-001`)

### 9.2 새 라운드에서 초기화되는 상태

- 재료, 강화석, 장비와 강화 단계
- 인벤토리 내용
- 시설 입력·작업·산출물
- 활성·대기 의뢰
- 라운드 점수, 철회 수, 과열 소실 수
- 일꾼 배정

## 10. 상태 전이

```text
새 프로필
  -> R1 열림
  -> R1 클리어 -> R2 열림
  -> R2 클리어 -> R3 열림
  -> R3 클리어 -> R4 열림
  -> R4 클리어 -> R5 잠금 조건 표시
  -> enhancement_capability_owned -> R5 열림
  -> R5 클리어 -> 계약 + MVP 완료
```

`SYS-ECON-001`은 R4 클리어 상태를 소비해 강화 능력 획득 경로를 제공한다. 진행 문서는 그 경로의 가격이나 상품 구성을 소유하지 않는다.

## 11. 시스템 인터페이스

| 대상 | 이 문서가 제공하는 것 | 이 문서가 받는 것 |
|---|---|---|
| `SYS-CORE-001` | 입장 승인과 새 라운드 초기화 요청 | 종료 점수와 통계 |
| `SYS-ECON-001` | 별 판정, 최초 클리어 여부, R4 클리어 상태, R5 요구 능력 | 골드 정산과 `enhancement_capability_owned` 상태 |
| `CNT-MVP-ROUND-001` | R1~R5 순차 구조와 R5 설비 조건 | 각 라운드 ID와 요구 설비 |
| `BAL-MVP-001` | 판정 구조 | 라운드별 세 커트라인 |
| `UI-MVP-001` | 열림·잠김·완료·계약 상태와 거절 사유 | 입장·재도전·재시작 의도 |
| `TECH-SIM-001` | 입장·결과 판정에 필요한 의미 | 결과 패킷의 실행 타입 |
| `TECH-DATA-001` | 영속 상태 목록 | 저장·복원·정산 트랜잭션 결과 |

## 12. 예외 및 경계 상태

- 점수가 커트라인과 같으면 해당 별을 획득한다.
- R4를 클리어했지만 강화 능력을 획득하지 않았다면 R5는 잠금 상태이며 필요한 능력 사유를 제공한다.
- 강화 설비 구매 후 저장이 정상 완료되기 전 앱이 종료되는 경우의 원자성은 `TECH-DATA-001`이 보장한다.
- 중단 저장 버전이 호환되지 않을 때 해당 판을 재시작하는 UX는 `TECH-DATA-001`과 `UX-MVP-001`이 정의한다. 영속 최고 기록은 손상시키지 않는다.
- R5 결과 처리에서 계약 저장이 실패하면 완료 화면이 영속 성공으로 표시되어서는 안 된다.

## 13. 미결정 사항

| 질문 | MVP 기본값 | 결정 방법 |
|---|---|---|
| 성공 결과에서 브리핑을 생략하고 즉시 재도전할 수 있는가? | 허용 여부를 UX에 위임한다. | `UX-MVP-001` 사용성 테스트 |
| 완료 횟수를 사용자에게 표시할 것인가? | 저장하되 표시하지 않는다. | QA·분석 필요성이 생기면 UI 개정 |

## 14. 검증 방법

- R1~R5를 별 1개로만 클리어해도 계약까지 갈 수 있어야 한다.
- 어떤 라운드도 별 2개나 3개를 다음 라운드의 조건으로 요구하지 않아야 한다.
- 재도전의 낮은 결과가 최고 기록을 낮추지 않아야 한다.
- 재시작이 완료 횟수·보상·기록을 바꾸지 않아야 한다.
- R5 클리어와 계약 상태가 하나의 영속 트랜잭션으로 처리되어야 한다.

## 15. 완료 기준

- R1~R5의 모든 잠금·해금·클리어 상태가 정의되어 있다.
- 점수에서 별로의 경계 조건과 기록 갱신 규칙이 명확하다.
- 경제 문서와 코어 문서 사이의 입출력이 중복 없이 정의되어 있다.
- MVP 종료 조건이 계약 하나로 명확하다.
- 복수 거래처와 상업 모델을 포함하지 않는다.
